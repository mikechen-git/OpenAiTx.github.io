简介

[!IMPORTANT] 本项目持续更新中！

本项目提供了许多使用 Zig 实现的恶意软件技术，因为我是它的超级粉丝。你可以用此仓库将 Zig 武器化。Black-Hat-Zig 持续更新，力求包含更多内容。如果你能为本项目提交 PR，将会非常完美。

好了，让我们一起 hack the planet 吧！

为什么选择 Zig?

  • 🤝 与 C/C++ 源码轻松交互
  • 🔎 更为新颖，因而更难被检测
  • 💪 极低级别的控制，甚至低于 C
  • 😱 由于现代编译器，逆向难度更高
  • 🛡️ 极少的运行时依赖——无庞大的标准库负担
  • 🎯 无未定义行为——显式处理边界情况，防止崩溃

编译代码

你可以在每个目录下查看代码。如果代码使用了 Windows API，你会在对应项目目录的 README.md 中看到提示，例如这个。

对于需要 Windows API 的项目，我已经将默认构建目标设置为 windows-x86_64，因此你可以在 Linux、Windows、甚至 MacOS 上直接复制粘贴以下命令进行编译！（但你仍然需要 Windows 环境来运行可执行文件）

zig build

载荷混淆

通过混淆技术伪装载荷，避免静态分析或特征匹配检测。

    • IPv6 反混淆

    • MAC 地址反混淆
  • UUID 混淆
    • UUID 混淆
    • UUID 反混淆

载荷加密

用于在存储或传输过程中隐藏载荷真实意图的各种加密方案。

  • XOR 加密
    • 使用标准库的 XOR
  • RC4 加密
    • 使用 SystemFunction032 的 RC4
  • AES 加密
    • 通过 Bcrypt.h 的 AES
    • 通过标准库的 AES
    • 通过 TinyAES 的 AES

恶意软件技术

一系列适配于 Zig 的常见进攻性技术。

  • 进程注入

    • Shellcode 注入
  • 载荷分阶段

    • 使用 EnumProcesses

恶意开发工具

  • ZYRA: 你的运行时护甲
    • 一个用 Zig 编写的可执行文件加壳器
  • ZYPE: 你的载荷加密器
    • 生成包含你加密/混淆载荷及其解密/反混淆函数的 Zig 代码模板。
