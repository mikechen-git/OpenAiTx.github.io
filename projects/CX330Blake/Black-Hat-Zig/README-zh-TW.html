介紹

[!IMPORTANT] 本專案持續更新中！

本專案提供許多利用 Zig 實現的惡意軟體技術，因為我非常喜愛這個語言。你可以利用本倉庫將 Zig 武器化。Black-Hat-Zig 持續更新，以確保內容盡可能豐富。如果你能為這個專案建立 PR 就太完美了。

好了，讓我們一起 hack the planet！

為什麼選擇 Zig?

  • 🤝 容易與 C/C++ 原始碼互動
  • 🔎 它較新，因此較難被偵測
  • 💪 強大的低階控制，比 C 還要低階
  • 😱 由於現代編譯器，逆向工程更困難
  • 🛡️ 最小化運行時依賴——沒有龐大的標準函式庫負擔
  • 🎯 沒有未定義行為——明確處理邊界情況以防止崩潰

你可以在每個目錄下檢查程式碼。如果程式碼使用了 Windows API，你會在對應的專案目錄 README.md 中看到提示，例如，這個。

我已為那些需要 Windows API 的程式碼設置了預設的建構目標為 windows-x86_64，所以你可以輕鬆複製並貼上下列指令，在 Linux、Windows 甚至 MacOS 上編譯程式碼！（但你仍然需要 Windows 環境才能執行產生的可執行檔）

有效載荷放置

將惡意有效載荷放置於可執行檔不同區段的技術。

  • .data 區段
  • .rdata 區段
  • .text 區段
  • .rsrc 區段

有效載荷混淆

用於偽裝有效載荷、避開靜態分析或模式匹配偵測的混淆技術。

  • IP 位址混淆
    • IPv4 混淆
    • IPv4 反混淆
    • IPv6 混淆
    • IPv6 反混淆
  • MAC 位址混淆
    • MAC 位址混淆
    • MAC 位址反混淆
  • UUID 混淆
    • UUID 混淆
    • UUID 反混淆

有效載荷加密

用於在儲存或傳輸過程中隱藏有效載荷真實意圖的各種加密方案。

  • XOR 加密
    • 使用標準函式庫的 XOR
  • RC4 加密
    • 使用 SystemFunction032 的 RC4
  • AES 加密
    • 使用 Bcrypt.h 的 AES
    • 使用標準函式庫的 AES
    • 使用 TinyAES 的 AES

  • Shellcode

惡意軟體技術

收錄各種常見進攻技術，並以 Zig 實現。

  • 行程注入
    • DLL 注入
    • Shellcode 注入
  • 有效載荷分段
    • 來自 Web 伺服器
    • 來自 Windows 登錄檔
  • 行程列舉
    • 使用 CreateToolhelp32Snapshot
    • 使用 EnumProcesses
    • 使用 NtQuerySystemInformation

惡意開發工具

  • ZYRA: 你的運行時防護
    • 一個以 Zig 撰寫的可執行檔打包器
  • ZYPE: 你的有效載荷加密器
    • 產生 Zig 程式碼範本，內含你的加密／混淆有效載荷及對應的解密／還原函式。
