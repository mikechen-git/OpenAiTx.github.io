  • 🛡️ ขึ้นกับ runtime น้อย ไม่มีภาระจากไลบรารีมาตรฐานขนาดใหญ่

การคอมไพล์โค้ด

คุณสามารถตรวจสอบโค้ดได้ภายในแต่ละไดเรกทอรี หากโค้ดนั้นใช้ Windows API คุณจะเห็นคำแนะนำในไฟล์ README.md ของโปรเจกต์นั้น ๆ เช่น ตัวอย่างนี้.

ผมได้ตั้งค่า target เริ่มต้นเป็น windows-x86_64 สำหรับโค้ดที่ต้องใช้ Windows API ไว้แล้ว คุณสามารถ copy & paste คำสั่งด้านล่างเพื่อคอมไพล์โค้ดได้เลย ไม่ว่าคุณจะอยู่บน Linux, Windows หรือแม้แต่ MacOS! (แต่คุณยังคงต้องใช้ Windows ในการรันไฟล์ที่คอมไพล์ออกมา)

  • .data Section
  • .rdata Section
  • .text Section
  • .rsrc Section

การทำให้ Payload ยากต่อการตรวจจับ

เทคนิคการทำให้ payload ถูกปกปิดและหลีกเลี่ยงการตรวจจับด้วย static analysis หรือ pattern matching

  • การปกปิด IP Address
    • การปกปิด IPv4
    • การแปลง IPv4 กลับ
    • การปกปิด IPv6
    • การแปลง IPv6 กลับ
  • การปกปิด MAC Address
    • การปกปิด MAC Address
    • การแปลง MAC Address กลับ
  • การปกปิด UUID
    • การปกปิด UUID
    • การแปลง UUID กลับ

การเข้ารหัส Payload

รูปแบบการเข้ารหัสต่าง ๆ เพื่อซ่อนเจตนาที่แท้จริงของ payload ในขณะจัดเก็บหรือส่งข้อมูล

  • XOR Encryption
    • XOR ด้วย Standard Library
  • RC4 Encryption
    • RC4 กับ SystemFunction032
  • AES Encryption
    • AES ด้วย Bcrypt.h
    • AES ด้วย Standard Library
    • AES ด้วย TinyAES

  • Shellcode

  • Reverse Shell ด้วย Standard Library

เทคนิคมัลแวร์

รวมเทคนิค offensive tradecraft ที่ถูกปรับมาใช้กับ Zig

  • Process Injection
    • DLL Injection
    • Shellcode Injection
  • Payload Staging
    • จาก Web Server
    • จาก Windows Registry
  • Process Enumeration
    • โดยใช้ CreateToolhelp32Snapshot
    • โดยใช้ EnumProcesses
    • โดยใช้ NtQuerySystemInformation

เครื่องมือ Maldev

  • ZYRA: Your Runtime Armor
    • โปรแกรม packer สำหรับ executable เขียนด้วย Zig
  • ZYPE: Your Payload Encryptor
    • สร้าง code template ใน Zig ที่มี payload ที่เข้ารหัส/ปกปิดไว้พร้อมฟังก์ชันถอดรหัส/ถอดปกปิด
