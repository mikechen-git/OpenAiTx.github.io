🚀 Clash सब्सक्रिप्शन कन्वर्टर

एक सुंदर और शक्तिशाली ऑनलाइन टूल, जिसका उपयोग विभिन्न प्रॉक्सी सब्सक्रिप्शन लिंक को Clash कॉन्फ़िगरेशन फॉर्मेट में बदलने के लिए किया जाता है। Cloudflare Workers & Pages के माध्यम से स्वचालित डिप्लॉयमेंट को सपोर्ट करता है, तेज़ और विश्वसनीय सब्सक्रिप्शन रूपांतरण सेवा प्रदान करता है।

✨ फीचर्स

  • 🎨 आकर्षक आधुनिक इंटरफ़ेस - ग्रेडिएंट बैकग्राउंड और कार्ड-आधारित डिज़ाइन
  • 🔄 मल्टी-प्रोटोकॉल सपोर्ट - V2Ray, VLESS, Shadowsocks, Trojan आदि प्रमुख प्रोटोकॉल को सपोर्ट करता है
  • ☁️ क्लाउड डिप्लॉयमेंट - Cloudflare Workers के माध्यम से वैश्विक CDN त्वरण
  • 📱 रिस्पॉन्सिव डिज़ाइन - डेस्कटॉप और मोबाइल दोनों पर पूरी तरह फिट
  • ⚡ तेज़ रूपांतरण - सब्सक्रिप्शन लिंक को तुरंत Clash कॉन्फ़िगरेशन में बदलता है
  • 📋 स्मार्ट कॉपी - जेनरेटेड YAML कॉन्फ़िगरेशन को एक क्लिक में कॉपी करें, सभी ब्राउज़रों के साथ संगत
  • 📡 सब्सक्रिप्शन लिंक जनरेशन - Clash में सीधे इम्पोर्ट किए जा सकने वाले सब्सक्रिप्शन लिंक को ऑटोमैटिक रूप से जनरेट करें
  • 🌐 ऑनलाइन YAML एक्सेस - कई प्रकार की YAML फाइल एक्सेस विधियाँ प्रदान करता है:
    • 📡 सब्सक्रिप्शन लिंक: Clash क्लाइंट के लिए ऑटो-अपडेट सपोर्ट
    • 🔗 ऑनलाइन व्यू: ब्राउज़र में कॉन्फ़िगरेशन कंटेंट देखें
    • 💾 फाइल डाउनलोड: .yaml फाइल को लोकल में डाउनलोड करें
    • ⚡ तेज़ एक्सेस: सिंप्लिफाइड डाउनलोड लिंक
  • 🔧 UTF-8 सपोर्ट - चीनी नोड नाम और कॉन्फ़िगरेशन को पूरी तरह सपोर्ट करता है
  • 🆓 पूर्णतः निःशुल्क - Cloudflare की फ्री सेवा पर आधारित, कोई उपयोग सीमा नहीं

clash/

🚀 त्वरित प्रारंभ

तरीका 1: लोकल रन करें

  1. प्रोजेक्ट क्लोन करें

    git clone <repository-url>
cd clash

  2. लोकल सर्वर शुरू करें

    # Python का उपयोग करें
python -m http.server 8000

# या Node.js का उपयोग करें
npx serve .

  3. एप्लिकेशन खोलें

    • ब्राउज़र में जाएँ: http://localhost:8000
    • लोकल वातावरण में डेमो कॉन्फ़िगरेशन दिखेगा

तरीका 2: Cloudflare Workers डिप्लॉयमेंट

चरण 1: वर्कर बनाएं

  1. Cloudflare Workers पर जाएं
  2. Cloudflare खाता रजिस्टर/लॉगिन करें
  3. "Create a Worker" पर क्लिक करें
  4. worker.js की कोड को एडिटर में पूरी तरह पेस्ट करें
  5. "Save and Deploy" पर क्लिक करें

चरण 2: Wrangler CLI से डिप्लॉय करें

# Wrangler CLI इंस्टॉल करें
npm install -g wrangler

# Cloudflare में लॉगिन करें
wrangler login

# Worker डिप्लॉय करें
npm run deploy:worker

चरण 3: डोमेन सेटअप (वैकल्पिक)

  1. Worker सेटिंग्स में कस्टम डोमेन जोड़ें
  2. या Cloudflare द्वारा दिए गए डिफ़ॉल्ट डोमेन का उपयोग करें

तरीका 3: Cloudflare Pages डिप्लॉयमेंट

विकल्प A: GitHub इंटीग्रेशन (अनुशंसित)

  1. रिपॉजिटरी तैयार करें

    • कोड को GitHub रिपॉजिटरी में पुश करें
    • सुनिश्चित करें कि सभी आवश्यक फाइलें शामिल हों

  2. Pages प्रोजेक्ट बनाना

    • Cloudflare Pages पर जाएं
    • अपने GitHub रिपॉजिटरी को कनेक्ट करें
    • प्रोजेक्ट रिपॉजिटरी चुनें

  3. बिल्ड सेटिंग्स

    • बिल्ड कमांड: खाली छोड़ें या echo "No build required"
    • बिल्ड आउटपुट डायरेक्टरी: / (रूट डायरेक्टरी)
    • एनवायरनमेंट वेरिएबल्स: सेट करने की आवश्यकता नहीं

विकल्प B: डायरेक्ट अपलोड

# Wrangler Pages का उपयोग करके डिप्लॉय करें
npm run deploy:pages

🛠️ तकनीकी आर्किटेक्चर

फ्रंटएंड टेक्नोलॉजी स्टैक

  • HTML5 - आधुनिक मार्कअप भाषा
  • CSS3 - रेस्पॉन्सिव डिजाइन और एनिमेशन इफेक्ट्स
  • Vanilla JavaScript - हल्का फ्रंटएंड इंटरएक्शन

बैकएंड टेक्नोलॉजी स्टैक

  • Cloudflare Workers - एज कंप्यूटिंग प्लेटफार्म
  • Web APIs - स्टैंडर्ड Fetch API और रिस्पॉन्स हैंडलिंग

समर्थित प्रोटोकॉल फॉर्मेट्स

  • VMess - V2Ray स्टैंडर्ड प्रोटोकॉल
  • VLESS - V2Ray हल्का प्रोटोकॉल (Reality सपोर्टेड)
  • Shadowsocks - क्लासिक प्रॉक्सी प्रोटोकॉल
  • Trojan - नया प्रॉक्सी प्रोटोकॉल

API एंडपॉइंट्स

  • GET / - मुख्य पेज
  • POST /convert - सब्सक्रिप्शन कन्वर्ज़न API
  • GET /clash/{config-id} - YAML कॉन्फ़िगरेशन फ़ाइल प्राप्त करें (सब्सक्रिप्शन लिंक)
  • GET /yaml/{config-id} - YAML कॉन्फ़िगरेशन फ़ाइल डाउनलोड करें
  • OPTIONS /* - CORS प्रीफ्लाइट रिक्वेस्ट

📖 उपयोग गाइड

बेसिक उपयोग

  1. सब्सक्रिप्शन लिंक दर्ज करें

    • अपने प्रॉक्सी सब्सक्रिप्शन लिंक को इनपुट बॉक्स में पेस्ट करें
    • Base64 एन्कोडेड सब्सक्रिप्शन लिंक सपोर्टेड हैं

  2. कॉन्फ़िगरेशन नाम सेट करें

    • आप चाहें तो कॉन्फ़िगरेशन फ़ाइल के लिए कस्टम नाम सेट कर सकते हैं
    • डिफ़ॉल्ट "My Clash Config" है

  3. कॉन्फ़िगरेशन कन्वर्ट करें

    • "सब्सक्रिप्शन कन्वर्ट करें" बटन पर क्लिक करें
    • कन्वर्ज़न पूरा होने तक प्रतीक्षा करें

  4. कॉन्फ़िगरेशन का उपयोग करें

    • जनरेटेड YAML कॉन्फ़िगरेशन को कॉपी करें
    • Clash क्लाइंट में इम्पोर्ट करें

  5. सब्सक्रिप्शन लिंक का उपयोग करें (अनुशंसित)

    • जनरेटेड सब्सक्रिप्शन लिंक को कॉपी करें
    • Clash क्लाइंट में सब्सक्रिप्शन जोड़ें
    • कॉन्फ़िगरेशन अपने आप सिंक हो जाएगा, मैन्युअल अपडेट की आवश्यकता नहीं

एडवांस्ड कॉन्फ़िगरेशन

जनरेटेड Clash कॉन्फ़िगरेशन में निम्नलिखित विशेषताएं शामिल हैं:

  • प्रॉक्सी ग्रुप पॉलिसी

    • 🚀 नोड चयन - मैन्युअल प्रॉक्सी चयन
    • ♻️ ऑटो चयन - लेटेंसी टेस्टिंग के आधार पर स्वचालित चयन
    • 🎯 ग्लोबल डायरेक्ट - डायरेक्ट कनेक्शन

  • DNS कॉन्फ़िगरेशन

    • DNS रिजॉल्यूशन सक्षम
    • fake-ip मोड सपोर्टेड
    • डोमेस्टिक और इंटरनेशनल DNS स्प्लिटिंग

  • रूटिंग नियम

    • लोकल एड्रेस डायरेक्ट कनेक्ट
    • डोमेस्टिक IP डायरेक्ट कनेक्ट
    • अन्य ट्रैफिक प्रॉक्सी से जाए

🔧 डिप्लॉयमेंट ट्रबलशूटिंग

सामान्य समस्याएं

1. Wrangler एंट्री पॉइंट नहीं ढूंढ पा रहा

त्रुटि संदेश: Missing entry-point to Worker script

समाधान:

  • सुनिश्चित करें कि प्रोजेक्ट की रूट डायरेक्टरी में wrangler.jsonc फ़ाइल हो
  • जांचें कि wrangler.jsonc में main फ़ील्ड worker.js की ओर इशारा कर रही है

2. GitHub Actions डिप्लॉयमेंट असफल

समाधान:

  • GitHub रिपॉजिटरी सेटिंग्स में निम्नलिखित Secrets जोड़ें:
    • CLOUDFLARE_API_TOKEN: Cloudflare API टोकन
    • CLOUDFLARE_ACCOUNT_ID: Cloudflare खाता ID

3. Pages निर्माण विफल

समाधान:

  • निर्माण कमांड को खाली या echo "No build required" पर सेट करें
  • सुनिश्चित करें कि निर्माण आउटपुट डायरेक्टरी / पर सेट है

डिप्लॉयमेंट कॉन्फ़िगरेशन फ़ाइल विवरण

  • wrangler.jsonc: Wrangler CLI कॉन्फ़िगरेशन, Worker डिप्लॉयमेंट के लिए
  • _headers: Cloudflare Pages HTTP हेडर कॉन्फ़िगरेशन
  • .github/workflows/deploy.yml: GitHub Actions स्वचालित डिप्लॉयमेंट कॉन्फ़िगरेशन
