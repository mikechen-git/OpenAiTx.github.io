🚀 Clash 订阅转换器

เครื่องมือออนไลน์ที่สวยงามและทรงพลัง สำหรับแปลงลิงก์สมัครสมาชิกพร็อกซีหลากหลายรูปแบบเป็นไฟล์คอนฟิก Clash รองรับการดีพลอยอัตโนมัติผ่าน Cloudflare Workers & Pages ให้บริการแปลงลิงก์สมัครสมาชิกที่รวดเร็วและเชื่อถือได้

✨ ฟีเจอร์เด่น

  • 🎨 อินเทอร์เฟซสวยงามทันสมัย - ใช้พื้นหลังไล่สีและดีไซน์แบบการ์ด
  • 🔄 รองรับหลายโปรโตคอล - รองรับ V2Ray, VLESS, Shadowsocks, Trojan และโปรโตคอลยอดนิยมอื่นๆ
  • ☁️ ดีพลอยบนคลาวด์ - ใช้ Cloudflare Workers เพื่อ CDN acceleration ทั่วโลก
  • 📱 ออกแบบ Responsive - เหมาะกับเดสก์ท็อปและมือถือทุกขนาดหน้าจอ
  • ⚡ แปลงลิงก์ได้รวดเร็ว - แปลงลิงก์สมัครสมาชิกเป็นไฟล์คอนฟิก Clash แบบเรียลไทม์
  • 📋 คัดลอกอัจฉริยะ - รองรับการคัดลอก YAML ที่สร้างขึ้นด้วยคลิกเดียว รองรับทุกเบราว์เซอร์
  • 📡 สร้างลิงก์สมัครสมาชิก - สร้างลิงก์สมัครสมาชิกสำหรับนำเข้า Clash ได้โดยอัตโนมัติ
  • 🌐 เข้าถึง YAML ออนไลน์ - มีหลายวิธีในการเข้าถึงไฟล์ YAML:
    • 📡 ลิงก์สมัครสมาชิก: รองรับอัปเดตอัตโนมัติใน Clash client
    • 🔗 ดูออนไลน์: เปิดดูคอนฟิกในเบราว์เซอร์โดยตรง
    • 💾 ดาวน์โหลดไฟล์: ดาวน์โหลดไฟล์ .yaml ลงเครื่อง
    • ⚡ เข้าถึงรวดเร็ว: ลิงก์ดาวน์โหลดแบบย่อ
  • 🔧 รองรับ UTF-8 - รองรับชื่อโหนดและคอนฟิกภาษาจีนอย่างสมบูรณ์แบบ
  • 🆓 ใช้งานฟรี 100% - ใช้บริการฟรีของ Cloudflare ไม่มีข้อจำกัด

🚀 เริ่มต้นอย่างรวดเร็ว

วิธีที่ 1: รันบนเครื่อง

  1. โคลนโปรเจกต์

    git clone <repository-url>
cd clash

  2. รันเซิร์ฟเวอร์บนเครื่อง

    # ใช้ Python
python -m http.server 8000

# หรือใช้ Node.js
npx serve .

  3. เข้าใช้งานแอป

    • เปิดเบราว์เซอร์ไปที่ http://localhost:8000
    • จะแสดงคอนฟิกตัวอย่างในสภาพแวดล้อมโลคอล

วิธีที่ 2: ดีพลอยผ่าน Cloudflare Workers

ขั้นตอนที่ 1: สร้าง Worker

  1. เข้าไปที่ Cloudflare Workers
  2. สมัคร/เข้าสู่ระบบบัญชี Cloudflare
  3. คลิก "Create a Worker"
  4. คัดลอกโค้ดจาก worker.js ไปวางใน editor
  5. คลิก "Save and Deploy"

ขั้นตอนที่ 2: ดีพลอยด้วย Wrangler CLI

# ติดตั้ง Wrangler CLI
npm install -g wrangler

# ล็อกอินเข้า Cloudflare
wrangler login

# ดีพลอย Worker
npm run deploy:worker

ขั้นตอนที่ 3: ตั้งค่าโดเมน (ไม่บังคับ)

  1. เพิ่มโดเมนที่ต้องการในหน้า Worker settings
  2. หรือใช้โดเมนดีฟอลต์ที่ Cloudflare ให้มา

วิธีที่ 3: ดีพลอยผ่าน Cloudflare Pages

ตัวเลือก A: เชื่อมต่อกับ GitHub (แนะนำ)

  1. เตรียม repository

    • ผลักดันซอร์สโค้ดขึ้น GitHub repository
    • ตรวจสอบให้แน่ใจว่ามีไฟล์ที่จำเป็นครบ

  2. สร้างโปรเจกต์ Pages

    • เข้าไปที่ Cloudflare Pages
    • เชื่อมต่อกับคลัง GitHub ของคุณ
    • เลือกคลังโปรเจกต์

  3. การตั้งค่าการสร้าง

    • คำสั่งสร้าง: เว้นว่างไว้หรือใส่ echo "No build required"
    • โฟลเดอร์ผลลัพธ์การสร้าง: / (ไดเรกทอรีราก)
    • ตัวแปรสภาพแวดล้อม: ไม่ต้องตั้งค่า

ตัวเลือก B: อัปโหลดโดยตรง

# ใช้ Wrangler Pages สำหรับดีพลอย
npm run deploy:pages

🛠️ สถาปัตยกรรมทางเทคนิค

เทคโนโลยีฝั่งหน้า (Frontend)

  • HTML5 - ภาษามาร์กอัปยุคใหม่
  • CSS3 - การออกแบบตอบสนองและเอฟเฟกต์แอนิเมชัน
  • Vanilla JavaScript - การโต้ตอบฝั่งหน้าขนาดเบา

เทคโนโลยีฝั่งหลัง (Backend)

  • Cloudflare Workers - แพลตฟอร์มประมวลผลขอบเครือข่าย (Edge Computing)
  • Web APIs - มาตรฐาน Fetch API และการจัดการ Response

รูปแบบโปรโตคอลที่รองรับ

  • VMess - โปรโตคอลมาตรฐาน V2Ray
  • VLESS - โปรโตคอลน้ำหนักเบา V2Ray (รองรับ Reality)
  • Shadowsocks - โปรโตคอลพร็อกซีคลาสสิก
  • Trojan - โปรโตคอลพร็อกซีสมัยใหม่

จุดเชื่อมต่อ API (API Endpoints)

  • GET / - หน้าแรก
  • POST /convert - API แปลงการสมัครสมาชิก
  • GET /clash/{config-id} - รับไฟล์คอนฟิก YAML (ลิงก์สมัครสมาชิก)
  • GET /yaml/{config-id} - ดาวน์โหลดไฟล์คอนฟิก YAML
  • OPTIONS /* - คำขอพรีไฟลท์ CORS

📖 คู่มือการใช้งาน

การใช้งานพื้นฐาน

  1. ใส่ลิงก์สมัครสมาชิก

    • วางลิงก์สมัครสมาชิกพร็อกซีของคุณลงในช่องอินพุต
    • รองรับลิงก์สมัครสมาชิกที่เข้ารหัส Base64

  2. ตั้งชื่อคอนฟิก

    • เลือกกำหนดชื่อไฟล์คอนฟิกเองได้
    • ค่าดีฟอลต์คือ "My Clash Config"

  3. แปลงคอนฟิก

    • คลิกปุ่ม "แปลงสมัครสมาชิก"
    • รอให้แปลงเสร็จ

  4. ใช้งานคอนฟิก

    • คัดลอกคอนฟิก YAML ที่สร้างขึ้น
    • นำเข้าไปใช้ในแอป Clash

  5. ใช้งานลิงก์สมัครสมาชิก (แนะนำ)

    • คัดลอกลิงก์สมัครสมาชิกที่สร้างขึ้น
    • เพิ่มลิงก์ในแอป Clash
    • คอนฟิกจะซิงค์อัตโนมัติ ไม่ต้องอัปเดตเอง

การตั้งค่าขั้นสูง

คอนฟิก Clash ที่สร้างขึ้นจะมีคุณสมบัติดังนี้:

  • นโยบายกลุ่มพร็อกซี

    • 🚀 เลือกโหนด - เลือกพร็อกซีเอง
    • ♻️ เลือกอัตโนมัติ - เลือกอัตโนมัติโดยทดสอบดีเลย์
    • 🎯 เชื่อมต่อทั่วโลกโดยตรง - เชื่อมต่อโดยตรง

  • การตั้งค่า DNS

    • เปิดใช้งานการแปล DNS
    • รองรับโหมด fake-ip
    • แยก DNS ในประเทศและต่างประเทศ

  • กฎเส้นทาง (Routing Rules)

    • ที่อยู่ท้องถิ่นเชื่อมต่อโดยตรง
    • IP ในประเทศเชื่อมต่อโดยตรง
    • ทราฟฟิกอื่นใช้พร็อกซี

🔧 การแก้ไขปัญหาการดีพลอย

ปัญหาที่พบบ่อย

1. Wrangler หา entry point ไม่เจอ

ข้อความผิดพลาด: Missing entry-point to Worker script

วิธีแก้ไข:

  • ตรวจสอบให้มีไฟล์ wrangler.jsonc อยู่ที่ไดเรกทอรีรากของโปรเจกต์
  • ตรวจสอบว่าใน wrangler.jsonc ฟิลด์ main ชี้ไปที่ worker.js

2. GitHub Actions ดีพลอยไม่สำเร็จ

วิธีแก้ไข:

  • เพิ่ม Secrets ด้านล่างนี้ใน Settings ของคลัง GitHub:
    • CLOUDFLARE_API_TOKEN: โทเคน API ของ Cloudflare
    • CLOUDFLARE_ACCOUNT_ID: Cloudflare Account ID

3. Pages สร้างไม่สำเร็จ

แนวทางแก้ไข:

  • ตั้งค่าคำสั่งสร้างเป็นค่าว่างหรือ echo "No build required"
  • ตรวจสอบให้แน่ใจว่าโฟลเดอร์ผลลัพธ์ของการสร้างถูกตั้งค่าเป็น /

คำอธิบายไฟล์กำหนดค่าการปรับใช้

  • wrangler.jsonc: การกำหนดค่า Wrangler CLI สำหรับการปรับใช้ Worker
  • _headers: การกำหนดค่า HTTP Header สำหรับ Cloudflare Pages
  • .github/workflows/deploy.yml: การกำหนดค่า GitHub Actions สำหรับการปรับใช้อัตโนมัติ

🔧 การกำหนดค่าแบบกำหนดเอง

แก้ไขกลุ่ม Proxy

สามารถแก้ไขฟังก์ชัน convertToClash ใน worker.js เพื่อปรับแต่งกลุ่ม Proxy ได้ดังนี้:

'proxy-groups': [
  {
    name: '🚀 เลือกโหนด',
    type: 'select',
    proxies: ['♻️ เลือกอัตโนมัติ', '🎯 ตรงทั่วโลก', ...proxyNames]
  },
  // เพิ่มกลุ่ม Proxy เพิ่มเติม...
]

กำหนดกฎเส้นทางเอง

แก้ไขอาร์เรย์ rules เพื่อเพิ่มกฎเส้นทางที่กำหนดเอง:

rules: [
  'DOMAIN-SUFFIX,youtube.com,🚀 เลือกโหนด',
  'DOMAIN-SUFFIX,local,DIRECT',
  // เพิ่มกฎเพิ่มเติม...
]

📊 คุณสมบัติด้านประสิทธิภาพ

  • CDN ทั่วโลก - Cloudflare มี Edge Node ครอบคลุมทั่วโลก
  • ตอบสนองรวดเร็ว - เวลาตอบสนองเฉลี่ย < 100ms
  • ความพร้อมใช้งานสูง - ความพร้อมให้บริการ 99.9%
  • โควต้าฟรี - 100,000 คำขอต่อวัน

🔒 หมายเหตุด้านความปลอดภัย

  • ความเป็นส่วนตัวของข้อมูล - ข้อมูลการสมัครสมาชิกจะใช้เฉพาะในกระบวนการแปลงและจะไม่ถูกจัดเก็บ
  • การเข้ารหัส HTTPS - การสื่อสารทั้งหมดถูกเข้ารหัสด้วย HTTPS
  • โอเพ่นซอร์สโปร่งใส - ซอร์สโค้ดทั้งหมดเปิดเผย สามารถตรวจสอบได้ด้วยตนเอง
