🚀 Clash 订阅转换器

Красивый и мощный онлайн-инструмент для преобразования различных прокси-подписок в формат конфигурации Clash. Поддерживает автоматическое развертывание через Cloudflare Workers & Pages, обеспечивая быстрый и надежный сервис преобразования подписок.

✨ Функциональные особенности

  • 🎨 Современный и красивый интерфейс — градиентный фон и карточный дизайн
  • 🔄 Поддержка множества протоколов — поддерживаются V2Ray, VLESS, Shadowsocks, Trojan и другие популярные протоколы
  • ☁️ Облачное развертывание — глобальное CDN-ускорение через Cloudflare Workers
  • 📱 Адаптивный дизайн — идеальная работа на десктопах и мобильных устройствах
  • ⚡ Быстрое преобразование — мгновенное преобразование ссылок-подписок в конфигурацию Clash
  • 📋 Умное копирование — поддержка копирования сгенерированной YAML-конфигурации в один клик, совместимость со всеми браузерами
  • 📡 Генерация подписочных ссылок — автоматическое создание ссылок для прямого импорта в Clash
  • 🌐 Онлайн-доступ к YAML — различные способы доступа к YAML-файлам:
  • 📡 Ссылка на подписку: поддержка автоматического обновления в Clash-клиенте
  • 🔗 Онлайн-просмотр: просмотр содержимого конфигурации прямо в браузере
  • 💾 Загрузка файла: скачивание .yaml-файла на локальный компьютер
  • ⚡ Быстрый доступ: упрощённые ссылки для скачивания
  • 🔧 Поддержка UTF-8 — полная поддержка китайских названий узлов и конфигураций
  • 🆓 Полностью бесплатно — на базе бесплатного сервиса Cloudflare, без ограничений по использованию

🔧 Устранение неполадок при деплое
Часто задаваемые вопросы
#### 1. Wrangler не находит точку входа
Сообщение об ошибке: Missing entry-point to Worker script
Решение: 

  • Убедитесь, что в корне проекта есть файл wrangler.jsonc
    • 

  • Проверьте, что поле main в wrangler.jsonc указывает на worker.js
#### 2. Ошибка деплоя через GitHub Actions
Решение:

  • Добавьте в настройках репозитория GitHub следующие Secrets:
    • 
  
  • CLOUDFLARE_API_TOKEN: Cloudflare API токен
    • 
  
  • CLOUDFLARE_ACCOUNT_ID: Cloudflare ID аккаунта
#### 3. Ошибка сборки Pages
Решение:

  • Установите команду сборки пустой или используйте echo "No build required"
    • 

  • Убедитесь, что выходной каталог сборки установлен как /
Описание файлов конфигурации деплоя
  • wrangler.jsonc: Конфигурация Wrangler CLI для деплоя Worker
    • 

  • _headers: Конфигурация HTTP-заголовков Cloudflare Pages
    • 

  • .github/workflows/deploy.yml: Конфигурация автоматического деплоя через GitHub Actions
🔧 Кастомная настройка
Изменение групп прокси
Можно отредактировать функцию convertToClash в worker.js для кастомизации групп прокси:
javascript 'proxy-groups': [ { name: '🚀 Выбор ноды', type: 'select', proxies: ['♻️ Автовыбор', '🎯 Прямая связь', ...proxyNames] }, // Добавьте больше групп прокси... ] 


Кастомные правила маршрутизации
Измените массив rules, чтобы добавить свои правила маршрутизации:
javascript rules: [ 'DOMAIN-SUFFIX,youtube.com,🚀 Выбор ноды', 'DOMAIN-SUFFIX,local,DIRECT', // Добавьте больше правил... ] `

📊 Особенности производительности

  • Глобальный CDN – Cloudflare с узлами на всех континентах
  • Быстрый отклик – среднее время отклика < 100мс

🔒 Информация по безопасности

  • Конфиденциальность данных – данные подписки используются только во время конвертации и не сохраняются
  • Шифрование HTTPS – все соединения зашифрованы по HTTPS
  • Открытый исходный код – весь исходный код доступен для самостоятельной проверки
