🚀 Clash 订阅转换器

Красивый и мощный онлайн-инструмент для преобразования различных прокси-подписок в формат конфигурации Clash. Поддерживает автоматическое развертывание через Cloudflare Workers & Pages, обеспечивая быстрый и надежный сервис преобразования подписок.

✨ Функциональные особенности

  • 🎨 Современный и красивый интерфейс — градиентный фон и карточный дизайн
  • 🔄 Поддержка множества протоколов — поддерживаются V2Ray, VLESS, Shadowsocks, Trojan и другие популярные протоколы
  • ☁️ Облачное развертывание — глобальное CDN-ускорение через Cloudflare Workers
  • 📱 Адаптивный дизайн — идеальная работа на десктопах и мобильных устройствах
  • ⚡ Быстрое преобразование — мгновенное преобразование ссылок-подписок в конфигурацию Clash
  • 📋 Умное копирование — поддержка копирования сгенерированной YAML-конфигурации в один клик, совместимость со всеми браузерами
  • 📡 Генерация подписочных ссылок — автоматическое создание ссылок для прямого импорта в Clash
  • 🌐 Онлайн-доступ к YAML — различные способы доступа к YAML-файлам:
    • 📡 Ссылка на подписку: поддержка автоматического обновления в Clash-клиенте
    • 🔗 Онлайн-просмотр: просмотр содержимого конфигурации прямо в браузере
    • 💾 Загрузка файла: скачивание .yaml-файла на локальный компьютер
    • ⚡ Быстрый доступ: упрощённые ссылки для скачивания
  • 🔧 Поддержка UTF-8 — полная поддержка китайских названий узлов и конфигураций
  • 🆓 Полностью бесплатно — на базе бесплатного сервиса Cloudflare, без ограничений по использованию

🚀 Быстрый старт

Способ 1: Запуск локально

  1. Клонировать проект

    git clone <repository-url>
cd clash

  2. Запустить локальный сервер

    # Используя Python
python -m http.server 8000

# Или используя Node.js
npx serve .

  3. Открыть приложение

    • Откройте браузер и перейдите по адресу http://localhost:8000
    • В локальной среде будет отображаться пример конфигурации

Способ 2: Развертывание на Cloudflare Workers

Шаг 1: Создать Worker

  1. Перейдите на Cloudflare Workers
  2. Зарегистрируйтесь/войдите в аккаунт Cloudflare
  3. Нажмите "Create a Worker"
  4. Полностью скопируйте код из worker.js в редактор
  5. Нажмите "Save and Deploy"

Шаг 2: Развертывание через Wrangler CLI

# Установить Wrangler CLI
npm install -g wrangler

# Войти в Cloudflare
wrangler login

# Развернуть Worker
npm run deploy:worker

Шаг 3: Настройка домена (опционально)

  1. Добавьте собственный домен в настройках Worker
  2. Или используйте домен по умолчанию, предоставленный Cloudflare

Способ 3: Развертывание на Cloudflare Pages

Вариант A: Интеграция с GitHub (рекомендуется)

  1. Подготовить репозиторий

    • Залейте код в репозиторий GitHub
    • Убедитесь, что включены все необходимые файлы

  2. Создание проекта Pages

    • Перейдите на Cloudflare Pages
    • Подключите ваш репозиторий GitHub
    • Выберите репозиторий проекта

  3. Настройки сборки

    • Команда сборки: оставить пустым или echo "No build required"
    • Директория вывода сборки: / (корневая директория)
    • Переменные среды: не требуется

Вариант B: Прямая загрузка

# Деплой с помощью Wrangler Pages
npm run deploy:pages

🛠️ Техническая архитектура

Фронтенд-стек

  • HTML5 — современный язык разметки
  • CSS3 — адаптивный дизайн и анимации
  • Vanilla JavaScript — легковесное фронтенд-взаимодействие

Бэкенд-стек

  • Cloudflare Workers — платформа edge-вычислений
  • Web APIs — стандартные Fetch API и обработка ответов

Поддерживаемые протоколы

  • VMess — стандартный протокол V2Ray
  • VLESS — облегчённый протокол V2Ray (поддержка Reality)
  • Shadowsocks — классический прокси-протокол
  • Trojan — новый прокси-протокол

API эндпоинты

  • GET / — главная страница
  • POST /convert — API для конвертации подписки
  • GET /clash/{config-id} — получить YAML-конфиг (ссылка на подписку)
  • GET /yaml/{config-id} — скачать YAML-конфиг
  • OPTIONS /* — предварительный CORS-запрос

📖 Руководство пользователя

Базовое использование

  1. Ввод ссылки на подписку

    • Вставьте ссылку на вашу прокси-подписку в поле ввода
    • Поддерживаются подписки в формате Base64

  2. Задание имени конфигурации

    • По желанию задайте собственное имя для файла конфигурации
    • По умолчанию: "My Clash Config"

  3. Конвертация конфигурации

    • Нажмите кнопку "Конвертировать подписку"
    • Дождитесь завершения конвертации

  4. Использование конфигурации

    • Скопируйте сгенерированный YAML-конфиг
    • Импортируйте его в клиент Clash

  5. Использование ссылки на подписку (рекомендуется)

    • Скопируйте сгенерированную ссылку на подписку
    • Добавьте её в клиент Clash
    • Конфиг будет синхронизироваться автоматически, без ручного обновления

Расширенная конфигурация

Сгенерированная конфигурация Clash включает следующие возможности:

  • Стратегии прокси-групп

    • 🚀 Выбор узла — ручной выбор прокси
    • ♻️ Автовыбор — автоматический выбор с тестом задержки
    • 🎯 Прямое глобальное соединение — прямое подключение

  • DNS-конфигурация

    • Включено разрешение DNS
    • Поддержка режима fake-ip
    • Разделение DNS для внутренних и внешних запросов

  • Правила маршрутизации

    • Прямое соединение для локальных адресов
    • Прямое соединение для китайских IP
    • Остальной трафик — через прокси

🔧 Устранение неполадок при деплое

Часто задаваемые вопросы

1. Wrangler не находит точку входа

Сообщение об ошибке: Missing entry-point to Worker script

Решение:

  • Убедитесь, что в корне проекта есть файл wrangler.jsonc
  • Проверьте, что поле main в wrangler.jsonc указывает на worker.js

2. Ошибка деплоя через GitHub Actions

Решение:

  • Добавьте в настройках репозитория GitHub следующие Secrets:
    • CLOUDFLARE_API_TOKEN: Cloudflare API токен
    • CLOUDFLARE_ACCOUNT_ID: Cloudflare ID аккаунта

3. Ошибка сборки Pages

Решение:

  • Установите команду сборки пустой или используйте echo "No build required"
  • Убедитесь, что выходной каталог сборки установлен как /

Описание файлов конфигурации деплоя

  • wrangler.jsonc: Конфигурация Wrangler CLI для деплоя Worker
  • _headers: Конфигурация HTTP-заголовков Cloudflare Pages
  • .github/workflows/deploy.yml: Конфигурация автоматического деплоя через GitHub Actions
