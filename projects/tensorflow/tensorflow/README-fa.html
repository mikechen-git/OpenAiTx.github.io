مستندات | ------------------- | |

TensorFlow یک پلتفرم متن‌باز انتها به انتها برای یادگیری ماشین است. این پلتفرم دارای اکوسیستمی جامع و انعطاف‌پذیر از ابزارها، کتابخانه‌ها و منابع جامعه است که به پژوهشگران امکان می‌دهد آخرین دستاوردهای یادگیری ماشین را ارتقا دهند و توسعه‌دهندگان بتوانند به‌راحتی برنامه‌های مبتنی بر یادگیری ماشین را ایجاد و پیاده‌سازی کنند.

TensorFlow در ابتدا توسط پژوهشگران و مهندسان تیم هوش ماشینی در Google Brain برای انجام پژوهش در یادگیری ماشین و شبکه‌های عصبی توسعه داده شد. با این حال، این چارچوب به اندازه کافی همه‌کاره است که بتوان آن را در حوزه‌های دیگر نیز به کار برد.

TensorFlow رابط‌های برنامه‌نویسی پایدار برای پایتون و ++C ارائه می‌دهد، همچنین رابط‌هایی که سازگاری بازگشتی تضمین‌شده‌ای ندارند برای زبان‌های دیگر نیز در دسترس است.

برای اطلاع از اطلاعیه‌های انتشار و به‌روزرسانی‌های امنیتی، در announce@tensorflow.org عضو شوید. تمام لیست‌های پستی را مشاهده کنید.

نصب

برای مشاهده راهنمای نصب TensorFlow به راهنمای نصب TensorFlow مراجعه کنید. این راهنما شامل نصب با pip، فعال‌سازی پشتیبانی GPU، استفاده از کانتینر Docker و ساخت از سورس است.

برای نصب نسخه فعلی که شامل پشتیبانی از کارت‌های GPU با قابلیت CUDA (در اوبونتو و ویندوز) می‌باشد:

$ pip install tensorflow

سایر دستگاه‌ها (DirectX و MacOS-metal) از طریق افزونه‌های دستگاه پشتیبانی می‌شوند.

یک بسته سبک‌تر فقط برای CPU نیز در دسترس است:

$ pip install tensorflow-cpu

برای به‌روزرسانی TensorFlow به آخرین نسخه، کافی است گزینه --upgrade را به دستورات بالا اضافه کنید.

باینری‌های شبانه برای آزمایش از طریق بسته‌های tf-nightly و tf-nightly-cpu در PyPI در دسترس هستند.

اولین برنامه TensorFlow خود را امتحان کنید

$ python

>>> import tensorflow as tf
>>> tf.add(1, 2).numpy()
3
>>> hello = tf.constant('Hello, TensorFlow!')
>>> hello.numpy()
b'Hello, TensorFlow!'

برای نمونه‌های بیشتر، به آموزش‌های TensorFlow مراجعه کنید.

دستورالعمل‌های مشارکت

اگر می‌خواهید در توسعه TensorFlow مشارکت کنید، حتماً دستورالعمل‌های مشارکت را مرور کنید. این پروژه از منشور رفتار TensorFlow پیروی می‌کند. با مشارکت در این پروژه، انتظار می‌رود که این منشور را رعایت کنید.

ما برای پیگیری درخواست‌ها و اشکالات از مسائل گیت‌هاب استفاده می‌کنیم. برای پرسش‌ها و بحث‌های عمومی به فروم TensorFlow مراجعه کنید و برای پرسش‌های تخصصی به Stack Overflow مراجعه نمایید.

پروژه TensorFlow تلاش می‌کند تا از بهترین شیوه‌های پذیرفته‌شده در توسعه نرم‌افزار متن‌باز پیروی کند.

دستورالعمل‌های اصلاح (Patching)

برای اصلاح یک نسخه خاص از TensorFlow، به عنوان مثال برای رفع باگ‌ها یا آسیب‌پذیری‌های امنیتی، مراحل زیر را دنبال کنید:

  • مخزن TensorFlow را کلون کرده و به شاخه متناظر با نسخه مورد نظر خود بروید، برای مثال شاخه r2.8 برای نسخه 2.8.
  • تغییرات مورد نظر خود را اعمال (cherry-pick) کرده و هر گونه تعارض کد را برطرف کنید.
  • تست‌های TensorFlow را اجرا کرده و اطمینان حاصل کنید که همه آن‌ها با موفقیت انجام می‌شوند.
  • بسته pip TensorFlow را از سورس بسازید.

وضعیت ساخت پیوسته (Continuous Build Status)

می‌توانید پلتفرم‌ها و پیکربندی‌های پشتیبانی‌شده توسط جامعه را در جدول ساخت جامعه SIG Build TensorFlow بیابید.
