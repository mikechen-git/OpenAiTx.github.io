البدء السريع (2 دقيقة)

1. إعداد بوابة الذكاء الاصطناعي الخاصة بك

# تشغيل البوابة محليًا (يتطلب Node.js و npm)
npx @portkey-ai/gateway

البوابة تعمل على http://localhost:8787/v1

وحدة تحكم البوابة تعمل على http://localhost:8787/public/

أدلة النشر:   سحابة Portkey (موصى بها)   دوكر   Node.js   كلاودفلير   ريبليت   أخرى...

2. إرسال أول طلب

# pip install -qU portkey-ai

from portkey_ai import Portkey

# عميل متوافق مع OpenAI
client = Portkey(
    provider="openai", # أو 'anthropic'، 'bedrock'، 'groq'، إلخ
    Authorization="sk-***" # مفتاح API الخاص بالمزود
)

# إرسال طلب عبر بوابة الذكاء الاصطناعي الخاصة بك
client.chat.completions.create(
    messages=[{"role": "user", "content": "كيف هو الطقس اليوم؟"}],
    model="gpt-4o-mini"
)

المكتبات المدعومة:   جافاسكريبت   بايثون   REST   حزم OpenAI   Langchain   LlamaIndex   Autogen   CrewAI   المزيد..

في وحدة تحكم البوابة (http://localhost:8787/public/) يمكنك رؤية جميع سجلاتك المحلية في مكان واحد.

3. التوجيه وقواعد الحماية

تسمح لك الإعدادات في بوابة النماذج اللغوية بإنشاء قواعد توجيه، إضافة الاعتمادية، وإعداد قواعد الحماية.

config = {
  "retry": {"attempts": 5},

  "output_guardrails": [{
    "default.contains": {"operator": "none", "words": ["Apple"]},
    "deny": True
  }]
}

# إرفاق الإعدادات بالعميل
client = client.with_options(config=config)

client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "رد عشوائيًا بكلمة Apple أو Bat"}]
)

# هذا سيقوم دائمًا بالرد بـ "Bat" لأن قاعدة الحماية تمنع جميع الردود التي تحتوي على "Apple". إعداد إعادة المحاولة سيحاول 5 مرات قبل التوقف.

الميزات الأساسية

التوجيه الموثوق

  • الاسترجاع (Fallbacks): الاسترجاع إلى مزود آخر أو نموذج آخر عند فشل الطلبات باستخدام بوابة LLM. يمكنك تحديد الأخطاء التي يتم عندها تفعيل الاسترجاع. يحسن من موثوقية تطبيقك.
  • إعادة المحاولة التلقائية: إعادة محاولة الطلبات الفاشلة تلقائيًا حتى 5 مرات. تعتمد استراتيجية التراجع الأسي للفصل بين محاولات الإعادة لمنع التحميل الزائد على الشبكة.
  • موازنة التحميل: توزيع طلبات LLM عبر عدة مفاتيح API أو مزودي الذكاء الاصطناعي مع تخصيص أوزان لضمان توافر عالي وأداء مثالي.
  • مهلات الطلبات: إدارة حالات LLM غير المتوقعة والتأخيرات من خلال تعيين مهلات دقيقة للطلبات، مما يسمح بإنهاء الطلبات تلقائيًا إذا تجاوزت مدة معينة.
  • بوابة LLM متعددة الوسائط: استدعاء نماذج الرؤية، والصوت (تحويل النص إلى كلام والعكس)، وتوليد الصور من عدة مزودين — جميعها باستخدام توقيع OpenAI المألوف.
  • واجهات برمجة التطبيقات اللحظية (Realtime APIs): استدعاء واجهات برمجة التطبيقات اللحظية التي أطلقتها OpenAI من خلال خادم WebSockets المدمج.

الأمان والدقة

  • الضوابط الوقائية (Guardrails): تحقق من مدخلات ومخرجات LLM لضمان الالتزام بالفحوصات التي تحددها. اختر من بين أكثر من 40 ضابط وقائي جاهز لضمان التوافق مع معايير الأمان والدقة. يمكنك إضافة ضوابطك الخاصة أو الاختيار من شركائنا العديدين.
  • إدارة المفاتيح الآمنة: استخدم مفاتيحك الخاصة أو أنشئ مفاتيح افتراضية بشكل فوري.
  • التحكم بالوصول قائم على الأدوار: تحكم دقيق في وصول المستخدمين ومساحات العمل ومفاتيح API الخاصة بك.
  • الامتثال وخصوصية البيانات: بوابة الذكاء الاصطناعي متوافقة مع SOC2 وHIPAA وGDPR وCCPA.

إدارة التكلفة

  • التخزين المؤقت الذكي: تخزين استجابات LLM مؤقتًا لتقليل التكاليف وتحسين زمن الاستجابة. يدعم التخزين المؤقت البسيط والدلالي*.
  • تحليلات الاستخدام: مراقبة وتحليل استخدام الذكاء الاصطناعي وLLM لديك، بما في ذلك حجم الطلبات، وزمن الاستجابة، والتكاليف، ومعدلات الأخطاء.
  • تحسين المزود*: التبديل تلقائيًا إلى المزود الأكثر فعالية من حيث التكلفة بناءً على أنماط الاستخدام ونماذج التسعير.

التعاون وسير العمل

  • دعم الوكلاء (Agents): تكامل سلس مع أطر عمل الوكلاء الشهيرة لبناء تطبيقات ذكاء اصطناعي معقدة. تتكامل البوابة بسهولة مع Autogen، وCrewAI، وLangChain، وLlamaIndex، وPhidata، وControl Flow، وحتى الوكلاء المخصصين.
  • إدارة قوالب الحوافز*: أنشئ وأدر وقم بإصدار نسخ من قوالب الحوافز الخاصة بك بشكل تعاوني من خلال بيئة عمل موحدة للحوافز.

* متوفرة في النسخ المستضافة ونسخ المؤسسات
