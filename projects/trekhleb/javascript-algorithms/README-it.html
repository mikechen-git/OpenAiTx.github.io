Strutture Dati

Una struttura dati è un modo particolare di organizzare e memorizzare i dati in un computer affinché possano essere accessibili e modificati in modo efficiente. Più precisamente, una struttura dati è una collezione di valori, le relazioni tra di essi e le funzioni o operazioni che possono essere applicate ai dati.

Ricorda che ogni struttura dati ha i suoi compromessi. È più importante capire perché scegli una certa struttura dati che come implementarla.

B - Base, A - Avanzato

  • B Lista Collegata (Linked List)
  • B Lista Doppiamente Collegata (Doubly Linked List)
  • B Coda (Queue)
  • B Pila (Stack)
  • B Tabella Hash (Hash Table)
  • B Heap - versioni max e min heap
  • B Coda a Priorità (Priority Queue)
  • A Trie
  • A Albero (Tree)
    • A Albero Binario di Ricerca (Binary Search Tree)
    • A Albero AVL (AVL Tree)
    • A Albero Rosso-Nero (Red-Black Tree)
    • A Segment Tree - esempi di query di intervallo min/max/somma
    • A Albero di Fenwick (Fenwick Tree) (Binary Indexed Tree)
  • A Grafo (Graph) (sia orientato che non orientato)
  • A Insieme Disgiunto (Disjoint Set) - struttura dati union–find o merge–find set
  • A Filtro di Bloom (Bloom Filter)
  • A Cache LRU (LRU Cache) - cache Least Recently Used (LRU)

Algoritmi

Un algoritmo è una specifica non ambigua su come risolvere una classe di problemi. È un insieme di regole che definiscono con precisione una sequenza di operazioni.

B - Base, A - Avanzato

Algoritmi per Paradigma

Un paradigma algoritmico è un metodo generico o approccio che sottende la progettazione di una classe di algoritmi. È un’astrazione superiore alla nozione di algoritmo, come un algoritmo è un’astrazione superiore rispetto a un programma.

  • Forza Bruta - valuta tutte le possibilità e seleziona la soluzione migliore
    • B Ricerca Lineare (Linear Search)
    • B Rain Terraces - problema del trapping rain water
    • B Scala Ricorsiva (Recursive Staircase) - conteggio dei modi per arrivare in cima
    • A Sottoarray Massimo (Maximum Subarray)
    • A Problema del Commesso Viaggiatore (Travelling Salesman Problem) - percorso più breve che visita tutte le città e ritorna all’origine
    • A Trasformata di Fourier Discreta (Discrete Fourier Transform) - decomposizione di un segnale nel dominio delle frequenze
  • Greedy - sceglie l’opzione migliore al momento, senza considerare il futuro
    • B Jump Game
    • A Knapsack Problem senza limiti (Unbound Knapsack Problem)
    • A Algoritmo di Dijkstra - ricerca del percorso più breve verso tutti i vertici
    • A Algoritmo di Prim (Prim’s Algorithm) - albero ricoprente minimo
    • A Algoritmo di Kruskal (Kruskal’s Algorithm) - albero ricoprente minimo
  • Divide et Impera - divide il problema in parti più piccole e risolve quelle
    • B Ricerca Binaria (Binary Search)
    • B Torre di Hanoi (Tower of Hanoi)
    • B Triangolo di Pascal (Pascal's Triangle)
    • B Algoritmo Euclideo (Euclidean Algorithm) - calcolo del GCD
    • B Merge Sort
    • B Quicksort
    • B Ricerca in Profondità su Alberi (Tree DFS)
    • B Ricerca in Profondità su Grafi (Graph DFS)
    • B Matrici (Matrices) - generazione e attraversamento di matrici di varie forme
    • B Jump Game
    • B Elevamento Rapido a Potenza (Fast Powering)
    • B Best Time To Buy Sell Stocks - esempi divide et impera e one-pass
    • A Permutazioni (Permutations) (con e senza ripetizioni)
    • A Combinazioni (Combinations) (con e senza ripetizioni)
    • A Sottoarray Massimo (Maximum Subarray)
  • Programmazione Dinamica - costruisce una soluzione usando sotto-soluzioni già trovate
    • B Numero di Fibonacci
    • B Jump Game
    • B Percorsi Unici (Unique Paths)
    • B Rain Terraces - problema del trapping rain water
    • B Scala Ricorsiva (Recursive Staircase) - conteggio dei modi per arrivare in cima
    • B Seam Carving - ridimensionamento immagini content-aware
    • A Distanza di Levenshtein (Levenshtein Distance) - distanza minima di modifica
    • A Sottosequenza Comune Massima (Longest Common Subsequence) (LCS)
    • A Sottostringa Comune Massima (Longest Common Substring)
    • A Sottosequenza Crescente Massima (Longest Increasing Subsequence)
    • A Supersequenza Comune più Breve (Shortest Common Supersequence)
    • A Problema Knapsack 0/1
    • A Partizione di Interi (Integer Partition)
    • A Sottoarray Massimo (Maximum Subarray)
    • A Algoritmo di Bellman-Ford - percorsi minimi su grafi
    • A Algoritmo di Floyd-Warshall - percorsi minimi tra tutte le coppie
    • A Matching con Espressioni Regolari (Regular Expression Matching)
  • Backtracking - simile alla forza bruta, cerca di generare tutte le soluzioni possibili, ma ogni volta che se ne genera una, la si testa se soddisfa tutte le condizioni e solo allora si continua, altrimenti si torna indietro (backtrack) e si prova un altro percorso. Solitamente viene usato DFS sullo spazio degli stati.
    • B Jump Game
    • B Percorsi Unici (Unique Paths)
    • B Insieme Potenza (Power Set) - tutti i sottoinsiemi
    • A Ciclo Hamiltoniano (Hamiltonian Cycle) - visita ogni vertice una volta
    • A Problema delle N-Regine (N-Queens Problem)
    • A Tour del Cavallo (Knight's Tour)
    • A Somma di Combinazioni (Combination Sum) - tutte le combinazioni per una somma specifica
  • Branch & Bound - ricorda la soluzione a costo più basso trovata a ogni stadio del backtracking, e usa il costo della soluzione migliore trovata come limite inferiore per scartare le soluzioni parziali peggiori. Solitamente si usa BFS in combinazione con DFS sullo spazio degli stati.

Come usare questo repository

Installa tutte le dipendenze

npm install

Esegui ESLint

Potresti volerlo eseguire per controllare la qualità del codice.

npm run lint

Esegui tutti i test

npm test

Esegui i test per nome

npm test -- 'LinkedList'

Risoluzione dei problemi

Se il linting o i test falliscono, prova a eliminare la cartella node_modules e reinstalla i pacchetti npm:

rm -rf ./node_modules
npm i

Inoltre, assicurati di usare la versione corretta di Node (>=16). Se usi nvm per la gestione delle versioni di Node puoi eseguire nvm use dalla cartella principale del progetto e verrà selezionata la versione corretta.

Playground

Puoi sperimentare con strutture dati e algoritmi nel file ./src/playground/playground.js e scrivere test in ./src/playground/__test__/playground.test.js.

Poi, semplicemente esegui il comando seguente per testare se il tuo codice playground funziona come previsto:

npm test -- 'playground'
