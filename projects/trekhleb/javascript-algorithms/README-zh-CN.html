数据结构

数据结构是一种在计算机中组织和存储数据的特定方式，以便高效地访问和修改。更准确地说，数据结构是一组数据值、它们之间的关系以及可对数据进行的操作和函数的集合。

请记住，每种数据结构都有其权衡。与其关注如何实现，不如更多关注为什么选择某种数据结构。

按范式分类的算法

算法范式是一种指导一类算法设计的通用方法或思想。它比“算法”的抽象层次更高，正如算法比计算机程序更抽象一样。

  • 暴力法 - 穷举所有可能，选择最优解
    • B 线性查找
    • B 接雨水
    • B 递归楼梯
    • A 最大子数组和
    • A 旅行商问题
    • A 离散傅里叶变换
  • 贪心法 - 每步选择当前最优，无视后续影响
    • B 跳跃游戏
    • A 完全背包问题
    • A Dijkstra 算法
    • A Prim 算法
    • A Kruskal 算法
  • 分治法 - 将问题分解为小部分分别解决
    • B 二分查找
    • B 汉诺塔
    • B 杨辉三角
    • B 欧几里得算法
    • B 归并排序
    • B 快速排序
    • B 树的深度优先遍历
    • B 图的深度优先遍历
    • B 矩阵 - 不同形状矩阵的生成与遍历
    • B 跳跃游戏
    • B 快速幂
    • B 买卖股票的最佳时机
    • A 全排列
    • A 组合
    • A 最大子数组和
  • 动态规划 - 利用已求解的子问题构建完整解
    • B 斐波那契数
    • B 跳跃游戏
    • B 不同路径
    • B 接雨水
    • B 递归楼梯
    • B 缝合裁剪
    • A Levenshtein 距离
    • A 最长公共子序列
    • A 最长公共子串
    • A 最长递增子序列
    • A 最短公共超序列
    • A 0/1 背包问题
    • A 整数划分
    • A 最大子数组和
    • A Bellman-Ford 算法
    • A Floyd-Warshall 算法
    • A 正则表达式匹配
  • 回溯法 - 类似暴力法，尝试生成所有解，每生成一个解时测试其合法性，合法则继续，不合法则回退并换路径。通常用 DFS 状态空间遍历。
    • B 跳跃游戏
    • B 不同路径
    • B 幂集
    • A 哈密顿回路
    • A N 皇后问题
    • A 骑士巡游
    • A 组合总和
  • 分支限界法 - 在回溯过程中记录当前已找到的最优解并用其作为下界，剪枝丢弃成本更高的部分解。通常结合 BFS 与 DFS 状态空间树遍历。

如何使用本仓库

安装所有依赖

npm install

运行 ESLint

可用于检查代码质量。

npm run lint

运行所有测试

npm test

按名称运行测试

npm test -- 'LinkedList'

故障排查

若 lint 或测试失败，尝试删除 node_modules 文件夹并重新安装 npm 包：

rm -rf ./node_modules
npm i

此外，确保你使用的 Node 版本为（>=16）。如果用 nvm 管理 Node 版本，可以在项目根目录执行 nvm use 自动切换到合适版本。

Playground

你可在 ./src/playground/playground.js 文件中自由试验数据结构和算法，并在 ./src/playground/__test__/playground.test.js 中为其编写测试。

然后只需运行以下命令测试你的 playground 代码：

npm test -- 'playground'
