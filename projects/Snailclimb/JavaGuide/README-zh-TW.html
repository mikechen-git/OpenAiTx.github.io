災備設計和異地多活

災備 = 容災 + 備份。

  • 備份：將系統所產生的所有重要資料多備份幾份。
  • 容災：在異地建立兩個完全相同的系統。當某個地方的系統突然掛掉，整個應用系統可以切換到另一個，這樣系統就可以正常提供服務了。

異地多活 描述的是將服務部署在異地並且服務同時對外提供服務。和傳統的災備設計最主要的區別在於「多活」，即所有站點都是同時在對外提供服務的。異地多活是為了應對突發狀況比如火災、地震等自然或者人為災害。
