autoResume

Otwarty kreator CV, wklej swoje linki, edytuj ręcznie i pozwól AI pomóc w dopracowaniu, aktualizacji oraz dostosowaniu Twojego CV.

Jak tego używać?

Link do bloga na Medium

Blog jest nieaktualny w stosunku do interfejsu użytkownika i użytego szablonu.

Funkcje

  • Łatwy import linków: Wklejaj linki ze stron internetowych i szybko twórz CV w formacie PDF.
  • Wbudowany edytor ręczny: Dopracuj lub zbuduj CV od podstaw za pomocą wbudowanego edytora kodu.
  • Inteligentne sugestie: Przekazuj opinie lub nowe linki, a funkcja smartedits pomoże ulepszyć i zaktualizować Twoje CV.
  • Dopasowanie do ofert pracy: Wklej opis stanowiska i otrzymaj rekomendacje, jak dostosować swoje CV.
  • Jakość LaTeX: Wszystkie CV są generowane za pomocą LaTeX, co zapewnia czytelny i profesjonalny układ.
  • Podgląd na żywo: Zobacz swoje CV jako PDF w czasie rzeczywistym.
  • Łatwy reset: Wyczyść i rozpocznij tworzenie CV od nowa jednym kliknięciem.
  • Lokalnie: Działa w pełni na Twoim komputerze przy użyciu Dockera.

Pierwsze kroki

Wymagania wstępne

  • Docker
  • Docker Compose

Instalacja

  • Sklonuj repozytorium:
``bash git clone https://github.com/aadya940/autoresume.git cd autoresume `
  • Zbuduj i uruchom aplikację:
`bash docker compose up --build `
  • Uzyskaj dostęp do aplikacji:
_http://localhost:5173_

Użytkowanie

  • Otwórz autoResume w swojej przeglądarce.
  • Kliknij ikonę :gear:, wklej swój KLUCZ API GOOGLE oraz swój e‑mail.
  • Wklej linki do swoich danych zawodowych (np. Github, strona osobista itp.).
  • Opcjonalnie dodaj opinię lub link do oferty pracy, aby dalej dostosować CV.
  • Kliknij Generate Resume i pozwól, by AI zrobiło resztę!

Przewodnik dla współtwórców

Frontend

  • Implementuj komponenty React w folderze frontend/src/components/
  • Używamy Chakra UI do implementacji komponentów.
  • Importuj i używaj swoich komponentów w frontend/src/App.jsx

Backend

  • Implementuj trasy API w katalogu backend/src/routes/
  • Dodawaj funkcjonalności AI w katalogu backend/src/ai/
  • Używamy formatowania kodu black dla Pythona
  • Zachowuj przejrzystość obsługi tras i oddzielaj logikę

Ogólne wytyczne

  • Twórz nową gałąź dla swojej funkcji/poprawki: git checkout -b twoja-nazwa-galezi`
  • Pisz czytelne komunikaty commitów
  • Przetestuj zmiany przed wysłaniem PR
