autoResume

Открытый конструктор резюме: вставьте свои ссылки, редактируйте вручную, а ИИ поможет с доработкой, обновлением и адаптацией вашего резюме.

Как этим пользоваться?

Ссылка на блог на Medium

Блог устарел относительно пользовательского интерфейса и используемого шаблона.

Возможности

  • Простой импорт ссылок: Вставляйте ссылки с сайтов и быстро создавайте резюме в PDF.
  • Встроенный ручной редактор: Тонко настраивайте или создавайте резюме с нуля с помощью встроенного редактора кода.
  • Умные подсказки: Делитесь обратной связью или новыми ссылками, а умные правки помогут доработать и обновить ваше резюме.
  • Адаптация под вакансии: Вставьте описание вакансии и получите рекомендации для оптимизации резюме.
  • Качество LaTeX: Все резюме генерируются с помощью LaTeX для получения чистого и профессионального макета.
  • Мгновенный предпросмотр: Смотрите свое резюме в формате PDF в реальном времени.
  • Легкий сброс: Очистите и начните создавать резюме заново одним кликом.
  • Локально: Работает полностью на вашем компьютере с помощью Docker.

Начало работы

Необходимые компоненты

  • Docker
  • Docker Compose

Установка

  • Клонируйте репозиторий:
``bash git clone https://github.com/aadya940/autoresume.git cd autoresume `
  • Соберите и запустите приложение:
`bash docker compose up --build `
  • Откройте приложения:
_http://localhost:5173_

Использование

  • Откройте autoResume в вашем браузере.
  • Нажмите на иконку :gear:, вставьте ваш GOOGLE API KEY и Email.
  • Вставьте ссылки на вашу профессиональную информацию (например, Github, личный сайт и т.д.).
  • При необходимости добавьте обратную связь или ссылку на вакансию для дополнительной настройки.
  • Нажмите Сгенерировать резюме и позвольте ИИ сделать остальное!

Лицензия

Этот проект распространяется по лицензии Apache 2.0.

Руководство по вкладу

Фронтенд

  • Реализуйте React-компоненты в папке frontend/src/components/
  • Для компонентов используется Chakra UI.
  • Импортируйте и используйте ваши компоненты в frontend/src/App.jsx

Бэкенд

  • Реализуйте API-маршруты в директории backend/src/routes/
  • Добавляйте функционал ИИ в директорию backend/src/ai/
  • Для Python-кода используется форматтер black
  • Держите обработчики маршрутов чистыми, а логику — раздельной

Общие рекомендации

  • Создайте новую ветку для вашей функции/исправления: git checkout -b your-branch-name`
  • Пишите понятные сообщения коммитов
  • Тестируйте изменения перед отправкой PR

Отправка изменений

  • Создайте Pull Request
  • В описании PR укажите:
  • Какие изменения вы внесли
  • Почему вы их внесли
  • Любые соответствующие скриншоты или результаты тестов
  • Любые критические изменения или шаги миграции
