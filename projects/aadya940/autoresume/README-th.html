autoResume

เครื่องมือสร้างเรซูเม่โอเพ่นซอร์ส เพียงวางลิงก์ของคุณ แก้ไขเองได้ และให้ AI ช่วยปรับแต่ง อัปเดต และทำให้เรซูเม่ของคุณเหมาะสมยิ่งขึ้น

ใช้งานอย่างไร?

ลิงก์บทความ Medium

บทความนี้ไม่อัปเดตกับส่วนติดต่อผู้ใช้และเทมเพลตที่ใช้งานในปัจจุบัน

ฟีเจอร์

  • นำเข้าลิงก์ได้ง่าย: วางลิงก์จากเว็บไซต์ แล้วสร้างเรซูเม่ PDF ได้อย่างรวดเร็ว
  • มีตัวแก้ไขแบบแมนนวล: ปรับแต่งหรือสร้างเรซูเม่ของคุณเองตั้งแต่เริ่มต้นด้วยตัวแก้ไขโค้ดในตัว
  • คำแนะนำอัจฉริยะ: แชร์ข้อเสนอแนะหรือลิงก์ใหม่ ๆ และ smartedits จะช่วยปรับแต่งและอัปเดตเรซูเม่ของคุณ
  • ปรับให้เหมาะกับงาน: วางคำอธิบายงาน แล้วรับคำแนะนำเพื่อปรับให้เรซูเม่ตรงกับงานนั้น
  • คุณภาพ LaTeX: เรซูเม่ทั้งหมดสร้างด้วย LaTeX เพื่อความเรียบร้อยและดูเป็นมืออาชีพ
  • ดูตัวอย่างทันที: ดูเรซูเม่ของคุณในรูปแบบ PDF แบบเรียลไทม์
  • รีเซ็ตง่าย: ล้างและเริ่มต้นเรซูเม่ใหม่ได้ในคลิกเดียว
  • ใช้งานภายในเครื่อง: ทำงานบนเครื่องของคุณเองโดยใช้ Docker

เริ่มต้นใช้งาน

ข้อกำหนดเบื้องต้น

  • Docker
  • Docker Compose

การติดตั้ง

  1. โคลน repository:

    git clone https://github.com/aadya940/autoresume.git
cd autoresume

  2. สร้างและเริ่มแอป:

    docker compose up --build

  3. เข้าถึงแอปได้ที่:

    http://localhost:5173

การใช้งาน

  1. เปิด autoResume ในเบราว์เซอร์ของคุณ
  2. คลิกไอคอน :gear: วาง GOOGLE API KEY และอีเมลของคุณ
  3. วางลิงก์ข้อมูลอาชีพของคุณ (เช่น Github, เว็บไซต์ส่วนตัว ฯลฯ)
  4. หากต้องการ สามารถเพิ่มข้อเสนอแนะหรือลิงก์ประกาศงานเพื่อปรับแต่งเพิ่มเติม
  5. คลิก Generate Resume แล้วปล่อยให้ AI จัดการส่วนที่เหลือ!

ใบอนุญาต

โปรเจกต์นี้ใช้สัญญาอนุญาต Apache 2.0

แนวทางการมีส่วนร่วม

ส่วนหน้า (Frontend)

  • สร้างคอมโพเนนต์ React ในโฟลเดอร์ frontend/src/components/
  • เราใช้ Chakra UI ในการสร้างคอมโพเนนต์
  • นำเข้าและใช้คอมโพเนนต์ของคุณใน frontend/src/App.jsx

ส่วนหลัง (Backend)

  • สร้างเส้นทาง API ในไดเรกทอรี backend/src/routes/
  • เพิ่มฟังก์ชัน AI ในไดเรกทอรี backend/src/ai/
  • เราใช้ black สำหรับจัดรูปแบบโค้ด Python
  • ควรแยก handler ของ route และ logic ออกจากกัน

แนวทางทั่วไป

  • สร้าง branch ใหม่สำหรับฟีเจอร์/แก้ไขของคุณ: git checkout -b your-branch-name
  • เขียนข้อความ commit ให้ชัดเจน
  • ทดสอบการเปลี่ยนแปลงก่อนส่ง PR

การส่งการเปลี่ยนแปลง

  1. สร้าง Pull Request
  2. ในคำอธิบาย PR ของคุณ กรุณาระบุ:
    • สิ่งที่คุณเปลี่ยนแปลง
    • เหตุผลที่เปลี่ยน
    • ภาพหน้าจอหรือผลการทดสอบ (ถ้ามี)
    • การเปลี่ยนแปลงที่กระทบหรือขั้นตอน migration (ถ้ามี)

เราขอขอบคุณสำหรับการมีส่วนร่วมของคุณ และจะตรวจสอบ PR ของคุณโดยเร็วที่สุด!
