autoResume

Construtor de currículos open‑source: cole seus links, edite manualmente e deixe a IA ajudar a aperfeiçoar, atualizar e personalizar seu currículo.

Como usar isto?

Link do Blog no Medium

O blog está desatualizado em relação à Interface do Usuário e ao Modelo utilizado.

Funcionalidades

  • Importação Fácil de Links: Cole links de sites e construa rapidamente um currículo em PDF.
  • Editor Manual Incluso: Ajuste ou crie seu currículo do zero com um editor de código integrado.
  • Sugestões Inteligentes: Compartilhe feedback ou novos links, e as edições inteligentes ajudarão a refinar e atualizar seu currículo.
  • Personalizado para Vagas: Cole uma descrição de vaga e receba recomendações para alinhar seu currículo.
  • Qualidade LaTeX: Todos os currículos são gerados usando LaTeX para um layout limpo e profissional.
  • Visualização Instantânea: Veja seu currículo em PDF em tempo real.
  • Reinício Fácil: Limpe e reinicie seu currículo com um clique.
  • Local: Roda totalmente em sua máquina usando Docker.

Primeiros Passos

Pré-requisitos

  • Docker
  • Docker Compose

Configuração

  • Clone o repositório:
``bash git clone https://github.com/aadya940/autoresume.git cd autoresume `
  • Construa e inicie o aplicativo:
`bash docker compose up --build `
  • Acesse os aplicativos:
_http://localhost:5173_

Uso

  • Abra o autoResume no seu navegador.
  • Clique no ícone :gear:, cole sua CHAVE DE API DO GOOGLE e seu Email.
  • Cole links com suas informações profissionais (por exemplo, Github, Site Pessoal, etc.).
  • Opcionalmente, adicione feedback ou o link de uma vaga para personalização adicional.
  • Clique em Gerar Currículo e deixe a IA fazer o resto!

Guia de Contribuição

Frontend

  • Implemente componentes React na pasta frontend/src/components/
  • Usamos Chakra UI para implementar os componentes.
  • Importe e utilize seus componentes em frontend/src/App.jsx

Backend

  • Implemente rotas de API no diretório backend/src/routes/
  • Adicione funcionalidades de IA no diretório backend/src/ai/
  • Utilizamos o formatador de código black para código Python
  • Mantenha os manipuladores de rotas limpos e a lógica separada

Diretrizes Gerais

  • Crie um novo branch para sua funcionalidade/correção: git checkout -b seu-nome-de-branch`
  • Escreva mensagens de commit claras
  • Teste suas alterações antes de enviar um PR

Enviando Alterações

  • Crie um Pull Request
  • Na descrição do seu PR, inclua:
  • Quais alterações você fez
  • Por que as fez
  • Quaisquer capturas de tela ou resultados de teste relevantes
  • Quaisquer mudanças incompatíveis ou etapas de migração necessárias
