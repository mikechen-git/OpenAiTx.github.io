How to Get Started Quickly

First, it is recommended to install mkdocs-material in a virtual environment:

pip install mkdocs-material

Method 1:
Directly download the Wcowin-for-MkDocs.zip file from releases, extract it to your local folder, then run mkdocs serve in the terminal in that directory.

Method 2:
In your local folder (mine is Wcowin.github.io), execute git clone git@github.com:Wcowin/Mkdocs-Wcowin.git in the terminal to clone the template locally.

Copy all contents from the cloned folder into your Wcowin.github.io folder (as shown below), then run mkdocs serve in the Wcowin.github.io directory terminal to start. Another way to start: there is a Mkdocs-serve.bat batch file in the root directory, simply double-click it (Windows only).

!Note: If you are prompted that plugins such as git-committers are not installed, just run pip install git-committers in the terminal. Install any missing plugins as prompted, or just run pip install -r requirements.txt to install all plugins.

For the initial tutorial, see: Deploying Static Websites to GitHub Pages with Mkdocs
