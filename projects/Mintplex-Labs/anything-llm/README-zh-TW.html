👉 AnythingLLM 桌面版（Mac、Windows、Linux）！ 立即下載

一個全端應用程式，讓你可以將任何文件、資源或內容轉化為 LLM 在對話時可用的參考上下文。此應用程式讓你可以選擇及決定要使用的 LLM 或向量資料庫，並支援多用戶管理和權限設定。

觀看示範！

產品總覽

AnythingLLM 是一個全端應用程式，你可以利用現成的商用 LLM 或流行的開源 LLM 及向量資料庫方案，打造一個毫無妥協、可在本地運行亦可遠端託管的私人 ChatGPT，並能與你所提供的任何文件進行智慧對話。

AnythingLLM 將你的文件劃分為稱作 workspaces 的物件。Workspace 的運作方式很像討論串，但多了文件容器化的特性。Workspace 之間可以共享文件，但彼此不會互通，因此每個 workspace 的上下文都能保持乾淨。

技術總覽

這個單一倉庫包含六個主要區塊：

  • frontend：viteJS + React 前端，可用來建立與管理所有 LLM 可用的內容。
  • server：NodeJS express 伺服器，負責所有互動及向量資料庫管理與 LLM 互通。
  • collector：NodeJS express 伺服器，用於從 UI 處理及解析文件。
  • docker：Docker 指令、建置流程與原始碼建置資訊。
  • embed：用於產生與建立網頁嵌入元件的子模組。
  • browser-extension：用於Chrome 瀏覽器擴充套件的子模組。

開發環境設置說明

  • 執行 yarn setup 以填寫每個應用區塊所需的 .env 檔（從倉庫根目錄）。
    • 請先填寫這些檔案再繼續。請確保 server/.env.development 已填寫，否則功能會異常。
  • 執行 yarn dev:server 以在本地啟動伺服器（從倉庫根目錄）。
  • 執行 yarn dev:frontend 以在本地啟動前端（從倉庫根目錄）。
  • 執行 yarn dev:collector 以啟動文件收集器（從倉庫根目錄）。

遙測與隱私

Mintplex Labs Inc 所開發的 AnythingLLM 內含收集匿名使用資訊的遙測功能。

關於 AnythingLLM 的遙測與隱私細節

為什麼？

我們利用這些資訊來了解 AnythingLLM 的使用情形，協助我們釐定新功能與錯誤修正的優先順序，並提升 AnythingLLM 的效能與穩定性。

如何選擇不參與

將伺服器或 Docker 的 .env 設定中的 DISABLE_TELEMETRY 設為 "true" 以選擇不參與遙測。你也可以在應用程式內側邊欄 > 隱私 頁面關閉遙測功能。

我們明確追蹤哪些資料？

我們只會追蹤有助於產品與路線圖決策的使用細節，具體如下：

  • 你的安裝類型（Docker 或 桌面版）

  • 文件被新增或移除的時機。不包含任何關於文件內容的資訊。僅追蹤事件發生，這讓我們能了解使用情況。

  • 使用中的向量資料庫類型。這有助於我們在該供應商有更新時優先調整。

  • 使用中的 LLM 供應商與模型標籤。這有助於我們在該供應商、模型或其組合（例如：推理型與一般型、多模態模型等）有更新時優先調整。

  • 聊天訊息發送的時機。這是最頻繁的「事件」，讓我們了解各安裝實例的日常活躍度。同樣地，僅會傳送事件本身——我們不會獲取任何聊天內容或性質的資訊。

你可以透過搜尋所有呼叫 Telemetry.sendTelemetry 的位置來驗證上述說明。此外，這些事件也會寫入輸出日誌，所以你可以看到實際傳送的資料（若已啟用）。不會收集任何 IP 或其他可識別資訊。遙測供應商為 PostHog —— 一個開源的遙測收集服務。

我們非常重視隱私，希望你理解我們希望了解工具被使用的方式，而不是用煩人的彈窗問卷，這樣我們才能打造值得使用的產品。這些匿名資料 絕不 會與第三方分享。
