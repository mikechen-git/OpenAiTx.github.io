👉 데스크톱용 AnythingLLM(Mac, Windows, & Linux)! 지금 다운로드

모든 문서, 리소스 또는 콘텐츠를 LLM이 채팅 중 참조할 수 있는 컨텍스트로 전환할 수 있는 풀스택 애플리케이션입니다. 이 애플리케이션은 사용자가 원하는 LLM 또는 벡터 데이터베이스를 선택할 수 있으며, 다중 사용자 관리 및 권한도 지원합니다.

데모 영상 보기!

제품 개요

AnythingLLM은 상용 LLM 또는 인기 있는 오픈소스 LLM 및 벡터DB 솔루션을 사용하여 타협 없는 프라이빗 ChatGPT를 구축할 수 있는 풀스택 애플리케이션입니다. 로컬로 실행하거나 원격으로 호스팅할 수 있으며, 제공하는 모든 문서와 지능적으로 대화할 수 있습니다.

AnythingLLM은 문서를 워크스페이스라는 객체로 분할합니다. 워크스페이스는 스레드와 비슷하게 동작하지만, 문서 컨테이너화 기능이 추가되어 있습니다. 워크스페이스는 문서를 공유할 수 있지만 서로 간에는 대화하지 않으므로 각 워크스페이스의 컨텍스트를 깔끔하게 유지할 수 있습니다.

AnythingLLM의 주요 기능

  • 🆕 완전 MCP 호환
  • 🆕 노코드 AI 에이전트 빌더

  • 커스텀 AI 에이전트
  • 👤 다중 사용자 인스턴스 지원 및 권한 관리 Docker 버전 한정
  • 🦾 워크스페이스 내 에이전트(웹 브라우징 등)
  • 💬 웹사이트용 커스텀 임베디드 챗 위젯 Docker 버전 한정
  • 📖 다양한 문서 형식 지원(PDF, TXT, DOCX 등)
  • 드래그 앤 드롭 기능과 명확한 인용이 가능한 간단한 챗 UI
  • 100% 클라우드 배포 준비 완료
  • 모든 인기 클로즈드 및 오픈소스 LLM 제공자와 호환
  • 다른 챗 UI 대비 매우 큰 문서 관리에 특화된 비용 및 시간 절감 기능 내장
  • 커스텀 통합을 위한 전체 개발자 API 제공
  • 그 외 다양한 기능… 직접 설치해서 확인해보세요!

기술 개요

이 모노레포는 여섯 개의 주요 섹션으로 구성되어 있습니다:

  • frontend: LLM이 사용할 모든 콘텐츠를 쉽게 생성 및 관리할 수 있는 viteJS + React 프런트엔드
  • server: 모든 상호작용과 벡터DB 관리 및 LLM 상호작용을 처리하는 NodeJS express 서버
  • collector: UI에서 문서를 처리하고 파싱하는 NodeJS express 서버
  • docker: 도커 빌드 및 소스 빌드를 위한 도커 지침 및 정보
  • embed: 웹 임베드 위젯 생성 및 생성을 위한 서브모듈
  • browser-extension: 크롬 브라우저 확장 프로그램을 위한 서브모듈

개발 환경 설정 방법

  • yarn setup 각 애플리케이션 섹션에 필요한 .env 파일을 채웁니다(레포 루트에서 실행).
    • 진행 전에 해당 파일을 모두 작성하세요. server/.env.development가 반드시 채워져 있어야 정상 동작합니다.
  • yarn dev:server 서버를 로컬에서 부팅합니다(레포 루트에서 실행).
  • yarn dev:frontend 프런트엔드를 로컬에서 부팅합니다(레포 루트에서 실행).
  • yarn dev:collector 문서 수집기를 실행합니다(레포 루트에서 실행).

명시적으로 추적하는 항목은 무엇인가요?

제품 및 로드맵 결정을 내리는 데 도움이 되는 사용 세부정보만을 추적합니다. 구체적으로는 다음과 같습니다:

  • 설치 유형 (Docker 또는 Desktop)

  • 문서가 추가되거나 제거될 때. 문서 자체 에 대한 정보는 없습니다. 단지 해당 이벤트가 발생했다는 사실만 기록합니다. 이를 통해 사용 현황을 파악합니다.

  • 사용 중인 벡터 데이터베이스 유형. 이는 해당 제공업체의 업데이트가 있을 때 변경 우선순위를 정하는 데 도움이 됩니다.

  • 사용 중인 LLM 공급자 및 모델 태그 유형. 이는 해당 공급자, 모델 또는 그 조합(예: 추론형 vs 일반형, 멀티모달 모델 등)에 대해 업데이트가 있을 때 변경 우선순위를 정하는 데 도움이 됩니다.

  • 채팅이 전송될 때. 이것이 가장 자주 발생하는 "이벤트"이며, 모든 설치본에서 이 프로젝트의 일일 활동량을 파악하는 데 도움이 됩니다. 다시 한 번, 이벤트만 전송되며, 채팅의 성격이나 내용에 대한 정보는 전혀 수집하지 않습니다.

이 주장들을 검증하려면 Telemetry.sendTelemetry가 호출되는 모든 위치를 찾아볼 수 있습니다. 또한, 이러한 이벤트들은 출력 로그에도 기록되므로(활성화된 경우) 전송된 구체적인 데이터를 직접 확인할 수 있습니다. IP 또는 기타 식별 정보는 수집되지 않습니다. 텔레메트리 제공업체는 PostHog이며, 오픈소스 텔레메트리 수집 서비스입니다.

우리는 개인정보 보호를 매우 중요하게 생각합니다. 귀하께서도 우리가 성가신 팝업 설문조사 없이 도구의 사용 방식을 학습하고, 더 가치 있는 제품을 만들고자 하는 의도를 이해해 주시길 바랍니다. 익명의 데이터는 절대 제3자와 공유되지 않습니다.
