👉 AnythingLLM para desktop (Mac, Windows e Linux)! Baixe Agora

Uma aplicação full-stack que permite transformar qualquer documento, recurso ou conteúdo em contexto para ser usado como referência por qualquer LLM durante conversas. Este aplicativo permite que você escolha qual LLM ou banco de dados vetorial deseja usar, além de suportar gerenciamento multiusuário e permissões.

Assista ao demo!

Visão Geral do Produto

AnythingLLM é uma aplicação full-stack onde você pode usar LLMs comerciais prontos ou populares LLMs open source e soluções vectorDB para construir um ChatGPT privado sem concessões, que pode ser executado localmente ou hospedado remotamente, permitindo conversar de forma inteligente com quaisquer documentos fornecidos.

AnythingLLM divide seus documentos em objetos chamados workspaces. Um Workspace funciona como uma thread, mas com a adição de conteinerização dos seus documentos. Workspaces podem compartilhar documentos, mas não se comunicam entre si, assim você mantém o contexto limpo para cada workspace.

Visão Técnica

Este monorepo consiste em seis seções principais:

  • frontend: Um frontend viteJS + React que você pode executar para criar e gerenciar facilmente todo o conteúdo que o LLM pode usar.
  • server: Um servidor NodeJS express para lidar com todas as interações e fazer toda a gestão do vectorDB e interações do LLM.
  • collector: Servidor NodeJS express que processa e analisa documentos da interface.
  • docker: Instruções e processo de build do Docker + informações para build a partir do código-fonte.
  • embed: Submódulo para geração e criação do widget web embed.
  • browser-extension: Submódulo para a extensão de navegador chrome.

Como configurar para desenvolvimento

  • yarn setup Para preencher os arquivos .env necessários em cada seção do aplicativo (a partir da raiz do repositório).
    • Preencha-os antes de prosseguir. Certifique-se de que server/.env.development está preenchido, caso contrário as coisas não funcionarão corretamente.
  • yarn dev:server Para iniciar o servidor localmente (a partir da raiz do repositório).
  • yarn dev:frontend Para iniciar o frontend localmente (a partir da raiz do repositório).
  • yarn dev:collector Para então rodar o coletor de documentos (a partir da raiz do repositório).

Telemetria & Privacidade

AnythingLLM da Mintplex Labs Inc contém um recurso de telemetria que coleta informações anônimas de uso.

Mais sobre Telemetria & Privacidade no AnythingLLM

Por quê?

Usamos essas informações para nos ajudar a entender como o AnythingLLM é utilizado, para ajudar a priorizar o trabalho em novos recursos e correções de bugs, e para melhorar o desempenho e a estabilidade do AnythingLLM.

Como desativar

Defina DISABLE_TELEMETRY em suas configurações .env do servidor ou do docker como "true" para desativar a telemetria. Você também pode fazer isso no aplicativo, indo para a barra lateral > Privacidade e desativando a telemetria.

O que rastreamos explicitamente?

Rastrearemos apenas detalhes de uso que nos ajudem a tomar decisões sobre o produto e o roteiro, especificamente:

  • Tipo da sua instalação (Docker ou Desktop)

  • Quando um documento é adicionado ou removido. Nenhuma informação sobre o documento. Apenas que o evento ocorreu. Isso nos dá uma ideia de uso.

  • Tipo de banco de dados vetorial em uso. Isso nos ajuda a priorizar mudanças quando atualizações chegam para esse provedor.

  • Tipo de provedor de LLM e tag do modelo em uso. Isso nos ajuda a priorizar mudanças quando atualizações chegam para aquele provedor ou modelo, ou combinação deles. ex: raciocínio vs regular, modelos multi-modais, etc.

  • Quando um chat é enviado. Este é o "evento" mais regular e nos dá uma ideia da atividade diária deste projeto em todas as instalações. Novamente, apenas o evento é enviado – não temos nenhuma informação sobre a natureza ou conteúdo do próprio chat.

Você pode verificar essas informações encontrando todos os locais onde Telemetry.sendTelemetry é chamado. Além disso, esses eventos são gravados no log de saída para que você também possa ver os dados específicos que foram enviados – se habilitado. Nenhum IP ou outra informação identificadora é coletada. O provedor de Telemetria é o PostHog – um serviço de coleta de telemetria open-source.

Levamos a privacidade muito a sério e esperamos que entenda que queremos aprender como nossa ferramenta é usada, sem utilizar pesquisas pop-up incômodas, para que possamos construir algo realmente útil. Os dados anônimos nunca são compartilhados com terceiros, em hipótese alguma.
