Cách hoạt động

Supabase là sự kết hợp của các công cụ mã nguồn mở. Chúng tôi đang xây dựng các tính năng của Firebase sử dụng các sản phẩm mã nguồn mở tiêu chuẩn doanh nghiệp. Nếu công cụ và cộng đồng đã tồn tại, với giấy phép mở như MIT, Apache 2, hoặc tương đương, chúng tôi sẽ sử dụng và hỗ trợ công cụ đó. Nếu công cụ chưa tồn tại, chúng tôi sẽ tự xây dựng và mở mã nguồn nó. Supabase không phải là một bản sao 1-1 của Firebase. Mục tiêu của chúng tôi là mang đến cho lập trình viên trải nghiệm phát triển giống Firebase với các công cụ mã nguồn mở.

Kiến trúc

Supabase là một nền tảng được lưu trữ. Bạn có thể đăng ký và bắt đầu sử dụng Supabase mà không cần cài đặt gì cả. Bạn cũng có thể tự lưu trữ và phát triển cục bộ.

  • Postgres là hệ quản trị cơ sở dữ liệu quan hệ đối tượng với hơn 30 năm phát triển liên tục, tạo nên uy tín vững chắc về độ tin cậy, tính năng mạnh mẽ và hiệu suất.
  • Realtime là máy chủ Elixir cho phép bạn lắng nghe các thao tác chèn, cập nhật và xóa trong PostgreSQL qua websockets. Realtime truy vấn tính năng sao chép tích hợp sẵn của Postgres để phát hiện thay đổi dữ liệu, chuyển đổi thay đổi thành JSON, rồi phát sóng qua websockets tới các client được ủy quyền.
  • PostgREST là máy chủ web biến cơ sở dữ liệu PostgreSQL của bạn thành một RESTful API trực tiếp.
  • GoTrue là API xác thực dựa trên JWT giúp đơn giản hóa việc đăng ký, đăng nhập và quản lý phiên người dùng trong ứng dụng của bạn.
  • Storage là API RESTful để quản lý tập tin trong S3, với Postgres quản lý quyền truy cập.
  • pg_graphql là extension PostgreSQL cung cấp API GraphQL.
  • postgres-meta là API RESTful để quản lý Postgres của bạn, cho phép truy vấn bảng, thêm role, chạy truy vấn, v.v.
  • Kong là API gateway cloud-native.
