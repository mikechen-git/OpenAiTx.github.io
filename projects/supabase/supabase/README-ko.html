작동 방식

Supabase는 오픈 소스 도구들의 조합입니다. 저희는 엔터프라이즈급 오픈 소스 제품을 사용하여 Firebase의 기능을 구현하고 있습니다. MIT, Apache 2 또는 이에 준하는 오픈 라이선스를 가진 도구와 커뮤니티가 존재하면 해당 도구를 사용 및 지원합니다. 만약 도구가 없다면, 직접 만들고 오픈 소스화합니다. Supabase는 Firebase와 1:1로 매핑되는 서비스가 아닙니다. 저희의 목표는 오픈 소스 도구를 활용해 개발자에게 Firebase와 유사한 개발 경험을 제공하는 것입니다.

아키텍처

Supabase는 호스팅 플랫폼입니다. 별도의 설치 없이 회원가입 후 바로 사용하실 수 있습니다. 또한 셀프 호스팅 및 로컬 개발도 가능합니다.

  • Postgres는 30년 이상 활발히 개발되어 온 객체-관계형 데이터베이스 시스템으로, 신뢰성, 기능의 견고함, 성능 면에서 높은 평가를 받고 있습니다.
  • Realtime은 Elixir 기반 서버로, PostgreSQL의 insert, update, delete 이벤트를 웹소켓을 통해 실시간으로 수신할 수 있습니다. Realtime은 Postgres의 내장 복제 기능을 활용해 데이터베이스 변경을 감지하고, 이를 JSON으로 변환하여 인증된 클라이언트에 웹소켓으로 방송합니다.
  • PostgREST는 PostgreSQL 데이터베이스를 RESTful API로 직접 변환해주는 웹 서버입니다.
  • GoTrue는 JWT 기반 인증 API로, 애플리케이션 내 사용자 가입, 로그인, 세션 관리를 간소화합니다.
  • Storage는 S3의 파일을 관리할 수 있는 RESTful API로, 권한 관리는 Postgres가 담당합니다.
  • pg_graphql은 GraphQL API를 제공하는 PostgreSQL 확장입니다.
  • postgres-meta는 Postgres 관리를 위한 RESTful API로, 테이블 조회, 역할 추가, 쿼리 실행 등이 가능합니다.
  • Kong은 클라우드 네이티브 API 게이트웨이입니다.

클라이언트 라이브러리

클라이언트 라이브러리에 대한 저희의 접근 방식은 모듈화입니다. 각 하위 라이브러리는 단일 외부 시스템을 위한 독립적 구현체입니다. 이를 통해 기존 도구를 지원합니다.
