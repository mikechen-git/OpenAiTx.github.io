Как это работает

Supabase — это комбинация инструментов с открытым исходным кодом. Мы реализуем функции Firebase, используя корпоративные продукты с открытым исходным кодом. Если инструменты и сообщества уже существуют и имеют открытую лицензию MIT, Apache 2 или аналогичную, мы будем использовать и поддерживать этот инструмент. Если инструмента нет, мы создаём и открываем его сами. Supabase не является полной копией Firebase. Наша цель — дать разработчикам опыт, похожий на работу с Firebase, используя открытые инструменты.

Архитектура

Supabase — это облачная платформа. Вы можете зарегистрироваться и начать использовать Supabase без установки чего-либо. Вы также можете развернуть самостоятельно и разрабатывать локально.

  • Postgres — это объектно-реляционная система управления базами данных с более чем 30-летней историей активной разработки, заслужившая репутацию надёжности, функциональности и высокой производительности.
  • Realtime — сервер на Elixir, который позволяет слушать вставки, обновления и удаления в PostgreSQL через WebSocket. Realtime опрашивает встроенную функцию репликации Postgres на предмет изменений в базе данных, преобразует изменения в JSON и транслирует JSON по WebSocket авторизованным клиентам.
  • PostgREST — веб-сервер, который превращает вашу базу данных PostgreSQL в RESTful API.
  • GoTrue — API аутентификации на основе JWT, упрощающий регистрацию пользователей, вход и управление сессиями в ваших приложениях.
  • Storage — RESTful API для управления файлами в S3, с контролем доступа через Postgres.
  • pg_graphql — расширение PostgreSQL, предоставляющее GraphQL API.
  • postgres-meta — RESTful API для управления вашей базой Postgres: получение таблиц, добавление ролей, выполнение запросов и др.
  • Kong — облачный API-шлюз.
