工作原理

Supabase 是多个开源工具的组合。我们正在使用企业级、开源产品构建 Firebase 的功能。如果这些工具和社区存在，并且采用 MIT、Apache 2 或等效的开放许可证，我们将使用并支持该工具。如果工具不存在，我们会自行构建并开源。Supabase 并不是 Firebase 的一对一映射。我们的目标是让开发者通过开源工具获得类似 Firebase 的开发体验。

架构

Supabase 是一个托管平台。你可以注册并开始使用 Supabase，无需安装任何东西。 你也可以自托管和本地开发。

  • Postgres 是一个拥有 30 多年活跃开发历史的对象关系数据库系统，以其可靠性、功能强大和高性能而著称。
  • Realtime 是一个 Elixir 服务器，允许你通过 websockets 监听 PostgreSQL 的插入、更新和删除操作。Realtime 通过轮询 Postgres 内置的复制功能获取数据库变更，将变更转换为 JSON，然后通过 websockets 广播给已授权的客户端。
  • PostgREST 是一个将你的 PostgreSQL 数据库直接变为 RESTful API 的 web 服务器。
  • GoTrue 是一个基于 JWT 的认证 API，简化了应用中的用户注册、登录和会话管理。
  • Storage 一个用于在 S3 管理文件的 RESTful API，由 Postgres 管理权限。
  • pg_graphql 一个为 PostgreSQL 提供 GraphQL API 的扩展。
  • postgres-meta 是一个用于管理你的 Postgres 的 RESTful API，允许你获取表、添加角色、执行查询等。
  • Kong 是一个云原生 API 网关。

客户端库

我们对客户端库的设计是模块化的。每个子库都是针对单一外部系统的独立实现。这是我们支持现有工具的方式之一。
