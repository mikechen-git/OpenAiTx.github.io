Come funziona

Supabase è una combinazione di strumenti open source. Stiamo costruendo le funzionalità di Firebase utilizzando prodotti open source di livello enterprise. Se esistono strumenti e comunità con licenza open come MIT, Apache 2 o equivalenti, li useremo e supporteremo. Se lo strumento non esiste, lo costruiamo e lo rendiamo open source noi stessi. Supabase non è una corrispondenza uno-a-uno con Firebase. Il nostro obiettivo è offrire agli sviluppatori un'esperienza simile a Firebase utilizzando strumenti open source.

Architettura

Supabase è una piattaforma ospitata. Puoi registrarti e iniziare a usare Supabase senza installare nulla. Puoi anche ospitare autonomamente e sviluppare in locale.

  • Postgres è un sistema di database relazionale a oggetti con oltre 30 anni di sviluppo attivo che gli ha conferito una solida reputazione per affidabilità, robustezza delle funzionalità e prestazioni.
  • Realtime è un server Elixir che permette di ascoltare inserimenti, aggiornamenti ed eliminazioni su PostgreSQL tramite websockets. Realtime interroga la funzionalità di replica integrata di Postgres per i cambiamenti del database, converte i cambiamenti in JSON e poi li trasmette tramite websocket ai client autorizzati.
  • PostgREST è un web server che trasforma il tuo database PostgreSQL direttamente in una API RESTful.
  • GoTrue è una API di autenticazione basata su JWT che semplifica la registrazione degli utenti, il login e la gestione delle sessioni nelle tue applicazioni.
  • Storage una API RESTful per la gestione dei file in S3, con Postgres che gestisce i permessi.
  • pg_graphql un'estensione PostgreSQL che espone una API GraphQL.
  • postgres-meta è una API RESTful per la gestione del tuo Postgres, che permette di ottenere tabelle, aggiungere ruoli, eseguire query, ecc.
  • Kong è un API gateway cloud-native.
