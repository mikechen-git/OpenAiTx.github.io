Wrappers

Wrappers는 Rust로 작성된 Postgres Foreign Data Wrappers(FDW) 개발 프레임워크입니다. 이 프레임워크의 목표는 Postgres FDW 개발을 더 쉽게 하면서도 Rust 언어의 현대적인 기능(고성능, 강력한 타입, 안전성 등)을 유지하는 것입니다.

Wrappers는 또한 Supabase에서 개발한 FDW 모음입니다. 현재 아래 FDW들을 지원하며, 더 많은 FDW가 개발 중입니다:

| FDW | 설명 | 읽기 | 수정 | | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ | ---- | ----- | | HelloWorld | 기본 FDW 개발 방법을 보여주는 데모 FDW | | | | BigQuery | Google BigQuery을 위한 FDW | ✅ | ✅ | | Clickhouse | ClickHouse를 위한 FDW | ✅ | ✅ | | Stripe | Stripe API를 위한 FDW | ✅ | ✅ | | Firebase | Google Firebase를 위한 FDW | ✅ | ❌ | | Airtable | Airtable API를 위한 FDW | ✅ | ❌ | | S3 | AWS S3를 위한 FDW | ✅ | ❌ | | Logflare | Logflare를 위한 FDW | ✅ | ❌ | | Auth0 | Auth0를 위한 FDW | ✅ | ❌ | | SQL Server | Microsoft SQL Server를 위한 FDW | ✅ | ❌ | | Redis | Redis를 위한 FDW | ✅ | ❌ | | AWS Cognito | AWS Cognito를 위한 FDW | ✅ | ❌ | | Notion | Notion을 위한 Wasm FDW | ✅ | ❌ | | Snowflake | Snowflake을 위한 Wasm FDW | ✅ | ✅ | | Paddle | Paddle을 위한 Wasm FDW | ✅ | ✅ | | Calendly | Calendly를 위한 Wasm FDW | ✅ | ❌ | | Cal.com | Cal.com을 위한 Wasm FDW | ✅ | ❌ | | Clerk | Clerk을 위한 Wasm FDW | ✅ | ❌ | | Cloudflare D1 | Cloudflare D1을 위한 Wasm FDW | ✅ | ✅ | | Orb | Orb를 위한 Wasm FDW | ✅ | ❌ | | HubSpot | HubSpot을 위한 Wasm FDW | ✅ | ❌ | | Slack | Slack을 위한 Wasm FDW | ✅ | ❌ | | Apache Iceberg | Apache Iceberg를 위한 FDW | ✅ | ❌ |

경고

Foreign table이 포함된 물리화 뷰가 있는 데이터베이스의 논리 백업을 복원할 때 실패할 수 있습니다. 이로 인해, 물리화 뷰에서 foreign table을 사용하지 않거나, 물리 백업이 활성화된 데이터베이스에서만 사용하는 것을 권장합니다.

특징

  • 최소한의 인터페이스와 손쉬운 구현.
  • 풍부한 데이터 타입 지원.
  • RDBMS, RESTful API, 플랫 파일 등 동기 및 비동기 백엔드 모두 지원.
  • pgrx 기반으로, 저수준 C API를 숨기지 않으면서 더 높은 수준의 인터페이스 제공.
  • WHERE, ORDER BY, LIMIT 푸시다운 지원.

설치

Wrappers는 pgrx 확장입니다. pgrx 설치 방법에 따라 Wrappers를 설치할 수 있습니다.

기본적으로, 아래 명령어로 FDW를 설치할 수 있습니다 (pgrx 설치 후). 예시:

cargo pgrx install --pg-config [path_to_pg_config] --features stripe_fdw
