설치

macOS 또는 Linux에서는 Homebrew를 이용해 The Fuck을 설치할 수 있습니다:

brew install thefuck

Ubuntu / Mint에서는 다음 명령어로 The Fuck을 설치하세요:

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

FreeBSD에서는 다음 명령어로 The Fuck을 설치하세요:

pkg install thefuck

ChromeOS에서는 chromebrew를 사용하여 설치할 수 있습니다:

crew install thefuck

Arch 기반 시스템에서는 다음 명령어로 설치하세요:

sudo pacman -S thefuck

기타 시스템에서는 pip을 이용해 The Fuck을 설치하세요:

pip install thefuck

또는 OS 패키지 매니저(OS X, Ubuntu, Arch)를 사용할 수도 있습니다.

# 아래 명령어를 .bash_profile, .bashrc, .zshrc 또는 기타 셸 시작 스크립트에 추가하는 것이 권장됩니다:

eval $(thefuck --alias)
# 원하는 별칭을 사용할 수 있습니다. 예: 월요일에는 이렇게:

  • Bash, Zsh, Fish, Powershell, tcsh 등 셸 설정 파일에서 thefuck 별칭 줄을 삭제하거나 주석 처리

  • apt_invalid_operation – 잘못된 apt 및 apt-get 호출 수정(apt-get isntall vim 등);
  • apt_list_upgradable – apt update 이후 apt list --upgradable 실행 안내;
  • apt_upgrade – apt list --upgradable 이후 apt upgrade 실행 안내;
  • brew_cask_dependency – cask 의존성 설치;
  • brew_install – 잘못된 formula명 수정;
  • brew_reinstall – brew install <formula>를 brew reinstall <formula>로 변경;
  • brew_link – 링크 실패 시 --overwrite --dry-run 추가;
  • brew_uninstall – 여러 버전 설치 시 brew uninstall에 --force 추가;
  • brew_unknown_command – 잘못된 brew 명령어 수정(brew docto/brew doctor 등);
  • brew_update_formula – brew update <formula>를 brew upgrade <formula>로 변경;
  • dnf_no_such_command – DNF 명령어 오타 수정;
  • nixos_cmd_not_found – NixOS에서 앱 설치;
  • pacman – 앱이 없을 때 pacman으로 설치(가능하면 yay, pikaur, yaourt 사용);
  • pacman_invalid_option – 소문자 pacman 옵션을 대문자로 변경;
  • pacman_not_found – 패키지명 오타 수정(pacman, yay, pikaur, yaourt 모두 지원);
  • yum_invalid_operation – 잘못된 yum 호출 수정(yum isntall vim 등);

아래 명령어들은 The Fuck에 포함되어 있으나 기본적으로 활성화되어 있지 않습니다:

  • git_push_force – git push에 --force-with-lease 추가(단, git_push_pull과 충돌할 수 있음);
  • rm_root – rm -rf / 명령어에 --no-preserve-root 추가.

사용자 정의 규칙 만들기

자신만의 규칙을 추가하려면, ~/.config/thefuck/rules 디렉토리에 your-rule-name.py 파일을 생성하세요. 규칙 파일에는 두 개의 함수가 포함되어야 합니다:

match(command: Command) -> bool
get_new_command(command: Command) -> str | list[str]

추가적으로, 규칙에는 선택적인 함수도 포함할 수 있습니다:

side_effect(old_command: Command, fixed_command: str) -> None

규칙은 enabled_by_default, requires_output, priority와 같은 선택적 변수도 가질 수 있습니다.

Command는 script, output, script_parts라는 세 가지 속성을 가집니다. 규칙에서 Command 객체 자체를 변경해서는 안 됩니다.

3.0 버전부터 규칙 API가 변경됨: 규칙의 설정에 접근하려면 from thefuck.conf import settings로 임포트하세요.

settings는 ~/.config/thefuck/settings.py와 환경 변수에서 조합된 특수 객체입니다(아래 참고).

다음은 sudo로 스크립트를 실행하는 간단한 예시입니다:

def match(command):
    return ('permission denied' in command.output.lower()
            or 'EACCES' in command.output)


def get_new_command(command):
    return 'sudo {}'.format(command.script)

# 선택 사항:
enabled_by_default = True

def side_effect(command, fixed_command):
    subprocess.call('chmod 777 .', shell=True)

priority = 1000  # 낮을수록 우선 적용, 기본값 1000

requires_output = True

규칙 예시 더 보기, 규칙용 유틸 함수 모음, 앱/OS별 헬퍼.

설정

여러 The Fuck 파라미터는 $XDG_CONFIG_HOME/thefuck/settings.py ($XDG_CONFIG_HOME의 기본값은 ~/.config) 파일에서 변경할 수 있습니다:

  • rules – 활성화할 규칙 리스트, 기본값은 thefuck.const.DEFAULT_RULES;
  • exclude_rules – 비활성화할 규칙 리스트, 기본값 [];
  • require_confirmation – 새로운 명령어 실행 전 확인 여부, 기본값 True;
  • wait_command – 이전 명령어 출력을 받기 위한 최대 대기 시간(초);
  • no_colors – 컬러 출력 비활성화;
  • priority – 규칙 우선순위 딕셔너리, 낮은 값이 먼저 적용;
  • debug – 디버그 출력 활성화, 기본값 False;
  • history_limit – 히스토리에서 검사할 명령어 개수(예: 2000);
  • alter_history – 수정된 명령어를 히스토리에 추가, 기본값 True;
  • wait_slow_command – slow_commands 목록 내 명령어에 대한 최대 대기 시간(초);
  • slow_commands – 느린 명령어 리스트;
  • num_close_matches – 추천할 유사 명령어 최대 개수, 기본값 3;
  • excluded_search_path_prefixes – 명령어 탐색 시 무시할 경로 접두사, 기본값 [].

settings.py 예시:

rules = ['sudo', 'no_command']
exclude_rules = ['git_push']
require_confirmation = True
wait_command = 10
no_colors = False
priority = {'sudo': 100, 'no_command': 9999}
debug = False
history_limit = 9999
wait_slow_command = 20
slow_commands = ['react-native', 'gradle']
num_close_matches = 5

환경 변수로도 설정할 수 있습니다:

  • THEFUCK_RULES – 활성화할 규칙 리스트, 예: DEFAULT_RULES:rm_root 또는 sudo:no_command;
  • THEFUCK_EXCLUDE_RULES – 비활성화할 규칙 리스트, 예: git_pull:git_push;
  • THEFUCK_REQUIRE_CONFIRMATION – 새 명령어 실행 전 확인, true/false;
  • THEFUCK_WAIT_COMMAND – 이전 명령어 출력 대기 시간(초);
  • THEFUCK_NO_COLORS – 컬러 출력 비활성화, true/false;
  • THEFUCK_PRIORITY – 규칙 우선순위, 예: no_command=9999:apt_get=100, 낮은 값이 먼저 적용;
  • THEFUCK_DEBUG – 디버그 출력, true/false;
  • THEFUCK_HISTORY_LIMIT – 히스토리 검사 개수, 예: 2000;
  • THEFUCK_ALTER_HISTORY – 수정 명령어 히스토리 추가 여부, true/false;
  • THEFUCK_WAIT_SLOW_COMMAND – 느린 명령어 출력 대기 시간(초);
  • THEFUCK_SLOW_COMMANDS – 느린 명령어 리스트, 예: lein:gradle;
  • THEFUCK_NUM_CLOSE_MATCHES – 유사 명령어 추천 개수, 예: 5;
  • THEFUCK_EXCLUDED_SEARCH_PATH_PREFIXES – 명령어 탐색 시 무시할 경로 접두사, 기본값 [].

예시:

export THEFUCK_RULES='sudo:no_command'
export THEFUCK_EXCLUDE_RULES='git_pull:git_push'
export THEFUCK_REQUIRE_CONFIRMATION='true'
export THEFUCK_WAIT_COMMAND=10
export THEFUCK_NO_COLORS='false'
export THEFUCK_PRIORITY='no_command=9999:apt_get=100'
export THEFUCK_HISTORY_LIMIT='2000'
export THEFUCK_NUM_CLOSE_MATCHES='5'

서드파티 패키지 규칙

비공개 규칙 세트를 만들고 싶지만 다른 사람들과 공유하고 싶다면, 아래와 같은 구조로 thefuck_contrib_*라는 패키지를 만드세요:

thefuck_contrib_foo
  thefuck_contrib_foo
    rules
      __init__.py
      *third-party rules*
    __init__.py
    *third-party-utils*
  setup.py

The Fuck은 rules 모듈 내의 규칙을 자동으로 찾습니다.

실험적 인스턴트 모드

기본적으로 The Fuck은 이전 명령어를 다시 실행하므로 시간이 소요됩니다. 인스턴트 모드에서는 script로 출력을 기록한 뒤 로그를 읽어 실행 시간을 단축합니다.

현재 인스턴트 모드는 Python 3와 bash 또는 zsh만 지원합니다. zsh의 자동 교정 기능도 비활성화해야 정상 동작합니다.

인스턴트 모드를 활성화하려면, .bashrc, .bash_profile 또는 .zshrc에서 별칭 초기화에 --enable-experimental-instant-mode를 추가하세요.

예시:

eval $(thefuck --alias --enable-experimental-instant-mode)
