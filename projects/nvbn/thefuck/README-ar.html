العودة إلى الفهرس

التثبيت

على macOS أو Linux، يمكنك تثبيت The Fuck عبر Homebrew:

brew install thefuck

على Ubuntu / Mint، ثبّت The Fuck بالأوامر التالية:

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

على FreeBSD، ثبّت The Fuck بالأوامر التالية:

pkg install thefuck

على ChromeOS، ثبّت The Fuck باستخدام chromebrew بالأمر التالي:

crew install thefuck

على الأنظمة المبنية على Arch، ثبّت The Fuck بالأمر التالي:

sudo pacman -S thefuck

على الأنظمة الأخرى، ثبّت The Fuck باستخدام pip:

pip install thefuck

بدلاً من ذلك، يمكنك استخدام مدير الحزم الخاص بنظام التشغيل (OS X, Ubuntu, Arch).

# يُنصح بإضافة هذا الأمر في ملف .bash_profile أو .bashrc أو .zshrc أو أي ملف تهيئة آخر لصدفتك:

eval $(thefuck --alias)
# يمكنك استخدام أي اسم مستعار تريده، مثلاً ليوم الاثنين:
eval $(thefuck --alias FUCK)

أو في ملف تهيئة الصدفة (Bash, Zsh, Fish, Powershell, tcsh).

التغييرات تتوفر فقط في جلسة صدفة جديدة. لجعل التغييرات فورية، شغّل source ~/.bashrc (أو ملف إعداد صدفتك مثل .zshrc).

لتشغيل الأوامر المصححة بدون تأكيد، استخدم الخيار --yeah (أو فقط -y للاختصار، أو --hard إذا كنت منزعجاً جداً):

fuck --yeah

لتصحيح الأوامر بشكل متكرر حتى تنجح، استخدم الخيار -r:

fuck -r

إزالة التثبيت

لإزالة The Fuck، اعكس عملية التثبيت:

  • احذف أو علّق سطر الاسم المستعار thefuck من ملف إعداد صدفتك (Bash, Zsh, Fish, Powershell, tcsh, ...)
  • استخدم مدير الحزم الخاص بك (brew, pip3, pkg, crew, pip) لإزالة الملفات التنفيذية

  • pacman – يثبت التطبيقات مع pacman إذا لم تكن مثبتة (يستخدم yay أو pikaur أو yaourt إذا كانت متوفرة)؛
  • pacman_invalid_option – يستبدل خيارات pacman الصغيرة بأحرف كبيرة.
  • pacman_not_found – يصحح اسم الحزمة مع pacman أو yay أو pikaur أو yaourt.
  • yum_invalid_operation – يصحح أوامر yum غير الصالحة مثل yum isntall vim;

الأوامر التالية مضمنة مع The Fuck ولكنها غير مفعلة افتراضياً:

  • git_push_force – يضيف --force-with-lease إلى git push (قد يتعارض مع git_push_pull);
  • rm_root – يضيف --no-preserve-root إلى أمر rm -rf /.

إنشاء قواعدك الخاصة

لإضافة قاعدة خاصة بك، أنشئ ملفاً باسم your-rule-name.py في ~/.config/thefuck/rules. يجب أن يحتوي ملف القاعدة على دالتين:

match(command: Command) -> bool
get_new_command(command: Command) -> str | list[str]

بالإضافة إلى ذلك، يمكن أن تحتوي القواعد على دوال اختيارية:

side_effect(old_command: Command, fixed_command: str) -> None

يمكن أن تحتوي القواعد أيضاً على المتغيرات الاختيارية enabled_by_default و requires_output و priority.

يحتوي كائن Command على ثلاث خصائص: script, output و script_parts. يجب ألا تغيّر القاعدة كائن Command.

تغيّر واجهة برمجة تطبيقات القواعد في 3.0: للوصول إلى إعدادات القاعدة، استخدم from thefuck.conf import settings

كائن settings يتم تجميعه من ~/.config/thefuck/settings.py، وقيم من المتغيرات البيئية (انظر المزيد أدناه).

مثال بسيط لقاعدة لتشغيل سكريبت باستخدام sudo:

def match(command):
    return ('permission denied' in command.output.lower()
            or 'EACCES' in command.output)


def get_new_command(command):
    return 'sudo {}'.format(command.script)

# اختياري:
enabled_by_default = True

def side_effect(command, fixed_command):
    subprocess.call('chmod 777 .', shell=True)

priority = 1000  # الأقل أولاً، الافتراضي 1000

requires_output = True

المزيد من أمثلة القواعد, دوال مساعدة للقواعد, مساعدات خاصة بالتطبيق/النظام.

الإعدادات

يمكن تعديل عدة معلمات في The Fuck في الملف $XDG_CONFIG_HOME/thefuck/settings.py (افتراضياً $XDG_CONFIG_HOME هو ~/.config):

  • rules – قائمة القواعد المفعلة، افتراضياً thefuck.const.DEFAULT_RULES;
  • exclude_rules – قائمة القواعد المعطلة، افتراضياً [];
  • require_confirmation – يتطلب التأكيد قبل تشغيل الأمر الجديد، افتراضياً True;
  • wait_command – الحد الأقصى بالثواني للحصول على إخراج الأمر السابق؛
  • no_colors – تعطيل الإخراج الملون؛
  • priority – معجم بأولويات القواعد، القاعدة ذات الأولوية الأقل يتم مطابقتها أولاً؛
  • debug – تفعيل إخراج التصحيح، افتراضياً False;
  • history_limit – القيمة العددية لعدد أوامر السجل التي سيتم فحصها، مثل 2000;
  • alter_history – إضافة الأمر المصحح للسجل، افتراضياً True;
  • wait_slow_command – الحد الأقصى بالثواني للحصول على إخراج الأمر السابق إذا كان في قائمة slow_commands;
  • slow_commands – قائمة بالأوامر البطيئة؛
  • num_close_matches – الحد الأقصى لعدد الاقتراحات المتشابهة، افتراضياً 3.
  • excluded_search_path_prefixes – بادئات المسارات التي يجب تجاهلها عند البحث عن الأوامر، افتراضياً [].

مثال على settings.py:

rules = ['sudo', 'no_command']
exclude_rules = ['git_push']
require_confirmation = True
wait_command = 10
no_colors = False
priority = {'sudo': 100, 'no_command': 9999}
debug = False
history_limit = 9999
wait_slow_command = 20
slow_commands = ['react-native', 'gradle']
num_close_matches = 5

أو عبر المتغيرات البيئية:

  • THEFUCK_RULES – قائمة القواعد المفعلة، مثل DEFAULT_RULES:rm_root أو sudo:no_command;
  • THEFUCK_EXCLUDE_RULES – قائمة القواعد المعطلة، مثل git_pull:git_push;
  • THEFUCK_REQUIRE_CONFIRMATION – يتطلب التأكيد قبل تشغيل الأمر الجديد، true/false;
  • THEFUCK_WAIT_COMMAND – الحد الأقصى بالثواني للحصول على إخراج الأمر السابق؛
  • THEFUCK_NO_COLORS – تعطيل الإخراج الملون، true/false;
  • THEFUCK_PRIORITY – أولوية القواعد، مثل no_command=9999:apt_get=100, القاعدة ذات الأولوية الأقل يتم مطابقتها أولاً؛
  • THEFUCK_DEBUG – تفعيل إخراج التصحيح، true/false;
  • THEFUCK_HISTORY_LIMIT – عدد أوامر السجل التي سيتم فحصها، مثل 2000;
  • THEFUCK_ALTER_HISTORY – إضافة الأمر المصحح للسجل true/false;
  • THEFUCK_WAIT_SLOW_COMMAND – الحد الأقصى بالثواني للحصول على إخراج الأمر السابق إذا كان في قائمة slow_commands;
  • THEFUCK_SLOW_COMMANDS – قائمة الأوامر البطيئة، مثل lein:gradle;
  • THEFUCK_NUM_CLOSE_MATCHES – الحد الأقصى لعدد الاقتراحات المتشابهة، مثل 5.
  • THEFUCK_EXCLUDED_SEARCH_PATH_PREFIXES – بادئات المسارات لتجاهلها عند البحث عن الأوامر، افتراضياً [].

مثال:

export THEFUCK_RULES='sudo:no_command'
export THEFUCK_EXCLUDE_RULES='git_pull:git_push'
export THEFUCK_REQUIRE_CONFIRMATION='true'
export THEFUCK_WAIT_COMMAND=10
export THEFUCK_NO_COLORS='false'
export THEFUCK_PRIORITY='no_command=9999:apt_get=100'
export THEFUCK_HISTORY_LIMIT='2000'
export THEFUCK_NUM_CLOSE_MATCHES='5'

وضع السرعة الفورية التجريبي

السلوك الافتراضي لـ The Fuck يتطلب وقتاً لإعادة تشغيل الأوامر السابقة. في وضع السرعة الفورية، يوفر The Fuck الوقت عبر تسجيل الإخراج باستخدام script، ثم قراءة السجل.

حالياً، وضع السرعة الفورية يدعم فقط Python 3 مع bash أو zsh. يجب أيضاً تعطيل خاصية التصحيح التلقائي في zsh ليعمل thefuck بشكل صحيح.

لتفعيل وضع السرعة الفورية، أضف --enable-experimental-instant-mode إلى تهيئة الاسم المستعار في .bashrc أو .bash_profile أو .zshrc.

مثال:

eval $(thefuck --alias --enable-experimental-instant-mode)
