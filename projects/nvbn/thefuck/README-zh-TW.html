安裝

在 macOS 或 Linux 上，你可以透過 Homebrew 安裝 The Fuck：

brew install thefuck

在 Ubuntu / Mint 上，使用以下指令安裝 The Fuck：

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

在 FreeBSD 上，使用以下指令安裝 The Fuck：

pkg install thefuck

在 ChromeOS 上，使用 chromebrew 安裝 The Fuck：

crew install thefuck

在 Arch 系統上，使用以下指令安裝 The Fuck：

sudo pacman -S thefuck

在其他系統上，可使用 pip 安裝 The Fuck：

pip install thefuck

你也可以選擇使用 OS 套件管理工具（OS X、Ubuntu、Arch）。

# 建議你將下列指令放入你的 .bash_profile、 .bashrc、.zshrc 或其他啟動腳本中：

eval $(thefuck --alias)
# 你可以自訂別名，例如專屬星期一的:
eval $(thefuck --alias FUCK)

或放在你的 shell 設定檔（Bash、Zsh、Fish、Powershell、tcsh）。

更動只會在新開的 shell session 有效。若要立即生效，請執行 source ~/.bashrc（或你的 shell 設定檔，如 .zshrc）。

若要無需確認直接執行修正後指令，請加上 --yeah 參數（或簡寫 -y，或當你特別火大時用 --hard）：

fuck --yeah

若要遞迴修正指令直到成功，使用 -r 參數：

fuck -r

移除安裝

要移除 The Fuck，請反向操作安裝流程：

  • 刪除或註解 shell 設定檔（Bash、Zsh、Fish、Powershell、tcsh ...）中的 thefuck 別名行
  • 使用你的套件管理工具（brew、pip3、pkg、crew、pip）解除安裝 binary 檔案

自訂規則

若要新增自訂規則，請在 ~/.config/thefuck/rules 新增名為 your-rule-name.py 的檔案。規則檔必須包含兩個函式：

match(command: Command) -> bool
get_new_command(command: Command) -> str | list[str]

另外，規則可選擇性包含下列函式：

side_effect(old_command: Command, fixed_command: str) -> None

規則也可包含可選變數 enabled_by_default、requires_output 和 priority。

Command 有三個屬性：script、output 和 script_parts。 你的規則不應改變 Command。

3.0 版起 Rules api 有變更： 若要存取規則設定，請用 from thefuck.conf import settings 匯入

settings 是一個由 ~/.config/thefuck/settings.py 及環境變數組成的特殊物件（見下文）。

一個簡單的 sudo 規則範例：

def match(command):
    return ('permission denied' in command.output.lower()
            or 'EACCES' in command.output)


def get_new_command(command):
    return 'sudo {}'.format(command.script)

# 可選：
enabled_by_default = True

def side_effect(command, fixed_command):
    subprocess.call('chmod 777 .', shell=True)

priority = 1000  # 數值越小優先度越高，預設 1000

requires_output = True

更多規則範例， 規則輔助函式， 應用/作業系統專屬輔助程式。

  • rules – 啟用規則列表，預設為 thefuck.const.DEFAULT_RULES；
  • exclude_rules – 停用規則列表，預設為 []；
  • require_confirmation – 執行新指令前是否需確認，預設為 True；
  • wait_command – 取得上個指令輸出的最長等待秒數；
  • no_colors – 關閉彩色輸出；
  • priority – 規則優先權 dict，priority 越小的規則會優先比對；
  • debug – 啟用 debug 輸出，預設為 False；
  • history_limit – 掃描歷史指令的數量上限，如 2000；
  • alter_history – 是否將修正後的指令加入歷史紀錄，預設為 True；
  • wait_slow_command – 若在 slow_commands 列表時，取得指令輸出的最長等待秒數；
  • slow_commands – 慢速指令列表；
  • num_close_matches – 建議相似指令的數量上限，預設為 3。
  • excluded_search_path_prefixes – 搜尋指令時需忽略的路徑前綴，預設為 []。

settings.py 範例：

rules = ['sudo', 'no_command']
exclude_rules = ['git_push']
require_confirmation = True
wait_command = 10
no_colors = False
priority = {'sudo': 100, 'no_command': 9999}
debug = False
history_limit = 9999
wait_slow_command = 20
slow_commands = ['react-native', 'gradle']
num_close_matches = 5

或透過環境變數：

  • THEFUCK_RULES – 啟用規則列表，如 DEFAULT_RULES:rm_root 或 sudo:no_command；
  • THEFUCK_EXCLUDE_RULES – 停用規則列表，如 git_pull:git_push；
  • THEFUCK_REQUIRE_CONFIRMATION – 執行新指令前是否需確認，true/false；
  • THEFUCK_WAIT_COMMAND – 取得上個指令輸出的最長等待秒數；
  • THEFUCK_NO_COLORS – 關閉彩色輸出，true/false；
  • THEFUCK_PRIORITY – 規則優先權，如 no_command=9999:apt_get=100， priority 越小的規則會優先比對；
  • THEFUCK_DEBUG – 啟用 debug 輸出，true/false；
  • THEFUCK_HISTORY_LIMIT – 掃描歷史指令的數量上限，如 2000；
  • THEFUCK_ALTER_HISTORY – 是否將修正後的指令加入歷史紀錄 true/false；
  • THEFUCK_WAIT_SLOW_COMMAND – 若在 slow_commands 列表時，取得指令輸出的最長等待秒數；
  • THEFUCK_SLOW_COMMANDS – 慢速指令列表，如 lein:gradle；
  • THEFUCK_NUM_CLOSE_MATCHES – 建議相似指令的數量上限，如 5。
  • THEFUCK_EXCLUDED_SEARCH_PATH_PREFIXES – 搜尋指令時需忽略的路徑前綴，預設為 []。

例如：

export THEFUCK_RULES='sudo:no_command'
export THEFUCK_EXCLUDE_RULES='git_pull:git_push'
export THEFUCK_REQUIRE_CONFIRMATION='true'
export THEFUCK_WAIT_COMMAND=10
export THEFUCK_NO_COLORS='false'
export THEFUCK_PRIORITY='no_command=9999:apt_get=100'
export THEFUCK_HISTORY_LIMIT='2000'
export THEFUCK_NUM_CLOSE_MATCHES='5'

含規則的第三方套件

如果你想建立一組特定但不公開的規則，並希望與其他人分享，可以建立一個名為 thefuck_contrib_* 的套件，結構如下：

thefuck_contrib_foo
  thefuck_contrib_foo
    rules
      __init__.py
      *第三方規則*
    __init__.py
    *第三方輔助工具*
  setup.py

The Fuck 會自動尋找 rules 模組內的規則。

實驗性即時模式

The Fuck 的預設行為需要花時間重新執行上一個指令。 在即時模式下，The Fuck 透過用 script 記錄輸出、再讀取 log 來節省時間。

目前即時模式僅支援 Python 3 搭配 bash 或 zsh。zsh 也需關閉自動校正功能，thefuck 才能正常運作。

啟用即時模式，請在 .bashrc、.bash_profile 或 .zshrc 的 alias 初始化中加上 --enable-experimental-instant-mode。

例如：

eval $(thefuck --alias --enable-experimental-instant-mode)
