نصب

در macOS یا لینوکس، می‌توانید The Fuck را از طریق Homebrew نصب کنید:

brew install thefuck

در Ubuntu / Mint، The Fuck را با دستورات زیر نصب کنید:

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

در FreeBSD، The Fuck را با دستورات زیر نصب کنید:

pkg install thefuck

در ChromeOS، The Fuck را با استفاده از chromebrew با دستور زیر نصب کنید:

crew install thefuck

در سیستم‌های مبتنی بر Arch، The Fuck را با دستور زیر نصب کنید:

sudo pacman -S thefuck

در سایر سیستم‌ها، The Fuck را با استفاده از pip نصب کنید:

pip install thefuck

یا می‌توانید از مدیر بسته سیستم عامل خود (OS X، Ubuntu، Arch) استفاده کنید.

# توصیه می‌شود این دستور را در فایل .bash_profile، .bashrc، .zshrc یا سایر اسکریپت‌های ابتدایی خود قرار دهید:

eval $(thefuck --alias)
# می‌توانید هر نام مستعاری که خواستید استفاده کنید، مثلاً برای روزهای دوشنبه:
eval $(thefuck --alias FUCK)

یا در تنظیمات شل خود (Bash, Zsh, Fish, Powershell, tcsh).

تغییرات فقط در نشست جدید شل در دسترس هستند. برای اعمال فوری تغییرات، دستور source ~/.bashrc (یا فایل تنظیمات شل خود مانند .zshrc) را اجرا کنید.

برای اجرای دستورات اصلاح‌شده بدون تأیید، از گزینه --yeah (یا فقط -y برای کوتاهی، یا --hard اگر خیلی ناراحت هستید) استفاده کنید:

fuck --yeah

برای اصلاح بازگشتی دستورات تا موفقیت، از گزینه -r استفاده کنید:

fuck -r

حذف نصب

برای حذف The Fuck، مراحل نصب را معکوس کنید:

  • خط alias مربوط به thefuck را از فایل تنظیمات شل خود (Bash, Zsh, Fish, Powershell, tcsh و ...) پاک یا کامنت کنید.
  • با استفاده از مدیر بسته خود (brew, pip3, pkg, crew, pip) فایل‌های اجرایی را حذف کنید.

  • apt_get_search – تغییر جستجو با apt-get به جستجو با apt-cache;
  • apt_invalid_operation – اصلاح فراخوانی اشتباه apt و apt-get مانند apt-get isntall vim;
  • apt_list_upgradable – کمک به اجرای apt list --upgradable بعد از apt update;
  • apt_upgrade – کمک به اجرای apt upgrade بعد از apt list --upgradable;
  • brew_cask_dependency – نصب وابستگی‌های cask;
  • brew_install – اصلاح نام فرمول برای brew install;
  • brew_reinstall – تبدیل brew install <formula> به brew reinstall <formula>;
  • brew_link – افزودن --overwrite --dry-run در صورت شکست لینک;
  • brew_uninstall – افزودن --force به brew uninstall اگر چند نسخه نصب شده باشد;
  • brew_unknown_command – اصلاح دستورات اشتباه brew، مثل brew docto/brew doctor;
  • brew_update_formula – تبدیل brew update <formula> به brew upgrade <formula>;
  • dnf_no_such_command – اصلاح دستورات اشتباه DNF;
  • nixos_cmd_not_found – نصب برنامه‌ها در NixOS;
  • pacman – نصب برنامه با pacman اگر نصب نشده باشد (از yay، pikaur یا yaourt اگر موجود باشد استفاده می‌کند);
  • pacman_invalid_option – جایگزینی گزینه‌های کوچک pacman با بزرگ.
  • pacman_not_found – اصلاح نام بسته با pacman، yay، pikaur یا yaourt.
  • yum_invalid_operation – اصلاح فراخوانی اشتباه yum مانند yum isntall vim;

دستورات زیر با The Fuck ارائه می‌شوند، اما به طور پیش‌فرض فعال نیستند:

  • git_push_force – افزودن --force-with-lease به git push (ممکن است با git_push_pull تداخل داشته باشد);
  • rm_root – افزودن --no-preserve-root به دستور rm -rf /.

ساخت قوانین شخصی

برای افزودن قانون خود، فایلی با نام your-rule-name.py در مسیر ~/.config/thefuck/rules بسازید. فایل قانون باید شامل دو تابع باشد:

match(command: Command) -> bool
get_new_command(command: Command) -> str | list[str]

علاوه بر این، قوانین می‌توانند توابع اختیاری نیز داشته باشند:

side_effect(old_command: Command, fixed_command: str) -> None

قوانین همچنین می‌توانند متغیرهای اختیاری enabled_by_default، requires_output و priority را داشته باشند.

Command دارای سه ویژگی است: script، output و script_parts. قانون شما نباید Command را تغییر دهد.

API قوانین در نسخه ۳.۰ تغییر کرده است: برای دسترسی به تنظیمات قانون، آن را با from thefuck.conf import settings وارد کنید.

settings یک شیء ویژه است که از ~/.config/thefuck/settings.py و مقادیر env (در ادامه ببینید) ساخته می‌شود.

مثالی ساده از قانون برای اجرای یک اسکریپت با sudo:

def match(command):
    return ('permission denied' in command.output.lower()
            or 'EACCES' in command.output)


def get_new_command(command):
    return 'sudo {}'.format(command.script)

# اختیاری:
enabled_by_default = True

def side_effect(command, fixed_command):
    subprocess.call('chmod 777 .', shell=True)

priority = 1000  # کوچکتر اول، پیش‌فرض 1000

requires_output = True

نمونه‌های بیشتر قوانین, توابع کمکی برای قوانین, ابزارهای ویژه برای برنامه/سیستم‌عامل.

تنظیمات

چندین پارامتر The Fuck را می‌توان در فایل $XDG_CONFIG_HOME/thefuck/settings.py (به طور پیش‌فرض $XDG_CONFIG_HOME برابر ~/.config است) تغییر داد:

  • rules – لیست قوانین فعال، به طور پیش‌فرض thefuck.const.DEFAULT_RULES;
  • exclude_rules – لیست قوانین غیرفعال، به طور پیش‌فرض [];
  • require_confirmation – نیاز به تأیید قبل از اجرای دستور جدید، به طور پیش‌فرض True;
  • wait_command – حداکثر زمان (ثانیه) برای دریافت خروجی دستور قبلی;
  • no_colors – غیرفعال کردن خروجی رنگی;
  • priority – دیکشنری با اولویت قوانین، قانونی با اولویت پایین‌تر زودتر بررسی می‌شود;
  • debug – فعال‌سازی خروجی debug، به طور پیش‌فرض False;
  • history_limit – تعداد دستورات تاریخچه که اسکن می‌شوند، مانند 2000;
  • alter_history – افزودن دستور اصلاح‌شده به تاریخچه، به طور پیش‌فرض True;
  • wait_slow_command – حداکثر زمان (ثانیه) برای دریافت خروجی دستور قبلی اگر در لیست slow_commands باشد;
  • slow_commands – لیست دستورات کند;
  • num_close_matches – حداکثر تعداد پیشنهاد دستورات مشابه، پیش‌فرض 3.
  • excluded_search_path_prefixes – پیشوند مسیرهایی که در جستجوی دستورات نادیده گرفته می‌شوند، پیش‌فرض [].

نمونه‌ای از settings.py:

rules = ['sudo', 'no_command']
exclude_rules = ['git_push']
require_confirmation = True
wait_command = 10
no_colors = False
priority = {'sudo': 100, 'no_command': 9999}
debug = False
history_limit = 9999
wait_slow_command = 20
slow_commands = ['react-native', 'gradle']
num_close_matches = 5

یا از طریق متغیرهای محیطی:

  • THEFUCK_RULES – لیست قوانین فعال، مانند DEFAULT_RULES:rm_root یا sudo:no_command;
  • THEFUCK_EXCLUDE_RULES – لیست قوانین غیرفعال، مانند git_pull:git_push;
  • THEFUCK_REQUIRE_CONFIRMATION – نیاز به تأیید قبل از اجرای دستور جدید، true/false;
  • THEFUCK_WAIT_COMMAND – حداکثر زمان (ثانیه) برای دریافت خروجی دستور قبلی;
  • THEFUCK_NO_COLORS – غیرفعال کردن خروجی رنگی، true/false;
  • THEFUCK_PRIORITY – اولویت قوانین، مانند no_command=9999:apt_get=100, قانون با اولویت پایین‌تر زودتر بررسی می‌شود;
  • THEFUCK_DEBUG – فعال‌سازی خروجی debug، true/false;
  • THEFUCK_HISTORY_LIMIT – تعداد دستورات تاریخچه که اسکن می‌شوند، مانند 2000;
  • THEFUCK_ALTER_HISTORY – افزودن دستور اصلاح‌شده به تاریخچه true/false;
  • THEFUCK_WAIT_SLOW_COMMAND – حداکثر زمان (ثانیه) برای دریافت خروجی دستور قبلی اگر در لیست slow_commands باشد;
  • THEFUCK_SLOW_COMMANDS – لیست دستورات کند، مانند lein:gradle;
  • THEFUCK_NUM_CLOSE_MATCHES – حداکثر تعداد پیشنهاد دستورات مشابه، مانند 5.
  • THEFUCK_EXCLUDED_SEARCH_PATH_PREFIXES – پیشوند مسیرهایی که در جستجوی دستورات نادیده گرفته می‌شوند، پیش‌فرض [].

برای مثال:

export THEFUCK_RULES='sudo:no_command'
export THEFUCK_EXCLUDE_RULES='git_pull:git_push'
export THEFUCK_REQUIRE_CONFIRMATION='true'
export THEFUCK_WAIT_COMMAND=10
export THEFUCK_NO_COLORS='false'
export THEFUCK_PRIORITY='no_command=9999:apt_get=100'
export THEFUCK_HISTORY_LIMIT='2000'
export THEFUCK_NUM_CLOSE_MATCHES='5'

thefuck_contrib_foo

      __init__.py
      *قوانین شخص ثالث*
    __init__.py
    *ابزارهای شخص ثالث*
  setup.py

The Fuck قوانین موجود در ماژول rules را پیدا خواهد کرد.

حالت آزمایشی فوری

رفتار پیش‌فرض The Fuck نیاز به زمان برای اجرای مجدد دستورات قبلی دارد. در حالت فوری، The Fuck با ثبت خروجی توسط script, سپس خواندن لاگ، زمان را ذخیره می‌کند.

در حال حاضر، حالت فوری فقط Python 3 با bash یا zsh را پشتیبانی می‌کند. عملکرد autocorrect در zsh نیز باید غیرفعال شود تا thefuck درست کار کند.

برای فعال‌سازی حالت فوری، --enable-experimental-instant-mode را به مقداردهی اولیه alias در .bashrc, .bash_profile یا .zshrc اضافه کنید.

برای مثال:

eval $(thefuck --alias --enable-experimental-instant-mode)
