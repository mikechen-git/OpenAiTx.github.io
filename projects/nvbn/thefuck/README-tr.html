İçindekilere Dön

Kurulum

macOS veya Linux üzerinde The Fuck'ı Homebrew ile kurabilirsiniz:

brew install thefuck

Ubuntu / Mint üzerinde The Fuck aşağıdaki komutlarla kurulur:

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

FreeBSD üzerinde The Fuck aşağıdaki komutlarla kurulur:

pkg install thefuck

ChromeOS üzerinde The Fuck, chromebrew ile şu komutla kurulur:

crew install thefuck

Arch tabanlı sistemlerde The Fuck aşağıdaki komutla kurulur:

sudo pacman -S thefuck

Diğer sistemlerde The Fuck'ı pip ile kurabilirsiniz:

pip install thefuck

Alternatif olarak, bir işletim sistemi paket yöneticisi (OS X, Ubuntu, Arch) de kullanılabilir.

# Bu komutu .bash_profile, .bashrc, .zshrc veya başka bir başlangıç betiğinize eklemeniz önerilir:

eval $(thefuck --alias)
# Pazartesiler için başka bir kısayol da kullanabilirsiniz:
eval $(thefuck --alias FUCK)

Ya da kabuk yapılandırmanıza (Bash, Zsh, Fish, Powershell, tcsh) ekleyin.

Değişiklikler yalnızca yeni bir kabuk oturumunda geçerli olur. Değişiklikleri hemen etkinleştirmek için source ~/.bashrc (veya .zshrc gibi kabuk dosyanızı) çalıştırın.

Düzeltilen komutları onay almadan çalıştırmak için --yeah seçeneğini kullanın (veya kısaca -y, veya özellikle sinirliyseniz --hard):

fuck --yeah

Komutları başarılı olana kadar özyinelemeli olarak düzeltmek için -r seçeneğini kullanın:

fuck -r

Güncelleme

pip3 install thefuck --upgrade

Kaldırma

The Fuck'ı kaldırmak için kurulum işlemini tersine çevirin:

  • Bash, Zsh, Fish, Powershell, tcsh, ... kabuk yapılandırmanızdan thefuck takma ad satırını silin veya yorum satırı yapın.
  • Paket yöneticinizi (brew, pip3, pkg, crew, pip) kullanarak ikili dosyaları kaldırın.

İçindekilere Dön

Kendi kurallarınızı oluşturma

Kendi kuralınızı eklemek için ~/.config/thefuck/rules dizininde your-rule-name.py adında bir dosya oluşturun. Kural dosyası şu iki fonksiyonu içermelidir:

match(command: Command) -> bool
get_new_command(command: Command) -> str | list[str]

Ek olarak, kurallar isteğe bağlı fonksiyonlar da içerebilir:

side_effect(old_command: Command, fixed_command: str) -> None

Kurallar ayrıca enabled_by_default, requires_output ve priority gibi isteğe bağlı değişkenler de içerebilir.

Command üç özelliğe sahiptir: script, output ve script_parts. Kuralınız Command'ı değiştirmemelidir.

Kurallar API'si 3.0'da değişti: Bir kuralın ayarlarına erişmek için şunu içe aktarın: from thefuck.conf import settings

settings, ~/.config/thefuck/settings.py ve ortam değişkenlerinden (aşağıya bakınız) derlenen özel bir nesnedir.

Bir scripti sudo ile çalıştırmak için basit bir örnek kural:

def match(command):
    return ('permission denied' in command.output.lower()
            or 'EACCES' in command.output)


def get_new_command(command):
    return 'sudo {}'.format(command.script)

# Opsiyonel:
enabled_by_default = True

def side_effect(command, fixed_command):
    subprocess.call('chmod 777 .', shell=True)

priority = 1000  # Küçük öncelik önce, varsayılan 1000'dir

requires_output = True

Daha fazla kural örneği, kurallar için yardımcı fonksiyonlar, uygulama/işletim sistemi özel yardımcılar.

Birçok The Fuck parametresi, $XDG_CONFIG_HOME/thefuck/settings.py dosyasında ($XDG_CONFIG_HOME varsayılan olarak ~/.config) değiştirilebilir:

  • rules – etkin kuralların listesi, varsayılan olarak thefuck.const.DEFAULT_RULES;
  • exclude_rules – devre dışı bırakılan kuralların listesi, varsayılan olarak [];
  • require_confirmation – yeni komutu çalıştırmadan önce onay ister, varsayılanı True;
  • wait_command – önceki komut çıktısını almak için maksimum saniye;
  • no_colors – renkli çıktıyı devre dışı bırakır;
  • priority – kuralların öncelik sözlüğü, düşük priority ilk eşleşir;
  • debug – hata ayıklama çıktısını etkinleştirir, varsayılanı False;
  • history_limit – kaç komutun geçmişte taranacağını sayısal olarak belirtir, örn: 2000;
  • alter_history – düzeltilen komutu geçmişe ekler, varsayılanı True;
  • wait_slow_command – slow_commands listesindeyse önceki komut çıktısını almak için maksimum saniye;
  • slow_commands – yavaş komutlar listesi;
  • num_close_matches – önerilecek en fazla benzer komut sayısı, varsayılan 3;
  • excluded_search_path_prefixes – komut ararken göz ardı edilecek yol önekleri, varsayılanı [].

Bir settings.py örneği:

rules = ['sudo', 'no_command']
exclude_rules = ['git_push']
require_confirmation = True
wait_command = 10
no_colors = False
priority = {'sudo': 100, 'no_command': 9999}
debug = False
history_limit = 9999
wait_slow_command = 20
slow_commands = ['react-native', 'gradle']
num_close_matches = 5

Veya ortam değişkenleriyle:

  • THEFUCK_RULES – etkin kuralların listesi, örn: DEFAULT_RULES:rm_root veya sudo:no_command;
  • THEFUCK_EXCLUDE_RULES – devre dışı bırakılan kurallar, örn: git_pull:git_push;
  • THEFUCK_REQUIRE_CONFIRMATION – yeni komutu çalıştırmadan önce onay ister, true/false;
  • THEFUCK_WAIT_COMMAND – önceki komut çıktısını almak için maksimum saniye;
  • THEFUCK_NO_COLORS – renkli çıktıyı devre dışı bırakır, true/false;
  • THEFUCK_PRIORITY – kuralların önceliği, örn: no_command=9999:apt_get=100, düşük priority ilk eşleşir;
  • THEFUCK_DEBUG – hata ayıklama çıktısını etkinleştirir, true/false;
  • THEFUCK_HISTORY_LIMIT – kaç komutun geçmişte taranacağı, örn: 2000;
  • THEFUCK_ALTER_HISTORY – düzeltilen komutu geçmişe ekler, true/false;
  • THEFUCK_WAIT_SLOW_COMMAND – slow_commands listesinde ise önceki komut çıktısını almak için maksimum saniye;
  • THEFUCK_SLOW_COMMANDS – yavaş komutlar listesi, örn: lein:gradle;
  • THEFUCK_NUM_CLOSE_MATCHES – önerilecek maksimum benzer komut sayısı, örn: 5;
  • THEFUCK_EXCLUDED_SEARCH_PATH_PREFIXES – komut ararken göz ardı edilecek yol önekleri, varsayılanı [].

Örneğin:

export THEFUCK_RULES='sudo:no_command'
export THEFUCK_EXCLUDE_RULES='git_pull:git_push'
export THEFUCK_REQUIRE_CONFIRMATION='true'
export THEFUCK_WAIT_COMMAND=10
export THEFUCK_NO_COLORS='false'
export THEFUCK_PRIORITY='no_command=9999:apt_get=100'
export THEFUCK_HISTORY_LIMIT='2000'
export THEFUCK_NUM_CLOSE_MATCHES='5'

Kurallı üçüncü taraf paketler

Özel bir dizi kapalı kural oluşturmak istiyor, fakat başkalarıyla da paylaşmak istiyorsanız, aşağıdaki yapıda thefuck_contrib_* adlı bir paket oluşturun:

thefuck_contrib_foo
  thefuck_contrib_foo
    rules
      __init__.py
      *üçüncü taraf kurallar*
    __init__.py
    *üçüncü taraf yardımcılar*
  setup.py

The Fuck kuralları rules modülünde bulacaktır.

Deneysel anında mod

The Fuck'ın varsayılan davranışı, önceki komutları tekrar çalıştırmak için zaman gerektirir. Anında modda, The Fuck script ile çıktıyı kaydederek zamandan tasarruf eder ve kaydı okur.

Şu anda anında mod yalnızca bash veya zsh ile Python 3'ü desteklemektedir. thefuck'ın düzgün çalışması için zsh'nin otomatik düzeltme işlevinin de devre dışı bırakılması gerekir.

Anında modu etkinleştirmek için .bashrc, .bash_profile veya .zshrc'daki alias başlatma kısmına --enable-experimental-instant-mode ekleyin.

Örneğin:

eval $(thefuck --alias --enable-experimental-instant-mode)
