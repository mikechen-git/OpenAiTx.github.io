इंस्टॉलेशन

macOS या Linux पर, आप Homebrew के माध्यम से द फक इंस्टॉल कर सकते हैं:

brew install thefuck

Ubuntu / Mint पर, द फक को निम्नलिखित कमांड्स से इंस्टॉल करें:

sudo apt update
sudo apt install python3-dev python3-pip python3-setuptools
pip3 install thefuck --user

FreeBSD पर, द फक को निम्नलिखित कमांड्स से इंस्टॉल करें:

pkg install thefuck

ChromeOS पर, chromebrew का उपयोग करके द फक इंस्टॉल करें:

crew install thefuck

Arch बेस्ड सिस्टम्स पर, द फक को निम्नलिखित कमांड से इंस्टॉल करें:

sudo pacman -S thefuck

अन्य सिस्टम्स पर, द फक को pip का उपयोग करके इंस्टॉल करें:

pip install thefuck

वैकल्पिक रूप से, आप OS पैकेज मैनेजर (OS X, Ubuntu, Arch) का उपयोग कर सकते हैं।

# यह अनुशंसा की जाती है कि आप इस कमांड को अपनी .bash_profile, .bashrc, .zshrc या अन्य स्टार्टअप स्क्रिप्ट में रखें:

eval $(thefuck --alias)
# आप चाहें तो कोई भी alias इस्तेमाल कर सकते हैं, जैसे सोमवार के लिए:
eval $(thefuck --alias FUCK)

या अपनी शेल कॉन्फ़िग (Bash, Zsh, Fish, Powershell, tcsh) में।

परिवर्तन केवल नए शेल सेशन में उपलब्ध होंगे। तुरंत प्रभाव के लिए, source ~/.bashrc (या आपकी शेल कॉन्फ़िग फाइल जैसे .zshrc) चलाएँ।

पुष्टि के बिना फिक्स्ड कमांड्स चलाने के लिए, --yeah विकल्प का उपयोग करें (या केवल -y के लिए, या अगर आप बहुत परेशान हैं तो --hard):

fuck --yeah

किसी कमांड को सफल होने तक पुनरावृत्त रूप से ठीक करने के लिए, -r विकल्प का उपयोग करें:

fuck -r
सामग्री पर वापस जाएँ

अपडेट करना

pip3 install thefuck --upgrade

अनइंस्टॉल

द फक को हटाने के लिए, इंस्टॉलेशन प्रक्रिया को उल्टा करें:

  • अपनी Bash, Zsh, Fish, Powershell, tcsh, ... शेल कॉन्फ़िग से thefuck alias लाइन मिटाएँ या कमेंट करें
  • अपने पैकेज मैनेजर (brew, pip3, pkg, crew, pip) का उपयोग करके बाइनरीज़ को अनइंस्टॉल करें

  • python_execute – Python फाइल्स चलाते समय मिसिंग .py जोड़ता है;

  • rm_dir – डायरेक्टरी हटाते समय -rf जोड़ता है;
  • scm_correction – गलत scm को सही करता है जैसे hg log को git log;

  • ssh_known_hosts – चेतावनी मिलने पर known_hosts से होस्ट हटाता है;

  • sudo_command_from_user_path – यूज़र के $PATH से कमांड्स को sudo के साथ चलाता है;
  • switch_lang – कमांड को आपकी लोकल लेआउट से en में बदलता है;
  • systemctl – systemctl के confusing parameters को सही क्रम में लगाता है;
  • terraform_init.py – plan या apply से पहले terraform init चलाता है;
  • terraform_no_command.py – अनरिज़नाइज़्ड terraform कमांड्स को ठीक करता है;

  • touch – "touch" करने से पहले मिसिंग डायरेक्टरी बनाता है;
  • tsuru_login – ऑथेंटिकेटेड न होने या सेशन एक्सपायर होने पर tsuru login चलाता है;
  • tsuru_not_command – गलत tsuru कमांड्स जैसे tsuru shell को ठीक करता है;
  • tmux – tmux कमांड्स को ठीक करता है;
  • unknown_command – hadoop hdfs-स्टाइल "unknown command" को ठीक करता है;
  • unsudo – पिछले कमांड से sudo हटाता है यदि प्रोसेस superuser privilege पर चलने से मना करे।
  • vagrant_up – vagrant instance चालू करता है;

अपना खुद का रूल बनाना

अपना खुद का रूल जोड़ने के लिए, ~/.config/thefuck/rules में your-rule-name.py नामक फाइल बनाएं। रूल फाइल में दो फंक्शन होने चाहिए:

match(command: Command) -> bool
get_new_command(command: Command) -> str | list[str]

इसके अलावा, रूल्स में वैकल्पिक फंक्शन हो सकते हैं:

side_effect(old_command: Command, fixed_command: str) -> None

रूल्स में वैकल्पिक वेरिएबल्स enabled_by_default, requires_output और priority भी हो सकते हैं।

Command में तीन attributes होते हैं: script, output और script_parts। आपका रूल Command को नहीं बदलना चाहिए।

रूल्स API v3.0 में बदला: रूल की सेटिंग्स एक्सेस करने के लिए from thefuck.conf import settings इम्पोर्ट करें

settings एक विशेष ऑब्जेक्ट है जो ~/.config/thefuck/settings.py, और env वेल्यूज़ से बनता है (नीचे और देखें)।

एक सिंपल उदाहरण sudo के साथ स्क्रिप्ट चलाने का:

def match(command):
    return ('permission denied' in command.output.lower()
            or 'EACCES' in command.output)


def get_new_command(command):
    return 'sudo {}'.format(command.script)

# वैकल्पिक:
enabled_by_default = True

def side_effect(command, fixed_command):
    subprocess.call('chmod 777 .', shell=True)

priority = 1000  # कम पहले, डिफ़ॉल्ट 1000

requires_output = True

अधिक रूल्स के उदाहरण, रूल्स के लिए यूटिलिटी फंक्शन, ऐप/ओएस-विशिष्ट हेल्पर्स।

सेटिंग्स

कई द फक पैरामीटर्स को $XDG_CONFIG_HOME/thefuck/settings.py ($XDG_CONFIG_HOME डिफ़ॉल्ट रूप से ~/.config) में बदला जा सकता है:

  • rules – सक्षम रूल्स की लिस्ट, डिफ़ॉल्ट thefuck.const.DEFAULT_RULES;
  • exclude_rules – अक्षम रूल्स की लिस्ट, डिफ़ॉल्ट [];
  • require_confirmation – नई कमांड चलाने से पहले पुष्टि आवश्यक, डिफ़ॉल्ट True;
  • wait_command – पिछली कमांड के आउटपुट के लिए अधिकतम समय (सेकंड में);
  • no_colors – रंगीन आउटपुट अक्षम करें;
  • priority – रूल्स की प्राथमिकता, कम priority वाला रूल पहले मैच होगा;
  • debug – डिबग आउटपुट सक्षम करें, डिफ़ॉल्ट False;
  • history_limit – कितनी history कमांड्स स्कैन होंगी, जैसे 2000;
  • alter_history – फिक्स्ड कमांड को history में डालना, डिफ़ॉल्ट True;
  • wait_slow_command – स्लो कमांड्स की लिस्ट में होने पर अधिकतम प्रतीक्षा समय;
  • slow_commands – स्लो कमांड्स की लिस्ट;
  • num_close_matches – सुझाए जाने वाले अधिकतम करीबी मैचों की संख्या, डिफ़ॉल्ट 3;
  • excluded_search_path_prefixes – कमांड खोजते समय नजरअंदाज किए जाने वाले पाथ प्रीफिक्स, डिफ़ॉल्ट [];

settings.py का एक उदाहरण:

rules = ['sudo', 'no_command']
exclude_rules = ['git_push']
require_confirmation = True
wait_command = 10
no_colors = False
priority = {'sudo': 100, 'no_command': 9999}
debug = False
history_limit = 9999
wait_slow_command = 20
slow_commands = ['react-native', 'gradle']
num_close_matches = 5

या एनवायरनमेंट वेरिएबल्स के माध्यम से:

  • THEFUCK_RULES – सक्षम रूल्स की लिस्ट;
  • THEFUCK_EXCLUDE_RULES – अक्षम रूल्स की लिस्ट;
  • THEFUCK_REQUIRE_CONFIRMATION – नई कमांड से पहले पुष्टि आवश्यक, true/false;
  • THEFUCK_WAIT_COMMAND – पिछली कमांड के आउटपुट के लिए अधिकतम समय;
  • THEFUCK_NO_COLORS – रंगीन आउटपुट अक्षम करें, true/false;
  • THEFUCK_PRIORITY – रूल्स की प्राथमिकता;
  • THEFUCK_DEBUG – डिबग आउटपुट सक्षम करें, true/false;
  • THEFUCK_HISTORY_LIMIT – कितनी history कमांड्स स्कैन होंगी;
  • THEFUCK_ALTER_HISTORY – फिक्स्ड कमांड को history में डालना true/false;
  • THEFUCK_WAIT_SLOW_COMMAND – स्लो कमांड्स के लिए अधिकतम प्रतीक्षा समय;
  • THEFUCK_SLOW_COMMANDS – स्लो कमांड्स की लिस्ट;
  • THEFUCK_NUM_CLOSE_MATCHES – सुझाए जाने वाले अधिकतम करीबी मैचों की संख्या;
  • THEFUCK_EXCLUDED_SEARCH_PATH_PREFIXES – कमांड खोजते समय नजरअंदाज किए जाने वाले पाथ प्रीफिक्स, डिफ़ॉल्ट [];

उदाहरण:

export THEFUCK_RULES='sudo:no_command'
export THEFUCK_EXCLUDE_RULES='git_pull:git_push'
export THEFUCK_REQUIRE_CONFIRMATION='true'
export THEFUCK_WAIT_COMMAND=10
export THEFUCK_NO_COLORS='false'
export THEFUCK_PRIORITY='no_command=9999:apt_get=100'
export THEFUCK_HISTORY_LIMIT='2000'
export THEFUCK_NUM_CLOSE_MATCHES='5'

थर्ड पार्टी पैकेजेस

यदि आप कुछ खास गैर-पब्लिक रूल्स बनाना चाहते हैं, लेकिन दूसरों के साथ साझा भी करना चाहते हैं, तो thefuck_contrib_* नाम से पैकेज बनाएं, जिसमें निम्नलिखित संरचना हो:

thefuck_contrib_foo
  thefuck_contrib_foo
    rules
      __init__.py
      *third-party rules*
    __init__.py
    *third-party-utils*
  setup.py

द फक rules मॉड्यूल में मौजूद रूल्स को खोज लेगा।

एक्सपेरिमेंटल इंस्टेंट मोड

द फक का डिफ़ॉल्ट व्यवहार पिछली कमांड्स को फिर से चलाने के लिए समय लेता है। इंस्टेंट मोड में, द फक script के साथ आउटपुट लॉग करता है, फिर लॉग को पढ़ता है।

फिलहाल, इंस्टेंट मोड केवल Python 3 के साथ bash या zsh को सपोर्ट करता है। zsh की autocorrect फ़ंक्शन को भी अक्षम करना आवश्यक है।

इंस्टेंट मोड सक्षम करने के लिए, .bashrc, .bash_profile या .zshrc में alias initialization में --enable-experimental-instant-mode जोड़ें।

उदाहरण:

eval $(thefuck --alias --enable-experimental-instant-mode)
