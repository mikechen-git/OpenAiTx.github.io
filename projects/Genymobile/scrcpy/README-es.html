scrcpy (v3.2)

pronunciado "screen copy"

Esta aplicación refleja dispositivos Android (video y audio) conectados vía USB o por TCP/IP y permite el control usando el teclado y ratón del ordenador. No requiere acceso root ni una app instalada en el dispositivo. Funciona en Linux, Windows y macOS.

Se enfoca en:

  • ligereza: nativo, muestra solo la pantalla del dispositivo
  • rendimiento: 30~120fps, dependiendo del dispositivo
  • calidad: 1920×1080 o superior
  • baja latencia: 35~70ms
  • bajo tiempo de inicio: ~1 segundo para mostrar la primera imagen
  • no intrusivo: no deja nada instalado en el dispositivo Android
  • beneficios para el usuario: sin cuenta, sin anuncios, sin internet requerido
  • libertad: software libre y de código abierto

Sus funciones incluyen:

  • reenvío de audio (Android 11+)
  • grabación
  • pantalla virtual
  • duplicación con pantalla del dispositivo Android apagada
  • copiar y pegar en ambas direcciones
  • calidad configurable
  • duplicado de cámara (Android 12+)
  • duplicación como webcam (V4L2) (solo en Linux)
  • simulación física de teclado y ratón (HID)
  • soporte de gamepad
  • modo OTG
  • y más…

Requisitos previos

El dispositivo Android requiere al menos API 21 (Android 5.0).

El reenvío de audio es compatible para API >= 30 (Android 11+).

Asegúrate de haber activado la depuración USB en tu(s) dispositivo(s).

En algunos dispositivos (especialmente Xiaomi), podrías obtener el siguiente error:

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

En ese caso, necesitas habilitar una opción adicional Depuración USB (Ajustes de seguridad) (esto es diferente de Depuración USB) para poder controlarlo usando teclado y ratón. Es necesario reiniciar el dispositivo una vez que esta opción esté activada.

Ten en cuenta que no se requiere la depuración USB para ejecutar scrcpy en modo OTG.

Consejos imprescindibles

  • Reducir la resolución puede mejorar mucho el rendimiento (scrcpy -m1024)
  • Clic derecho acciona ATRÁS
  • Clic central acciona INICIO
  • Alt+f alterna pantalla completa
  • Hay muchos otros atajos

Ejemplos de uso

Hay muchas opciones documentadas en páginas separadas. Aquí tienes algunos ejemplos comunes.

  • Captura la pantalla en H.265 (mejor calidad), limita el tamaño a 1920, limita la tasa de fotogramas a 60fps, desactiva el audio y controla el dispositivo simulando un teclado físico:

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # versión corta

  • Inicia VLC en una nueva pantalla virtual (separada de la pantalla del dispositivo):

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • Graba la cámara del dispositivo en H.265 a 1920x1080 (y micrófono) en un archivo MP4:

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • Captura la cámara frontal del dispositivo y expónla como webcam en el ordenador (en Linux):

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • Controla el dispositivo sin duplicar la pantalla, simulando un teclado y ratón físicos (no requiere depuración USB):

    scrcpy --otg

  • Controla el dispositivo usando mandos de juego conectados al ordenador:

    scrcpy --gamepad=uhid
scrcpy -G  # versión corta
