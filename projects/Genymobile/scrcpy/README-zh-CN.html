此 GitHub 仓库（https://github.com/Genymobile/scrcpy）是该项目唯一的官方来源。不要从随机网站下载发布版，即使其名称中包含 scrcpy。

scrcpy（v3.2）

发音为 “screen copy”

本应用可镜像通过 USB 或 TCP/IP 连接的 Android 设备（视频与音频），并允许通过电脑的键盘和鼠标进行控制。它不需要 root 权限或在设备上安装应用。支持 Linux、Windows 和 macOS。

其特点为：

  • 轻量：原生，只显示设备屏幕
  • 高性能：30~120fps，取决于设备
  • 高画质：1920×1080 或更高
  • 低延迟：35~70ms
  • 启动快：~1 秒即可显示首帧画面
  • 非侵入性：不会在 Android 设备上留下任何安装内容
  • 用户友好：无需账号、无广告、无需联网
  • 自由：自由开源软件

其功能包括：

  • 音频转发（Android 11+）
  • 录屏
  • 虚拟显示
  • 在 Android 设备屏幕关闭时镜像
  • 双向复制粘贴
  • 可配置画质
  • 相机镜像（Android 12+）
  • 作为网络摄像头（V4L2）镜像（仅限 Linux）
  • 物理键盘和鼠标模拟（HID）
  • 手柄支持
  • OTG 模式
  • 以及更多功能……

先决条件

Android 设备需至少为 API 21（Android 5.0）。

音频转发 支持 API >= 30（Android 11+）。

请确保你已在设备上启用 USB 调试。

在某些设备（特别是小米）上，你可能会遇到如下错误：

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

此时，你需要启用额外选项“USB 调试（安全设置）”（与“USB 调试”为不同条目），以便通过键盘和鼠标进行控制。设置后需要重启设备。

请注意，运行 scrcpy 的OTG 模式时无需 USB 调试。

获取应用

  • Linux
  • Windows（请阅读如何运行）
  • macOS

必备技巧

  • 降低分辨率 可大幅提升性能（scrcpy -m1024）
  • 右键点击触发 BACK（返回）
  • 中键点击触发 HOME（主页）
  • Alt+f 切换全屏
  • 还有许多其他快捷键

使用示例

有许多选项，已在单独页面中文档化。以下为常用示例。

  • 以 H.265 编码（更高画质）捕获屏幕，限制分辨率为 1920，帧率上限为 60fps，禁用音频，并通过物理键盘模拟控制设备：

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # 简写

  • 在新的虚拟显示屏启动 VLC（与设备屏幕分离）：

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • 以 H.265 编码和 1920x1080 分辨率（含麦克风）录制设备相机到 MP4 文件：

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • 捕获设备前置摄像头并在电脑上作为网络摄像头（Linux）：

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • 通过物理键盘和鼠标模拟控制设备而不镜像屏幕（无需 USB 调试）：

    scrcpy --otg

  • 用连接在电脑上的手柄控制设备：

    scrcpy --gamepad=uhid
scrcpy -G  # 简写
