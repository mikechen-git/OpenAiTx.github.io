這個 GitHub 儲存庫 (https://github.com/Genymobile/scrcpy) 是本專案的唯一官方來源。請勿從隨機網站下載發佈版本，即使其名稱中包含 scrcpy。

scrcpy (v3.2)

發音為 "screen copy"

本應用程式可鏡像透過 USB 或 TCP/IP 連接的 Android 裝置（影像與音訊），並允許使用電腦的鍵盤與滑鼠進行控制。它不需要 root 權限或在裝置上安裝應用程式。支援 Linux、Windows 與 macOS。

主要特點：

  • 輕量化：原生，僅顯示裝置螢幕
  • 高效能：30~120fps，視裝置而定
  • 高畫質：1920×1080 或更高
  • 低延遲：35~70ms
  • 啟動快：~1 秒顯示第一張畫面
  • 非侵入性：Android 裝置上不會留下任何安裝檔
  • 用戶友好：無需帳號、無廣告、不需網路
  • 自由：自由且開源的軟體

其功能包含：

  • 音訊轉發（Android 11+）
  • 螢幕錄影
  • 虛擬顯示器
  • 裝置螢幕關閉下的鏡像
  • 雙向 複製貼上
  • 可調整畫質
  • 鏡像相機（Android 12+）
  • 作為網路攝影機鏡像（V4L2）（僅限 Linux）
  • 實體 鍵盤 與 滑鼠 模擬（HID）
  • 遊戲手把 支援
  • OTG 模式
  • 以及更多……

先決條件

Android 裝置至少需為 API 21（Android 5.0）。

音訊轉發 僅支援 API >= 30（Android 11+）。

請確保你已在裝置上啟用 USB 除錯。

在某些裝置（特別是小米）上，你可能會遇到以下錯誤：

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

此時，你需要啟用額外選項 USB 除錯（安全設置）（這與 USB 除錯 是不同項目），以便使用鍵盤和滑鼠控制。設定後需重啟裝置。

請注意，若使用 OTG 模式 運行 scrcpy，則不需 USB 除錯功能。

取得應用程式

  • Linux
  • Windows（請閱讀如何執行）
  • macOS

必知技巧

  • 降低解析度 可大幅提升效能（scrcpy -m1024）
  • 右鍵點擊 觸發 BACK
  • 中鍵點擊 觸發 HOME
  • Alt+f 切換 全螢幕
  • 還有許多其他 快捷鍵

使用範例

有許多選項，詳細說明於不同頁面。這裡僅列出常見範例。

  • 以 H.265（較佳畫質）擷取畫面，限制最大尺寸為 1920，最大幀率為 60fps，關閉音訊，並以實體鍵盤模擬控制裝置：

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # 簡寫版

  • 在新的虛擬顯示器（與裝置螢幕分離）中啟動 VLC：

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • 以 H.265 及 1920x1080（含麥克風）錄製裝置相機，並儲存為 MP4 檔：

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • 擷取裝置前鏡頭並將其作為電腦網路攝影機（僅限 Linux）：

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • 僅以實體鍵盤與滑鼠模擬控制裝置，不進行螢幕鏡像（不需 USB 除錯）：

    scrcpy --otg

  • 使用插入電腦的遊戲手把控制裝置：

    scrcpy --gamepad=uhid
scrcpy -G  # 簡寫版
