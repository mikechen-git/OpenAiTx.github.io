scrcpy (v3.2)

wymawiane "screen copy"

Ta aplikacja wyświetla lustrzane odbicie urządzeń z Androidem (wideo i audio) podłączonych przez USB lub TCP/IP i pozwala na kontrolę za pomocą klawiatury i myszy komputera. Nie wymaga roota ani instalowania aplikacji na urządzeniu. Działa na Linuksie, Windows i macOS.

Główne cechy:

  • lekkość: natywna, wyświetla tylko ekran urządzenia
  • wydajność: 30~120fps, w zależności od urządzenia
  • jakość: 1920×1080 lub wyżej
  • niskie opóźnienia: 35~70ms
  • krótki czas uruchamiania: ~1 sekunda do wyświetlenia pierwszego obrazu
  • nieinwazyjność: nic nie zostaje zainstalowane na urządzeniu z Androidem
  • korzyści dla użytkownika: brak konta, brak reklam, brak wymaganego Internetu
  • wolność: wolne i otwarte oprogramowanie

Funkcje obejmują:

  • przesyłanie dźwięku (Android 11+)
  • nagrywanie
  • wirtualny wyświetlacz
  • lustrzane odbicie z wyłączonym ekranem urządzenia
  • kopiowanie-wklejanie w obu kierunkach
  • konfigurowalna jakość
  • lustrzane odbicie kamery (Android 12+)
  • lustrzane odbicie jako kamera internetowa (V4L2) (tylko Linux)
  • fizyczna symulacja klawiatury i myszy (HID)
  • obsługa gamepada
  • tryb OTG
  • i więcej…

Wymagania wstępne

Urządzenie z Androidem musi mieć co najmniej API 21 (Android 5.0).

Przesyłanie dźwięku jest obsługiwane dla API >= 30 (Android 11+).

Upewnij się, że włączyłeś debugowanie USB na swoim urządzeniu.

Na niektórych urządzeniach (szczególnie Xiaomi) możesz napotkać taki błąd:

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

W takim przypadku musisz włączyć dodatkową opcję Debugowanie USB (Ustawienia zabezpieczeń) (to inna pozycja niż Debugowanie USB), aby kontrolować urządzenie za pomocą klawiatury i myszy. Po ustawieniu tej opcji konieczne jest ponowne uruchomienie urządzenia.

Zwróć uwagę, że debugowanie USB nie jest wymagane do uruchomienia scrcpy w trybie OTG.

Wskazówki, które musisz znać

  • Zmniejszenie rozdzielczości może znacznie poprawić wydajność (scrcpy -m1024)
  • Prawy przycisk myszy wywołuje BACK
  • Środkowy przycisk myszy wywołuje HOME
  • Alt+f przełącza pełny ekran
  • Istnieje wiele innych skrótów

Przykłady użycia

Istnieje wiele opcji, opisanych na osobnych stronach. Oto kilka najczęstszych przykładów.

  • Przechwyć ekran w H.265 (lepsza jakość), ogranicz rozmiar do 1920, liczbę klatek do 60fps, wyłącz audio i steruj urządzeniem przez symulację fizycznej klawiatury:

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # wersja skrócona

  • Uruchom VLC na nowym wirtualnym wyświetlaczu (osobno od wyświetlacza urządzenia):

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • Nagraj kamerę urządzenia w H.265 w 1920x1080 (oraz mikrofon) do pliku MP4:

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • Przechwyć przednią kamerę urządzenia i udostępnij ją jako kamerę internetową na komputerze (na Linuksie):

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • Steruj urządzeniem bez lustrzanego odbicia przez symulację fizycznej klawiatury i myszy (debugowanie USB nie jest wymagane):

    scrcpy --otg

  • Steruj urządzeniem za pomocą padów podłączonych do komputera:

    scrcpy --gamepad=uhid
scrcpy -G  # wersja skrócona
