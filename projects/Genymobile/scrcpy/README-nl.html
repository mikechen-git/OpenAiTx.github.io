Deze GitHub-repo (https://github.com/Genymobile/scrcpy) is de enige officiële bron voor het project. Download geen releases van willekeurige websites, zelfs niet als hun naam scrcpy bevat.

scrcpy (v3.2)

uitgesproken als "screen copy"

Deze applicatie spiegelt Android-apparaten (video en audio) die verbonden zijn via USB of TCP/IP en maakt bediening mogelijk met het toetsenbord en de muis van de computer. Het vereist geen root-toegang of een app die op het apparaat is geïnstalleerd. Het werkt op Linux, Windows en macOS.

Het richt zich op:

  • lichtgewicht: native, toont alleen het apparaatscherm
  • prestaties: 30~120fps, afhankelijk van het apparaat
  • kwaliteit: 1920×1080 of hoger
  • lage latency: 35~70ms
  • korte opstarttijd: ~1 seconde om de eerste afbeelding te tonen
  • niet-invasief: er blijft niets geïnstalleerd achter op het Android-apparaat
  • gebruikersvoordelen: geen account, geen advertenties, geen internet nodig
  • vrijheid: vrije en open source software

De functies omvatten:

  • audio-doorsturen (Android 11+)
  • opnemen
  • virtueel scherm
  • spiegelen met Android-scherm uit
  • kopiëren-plakken in beide richtingen
  • instelbare kwaliteit
  • camera-spiegeling (Android 12+)
  • spiegelen als webcam (V4L2) (alleen Linux)
  • fysieke toetsenbord en muis simulatie (HID)
  • gamepad ondersteuning
  • OTG-modus
  • en meer…

Vereisten

Het Android-apparaat vereist minimaal API 21 (Android 5.0).

Audio-doorsturen wordt ondersteund vanaf API >= 30 (Android 11+).

Zorg ervoor dat je USB-debugging hebt ingeschakeld op je apparaat/apparaten.

Op sommige apparaten (vooral Xiaomi) kun je de volgende foutmelding krijgen:

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

In dat geval moet je een extra optie inschakelen: USB-debugging (Beveiligingsinstellingen) (dit is een ander item dan USB-debugging) om het te kunnen bedienen met een toetsenbord en muis. Het apparaat moet opnieuw worden opgestart nadat deze optie is ingesteld.

Let op: USB-debugging is niet vereist om scrcpy in de OTG-modus uit te voeren.

Belangrijke tips

  • Resolutie verlagen kan de prestaties aanzienlijk verbeteren (scrcpy -m1024)
  • Rechtermuisknop activeert BACK
  • Middelste muisknop activeert HOME
  • Alt+f schakelt volledig scherm in/uit
  • Er zijn veel andere sneltoetsen

Voorbeelden van gebruik

Er zijn veel opties, gedocumenteerd op aparte pagina's. Hier zijn enkele veelvoorkomende voorbeelden.

  • Leg het scherm vast in H.265 (betere kwaliteit), beperk de grootte tot 1920, beperk de framerate tot 60fps, schakel audio uit en bedien het apparaat door een fysiek toetsenbord te simuleren:

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # korte versie

  • Start VLC in een nieuw virtueel scherm (gescheiden van het apparaatscherm):

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • Neem de apparaatcamera op in H.265 op 1920x1080 (en microfoon) naar een MP4-bestand:

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • Leg de frontcamera van het apparaat vast en stel deze beschikbaar als webcam op de computer (op Linux):

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • Bedien het apparaat zonder spiegeling door een fysiek toetsenbord en muis te simuleren (USB-debugging niet vereist):

    scrcpy --otg

  • Bedien het apparaat met gamepad-controllers die op de computer zijn aangesloten:

    scrcpy --gamepad=uhid
scrcpy -G  # korte versie
