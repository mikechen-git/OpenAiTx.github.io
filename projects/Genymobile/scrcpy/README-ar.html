هذا المستودع على GitHub (https://github.com/Genymobile/scrcpy) هو المصدر الرسمي الوحيد للمشروع. لا تقم بتنزيل الإصدارات من مواقع عشوائية، حتى وإن كان اسمها يحتوي على scrcpy.

scrcpy (الإصدار 3.2)

تنطق "screen copy"

يعمل هذا التطبيق على عكس أجهزة أندرويد (الفيديو والصوت) المتصلة عبر USB أو TCP/IP، ويسمح بالتحكم باستخدام لوحة مفاتيح وفأرة الكمبيوتر. لا يتطلب صلاحية root أو تثبيت تطبيق على الجهاز. يعمل على أنظمة Linux و_Windows_ و_macOS_.

يركز على:

  • الخفة: أصلي، يعرض فقط شاشة الجهاز
  • الأداء: 30~120 إطار في الثانية، حسب الجهاز
  • الجودة: 1920×1080 أو أكثر
  • زمن استجابة منخفض: 35~70 مللي ثانية
  • زمن بدء تشغيل منخفض: ~1 ثانية لعرض أول صورة
  • عدم التدخل: لا يُترك أي شيء مثبت على جهاز الأندرويد
  • فائدة المستخدم: لا حساب، لا إعلانات، لا يتطلب الإنترنت
  • الحرية: برنامج مجاني ومفتوح المصدر

تشمل ميزاته:

  • تمرير الصوت (أندرويد 11+)
  • التسجيل
  • شاشة افتراضية
  • العكس مع إيقاف شاشة جهاز الأندرويد
  • النسخ واللصق في كلا الاتجاهين
  • جودة قابلة للتكوين
  • عكس الكاميرا (أندرويد 12+)
  • العكس ككاميرا ويب (V4L2) (لينكس فقط)
  • محاكاة لوحة المفاتيح والفأرة الفعلية (HID)
  • دعم يد التحكم بالألعاب
  • وضع OTG
  • والمزيد...

المتطلبات الأساسية

يتطلب جهاز الأندرويد إصدار API 21 على الأقل (أندرويد 5.0).

تمرير الصوت مدعوم للإصدار API >= 30 (أندرويد 11+).

تأكد من أنك فعّلت تصحيح أخطاء USB على جهازك.

على بعض الأجهزة (خاصة Xiaomi)، قد تظهر الرسالة التالية:

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

في هذه الحالة، تحتاج إلى تفعيل خيار إضافي وهو تصحيح أخطاء USB (إعدادات الأمان) (هذا خيار مختلف عن تصحيح أخطاء USB) للتحكم فيه باستخدام لوحة المفاتيح والفأرة. يجب إعادة تشغيل الجهاز بعد تفعيل هذا الخيار.

لاحظ أن تصحيح أخطاء USB غير مطلوب لتشغيل scrcpy في وضع OTG.

الحصول على التطبيق

  • لينكس
  • ويندوز (اقرأ كيفية التشغيل)
  • ماك

نصائح يجب معرفتها

  • تقليل الدقة قد يحسن الأداء بشكل كبير (scrcpy -m1024)
  • النقر بزر الفأرة الأيمن ينفذ أمر BACK
  • النقر بزر الفأرة الأوسط ينفذ أمر HOME
  • Alt+f يبدل وضع ملء الشاشة
  • هناك العديد من اختصارات لوحة المفاتيح الأخرى

أمثلة للاستخدام

هناك العديد من الخيارات، موثقة في صفحات منفصلة. فيما يلي بعض الأمثلة الشائعة فقط.

  • التقاط الشاشة بصيغة H.265 (جودة أفضل)، تحديد الحجم إلى 1920، تحديد معدل الإطارات إلى 60 إطار بالثانية، تعطيل الصوت، والتحكم بالجهاز بمحاكاة لوحة مفاتيح فعلية:

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # نسخة مختصرة

  • تشغيل VLC في شاشة افتراضية جديدة (منفصلة عن شاشة الجهاز):

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • تسجيل كاميرا الجهاز بصيغة H.265 بدقة 1920x1080 (والميكروفون) إلى ملف MP4:

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • التقاط الكاميرا الأمامية للجهاز وعرضها ككاميرا ويب على الكمبيوتر (على لينكس):

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • التحكم بالجهاز بدون عكس الشاشة بمحاكاة لوحة مفاتيح وفأرة فعلية (لا يتطلب تصحيح أخطاء USB):

    scrcpy --otg

  • التحكم بالجهاز باستخدام وحدات التحكم بالألعاب المتصلة بالكمبيوتر:

    scrcpy --gamepad=uhid
scrcpy -G  # نسخة مختصرة
