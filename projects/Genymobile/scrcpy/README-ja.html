このGitHubリポジトリ（https://github.com/Genymobile/scrcpy）が本プロジェクトの唯一の公式ソースです。
scrcpyという名前が含まれていても、無作為なウェブサイトからリリースをダウンロードしないでください。

scrcpy (v3.2)

発音は "screen copy"（スクリーン・コピー）

このアプリケーションは、USBまたはTCP/IP経由で接続されたAndroidデバイス（ビデオとオーディオ）をミラーリングし、コンピューターのキーボードとマウスで操作できるようにします。_root_権限やデバイスへのアプリのインストールは不要です。Linux、Windows、_macOS_で動作します。

主な特徴は次の通りです:

  • 軽量: ネイティブで、デバイス画面のみを表示
  • 高パフォーマンス: デバイスにより30～120fps
  • 高品質: 1920×1080以上
  • 低遅延: 35～70ms
  • 高速起動: 最初の画像表示まで約1秒
  • 非侵襲性: Androidデバイスに何もインストールされません
  • ユーザー利益: アカウント不要、広告なし、インターネット不要
  • 自由: 無料かつオープンソースソフトウェア

主な機能には次が含まれます:

  • オーディオ転送（Android 11以降）
  • 録画
  • 仮想ディスプレイ
  • Androidデバイスの画面オフ状態でのミラーリング
  • 双方向のコピーペースト
  • 画質の設定可能
  • カメラミラーリング（Android 12以降）
  • Webカメラ（V4L2）としてのミラーリング（Linux限定）
  • 物理キーボードおよびマウスのシミュレーション（HID）
  • ゲームパッド対応
  • OTGモード
  • その他多数…

前提条件

AndroidデバイスはAPI 21（Android 5.0）以上が必要です。

オーディオ転送はAPI 30（Android 11以降）でサポートされます。

デバイスでUSBデバッグを有効にしてください。

一部のデバイス（特にXiaomi）では、次のエラーが表示されることがあります:

java.lang.SecurityException: Injecting input events requires the caller (or the source of the instrumentation, if any) to have the INJECT_EVENTS permission.

この場合、追加オプション「USBデバッグ（セキュリティ設定）」
（USBデバッグとは異なる項目です）を有効にして、キーボードやマウスで操作できるようにしてください。
このオプション設定後は、デバイスの再起動が必要です。

なお、OTGモードでscrcpyを実行する場合、USBデバッグは不要です。

アプリの入手

  • Linux
  • Windows（実行方法を参照）
  • macOS

必須のヒント

  • 解像度を下げることでパフォーマンスが大幅に向上する場合があります
    （scrcpy -m1024）
  • 右クリックでBACKをトリガー
  • ミドルクリックでHOMEをトリガー
  • Alt+fで全画面表示の切り替え
  • その他にも多くのショートカットがあります

使用例

多くのオプションがあり、個別ページで説明されています。 ここでは一般的な例を紹介します。

  • H.265で画面をキャプチャし（高画質）、サイズを1920に制限し、フレームレートを60fpsに制限、オーディオを無効にし、物理キーボードをシミュレートしてデバイスを操作:

    scrcpy --video-codec=h265 --max-size=1920 --max-fps=60 --no-audio --keyboard=uhid
scrcpy --video-codec=h265 -m1920 --max-fps=60 --no-audio -K  # ショートバージョン

  • 新しい仮想ディスプレイ（デバイスディスプレイとは別）でVLCを起動:

    scrcpy --new-display=1920x1080 --start-app=org.videolan.vlc

  • デバイスのカメラをH.265で1920x1080（およびマイク）でMP4ファイルに録画:

    scrcpy --video-source=camera --video-codec=h265 --camera-size=1920x1080 --record=file.mp4

  • デバイスのフロントカメラをキャプチャし、コンピューター上でWebカメラとして公開（Linuxの場合）:

    scrcpy --video-source=camera --camera-size=1920x1080 --camera-facing=front --v4l2-sink=/dev/video2 --no-playback

  • ミラーリングせず、物理キーボードおよびマウスをシミュレートしてデバイスを操作（USBデバッグ不要）:

    scrcpy --otg

  • コンピューターに接続したゲームパッドでデバイスを操作:

    scrcpy --gamepad=uhid
scrcpy -G  # ショートバージョン
