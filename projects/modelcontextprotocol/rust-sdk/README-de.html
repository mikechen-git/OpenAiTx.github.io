RMCP

Warten auf die erste Veröffentlichung.

Eine offizielle Rust Model Context Protocol SDK-Implementierung mit Tokio-Async-Runtime.

Dieses Repository enthält die folgenden Crates:

  • rmcp: Das Kern-Crate, das die RMCP-Protokollimplementierung bereitstellt (Wenn Sie mehr Informationen erhalten möchten, besuchen Sie bitte rmcp)
  • rmcp-macros: Ein Prozedur-Makro-Crate zur Generierung von RMCP-Tool-Implementierungen (Wenn Sie mehr Informationen erhalten möchten, besuchen Sie bitte rmcp-macros)

Verwendung

Importieren des Crates

rmcp = { version = "0.1", features = ["server"] }
## oder Dev-Channel
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Drittabhängigkeiten

Grundlegende Abhängigkeiten:

  • tokio erforderlich
  • serde erforderlich

Client erstellen

Client starten 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Server erstellen

Transport aufbauen 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Service aufbauen

Sie können ganz einfach einen Service mit ServerHandler oder ClientHandler erstellen.

let service = common::counter::Counter::new();
Server starten 
// Dieser Aufruf schließt den Initialisierungsprozess ab
let server = service.serve(transport).await?;
Mit dem Server interagieren

Sobald der Server initialisiert ist, können Sie Anfragen oder Benachrichtigungen senden:

// Anfrage
let roots = server.list_roots().await?;

// oder Benachrichtigung senden
server.notify_cancelled(...).await?;
Warten auf das Herunterfahren des Dienstes 
let quit_reason = server.waiting().await?;
// oder abbrechen
let quit_reason = server.cancel().await?;
