RMCP

Espere el primer lanzamiento.

Una implementación oficial del SDK de Model Context Protocol en Rust con runtime asíncrono tokio.

Este repositorio contiene los siguientes crates:

  • rmcp: El crate principal que proporciona la implementación del protocolo RMCP (Si desea obtener más información, visite rmcp)
  • rmcp-macros: Un crate de macros procedurales para generar implementaciones de herramientas RMCP (Si desea obtener más información, visite rmcp-macros)

Uso

Importar el crate

rmcp = { version = "0.1", features = ["server"] }
## o canal de desarrollo
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Dependencias de terceros

Dependencias básicas:

  • tokio requerido
  • serde requerido

Construir un Cliente

Iniciar un cliente 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Construir un Servidor

Construir un transporte 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Construir un servicio

Puede construir fácilmente un servicio usando ServerHandler o ClientHandler.

let service = common::counter::Counter::new();
Iniciar el servidor 
// esta llamada completará el proceso de inicialización
let server = service.serve(transport).await?;
Interactuar con el servidor

Una vez que el servidor está inicializado, puede enviar solicitudes o notificaciones:

// solicitud
let roots = server.list_roots().await?;

// o enviar una notificación
server.notify_cancelled(...).await?;
Esperar el apagado del servicio 
let quit_reason = server.waiting().await?;
// o cancelarlo
let quit_reason = server.cancel().await?;
