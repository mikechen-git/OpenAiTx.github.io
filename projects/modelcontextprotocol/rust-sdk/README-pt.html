RMCP

Aguarde o primeiro lançamento.

Uma implementação oficial do SDK Model Context Protocol em Rust com runtime assíncrono tokio.

Este repositório contém os seguintes crates:

  • rmcp: O crate principal que fornece a implementação do protocolo RMCP (Se quiser mais informações, visite rmcp)
  • rmcp-macros: Um crate de macros procedurais para geração de implementações de ferramentas RMCP (Se quiser mais informações, visite rmcp-macros)

Uso

Importe o crate

rmcp = { version = "0.1", features = ["server"] }
## ou canal de desenvolvimento
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Dependências Terceiras

Dependências básicas:

  • tokio obrigatório
  • serde obrigatório

Construir um Cliente

Iniciar um cliente 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Construir um Servidor

Construir um transporte 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Construir um serviço

Você pode facilmente construir um serviço utilizando ServerHandler ou ClientHandler.

let service = common::counter::Counter::new();
Iniciar o servidor 
// esta chamada finalizará o processo de inicialização
let server = service.serve(transport).await?;
Interagir com o servidor

Uma vez que o servidor esteja inicializado, você pode enviar requisições ou notificações:

// requisição
let roots = server.list_roots().await?;

// ou enviar notificação
server.notify_cancelled(...).await?;
Aguardando o encerramento do serviço 
let quit_reason = server.waiting().await?;
// ou cancelar
let quit_reason = server.cancel().await?;
