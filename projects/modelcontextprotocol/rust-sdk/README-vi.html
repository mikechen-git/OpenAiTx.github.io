RMCP

Chờ phát hành phiên bản đầu tiên.

Một SDK chính thức của Model Context Protocol cho Rust với runtime bất đồng bộ tokio.

Kho lưu trữ này bao gồm các crate sau:

  • rmcp: Crate lõi cung cấp triển khai giao thức RMCP (Nếu bạn muốn biết thêm thông tin, vui lòng truy cập rmcp)
  • rmcp-macros: Crate macro thủ tục dùng để sinh các triển khai công cụ RMCP (Nếu bạn muốn biết thêm thông tin, vui lòng truy cập rmcp-macros)

Sử dụng

Thêm crate

rmcp = { version = "0.1", features = ["server"] }
## hoặc kênh dev
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Phụ thuộc bên thứ ba

Các phụ thuộc cơ bản:

  • tokio yêu cầu
  • serde yêu cầu

Xây dựng Client

Khởi động một client 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Xây dựng Server

Tạo một transport 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Xây dựng một service

Bạn có thể dễ dàng xây dựng một service bằng cách sử dụng ServerHandler hoặc ClientHandler.

let service = common::counter::Counter::new();
Khởi động server 
// lệnh này sẽ hoàn tất quá trình khởi tạo
let server = service.serve(transport).await?;
Tương tác với server

Sau khi server đã khởi tạo, bạn có thể gửi các yêu cầu hoặc thông báo:

// request
let roots = server.list_roots().await?;

// hoặc gửi notification
server.notify_cancelled(...).await?;
Chờ server tắt 
let quit_reason = server.waiting().await?;
// hoặc hủy bỏ
let quit_reason = server.cancel().await?;
