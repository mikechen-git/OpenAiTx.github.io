RMCP

Poczekaj na pierwsze wydanie.

Oficjalna implementacja Model Context Protocol SDK w języku Rust z wykorzystaniem asynchronicznego środowiska uruchomieniowego tokio.

To repozytorium zawiera następujące craty:

  • rmcp: Główny crate dostarczający implementację protokołu RMCP (Aby uzyskać więcej informacji, odwiedź rmcp)
  • rmcp-macros: Crate proceduralny do generowania implementacji narzędzi RMCP (Aby uzyskać więcej informacji, odwiedź rmcp-macros)

Użycie

Importuj crate

rmcp = { version = "0.1", features = ["server"] }
## lub kanał deweloperski
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Zewnętrzne zależności

Podstawowe zależności:

  • wymagany tokio
  • wymagany serde

Budowanie klienta

Uruchom klienta 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Budowanie serwera

Budowanie transportu 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Budowanie usługi

Możesz łatwo zbudować usługę używając ServerHandler lub ClientHandler.

let service = common::counter::Counter::new();
Uruchom serwer 
// to wywołanie zakończy proces inicjalizacji
let server = service.serve(transport).await?;
Interakcja z serwerem

Po zainicjowaniu serwera możesz wysyłać żądania lub powiadomienia:

// żądanie
let roots = server.list_roots().await?;

// lub wyślij powiadomienie
server.notify_cancelled(...).await?;
Oczekiwanie na zamknięcie usługi 
let quit_reason = server.waiting().await?;
// lub anuluj
let quit_reason = server.cancel().await?;
