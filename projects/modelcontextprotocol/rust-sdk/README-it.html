RMCP

Attendere la prima release.

Un'implementazione ufficiale dell'SDK Model Context Protocol in Rust con runtime asincrono tokio.

Questo repository contiene i seguenti crate:

  • rmcp: Il crate principale che fornisce l'implementazione del protocollo RMCP (per ulteriori informazioni, visita rmcp)
  • rmcp-macros: Un crate di macro procedurali per generare implementazioni di tool RMCP (per ulteriori informazioni, visita rmcp-macros)

Utilizzo

Importa il crate

rmcp = { version = "0.1", features = ["server"] }
## oppure canale dev
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Dipendenze di terze parti

Dipendenze di base:

  • tokio richiesto
  • serde richiesto

Costruire un Client

Avvia un client 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Costruire un Server

Costruire un transport 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Costruire un servizio

Puoi facilmente costruire un servizio utilizzando ServerHandler o ClientHandler.

let service = common::counter::Counter::new();
Avvia il server 
// questa chiamata completerà il processo di inizializzazione
let server = service.serve(transport).await?;
Interagire con il server

Una volta che il server è stato inizializzato, puoi inviare richieste o notifiche:

// richiesta
let roots = server.list_roots().await?;

// oppure invia una notifica
server.notify_cancelled(...).await?;
Attendere l'arresto del servizio 
let quit_reason = server.waiting().await?;
// oppure annullalo
let quit_reason = server.cancel().await?;
