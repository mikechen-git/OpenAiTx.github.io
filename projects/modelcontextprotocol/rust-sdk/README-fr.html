RMCP

En attente de la première version.

Une implémentation officielle du SDK Rust Model Context Protocol avec l'environnement d'exécution asynchrone tokio.

Ce dépôt contient les crates suivantes :

  • rmcp : La crate principale fournissant l’implémentation du protocole RMCP (Pour plus d’informations, veuillez consulter rmcp)
  • rmcp-macros : Une crate de macros procédurales pour générer des outils RMCP (Pour plus d’informations, veuillez consulter rmcp-macros)

Utilisation

Importer la crate

rmcp = { version = "0.1", features = ["server"] }
## ou canal dev
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

Dépendances tierces

Dépendances de base :

  • tokio requis
  • serde requis

Construire un client

Démarrer un client 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

Construire un serveur

Construire un transport 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
Construire un service

Vous pouvez facilement construire un service en utilisant ServerHandler ou ClientHandler.

let service = common::counter::Counter::new();
Démarrer le serveur 
// cet appel termine le processus d’initialisation
let server = service.serve(transport).await?;
Interagir avec le serveur

Une fois le serveur initialisé, vous pouvez envoyer des requêtes ou des notifications :

// requête
let roots = server.list_roots().await?;

// ou envoyer une notification
server.notify_cancelled(...).await?;
Attendre l’arrêt du service 
let quit_reason = server.waiting().await?;
// ou l'annuler
let quit_reason = server.cancel().await?;
