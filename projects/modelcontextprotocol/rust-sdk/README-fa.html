RMCP

منتظر اولین انتشار باشید.

یک پیاده‌سازی رسمی SDK پروتکل Model Context به زبان rust با اجرای async توسط tokio.

این مخزن شامل کرِیت‌های زیر است:

  • rmcp: کرِیت اصلی که پیاده‌سازی پروتکل RMCP را فراهم می‌کند (برای اطلاعات بیشتر به rmcp مراجعه کنید)
  • rmcp-macros: یک کرِیت macro رویه‌ای برای تولید پیاده‌سازی ابزارهای RMCP (برای اطلاعات بیشتر به rmcp-macros مراجعه کنید)

نحوه استفاده

وارد کردن کرِیت

rmcp = { version = "0.1", features = ["server"] }
## یا شاخه توسعه
rmcp = { git = "https://github.com/modelcontextprotocol/rust-sdk", branch = "main" }

وابستگی‌های ثالث

وابستگی‌های پایه:

  • tokio مورد نیاز
  • serde مورد نیاز

ساخت کلاینت

راه‌اندازی یک کلاینت 
use rmcp::{ServiceExt, transport::{TokioChildProcess, ConfigureCommandExt}};
use tokio::process::Command;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = ().serve(TokioChildProcess::new(Command::new("npx").configure(|cmd| {
        cmd.arg("-y").arg("@modelcontextprotocol/server-everything");
    }))?).await?;
    Ok(())
}

ساخت سرور

ساخت ترنسپورت 
use tokio::io::{stdin, stdout};
let transport = (stdin(), stdout());
ساخت سرویس

شما می‌توانید به راحتی یک سرویس با استفاده از ServerHandler یا ClientHandler بسازید.

let service = common::counter::Counter::new();
راه‌اندازی سرور 
// این فراخوانی فرایند مقداردهی اولیه را تکمیل می‌کند
let server = service.serve(transport).await?;
تعامل با سرور

پس از مقداردهی اولیه سرور، می‌توانید درخواست‌ها یا اعلان‌ها را ارسال کنید:

// درخواست
let roots = server.list_roots().await?;

// یا ارسال اعلان
server.notify_cancelled(...).await?;
منتظر خاموش شدن سرویس باشید 
let quit_reason = server.waiting().await?;
// یا آن را لغو کنید
let quit_reason = server.cancel().await?;
