Uw Ster en Donatie kunnen MiniExcel beter maken

Introductie

MiniExcel is eenvoudig en efficiënt om OOM's te voorkomen als .NET Excel-verwerkingstool.

De meeste populaire frameworks laden momenteel alle data in het geheugen om bewerkingen te vergemakkelijken, maar dit veroorzaakt problemen met geheugengebruik. MiniExcel probeert algoritmen te gebruiken op basis van streams om het oorspronkelijke geheugengebruik van 1000 MB te reduceren naar enkele MB's en zo OOM (out of memory) te voorkomen.

Functies

  • Laag geheugengebruik, voorkom OOM (out of memory) en volledige GC
  • Ondersteunt realtime bewerking van elke rij data
  • Ondersteunt LINQ deferred execution, kan met laag verbruik snel pagineren en andere complexe queries uitvoeren
  • Lichtgewicht, geen Microsoft Office nodig, geen COM+, DLL-grootte is minder dan 400KB
  • Eenvoudige API-stijl om Excel te lezen/schrijven/invullen

Aan de slag

  • Excel Importeren/Opvragen

  • Excel Exporteren/Aanmaken

  • Excel Sjabloon

  • Excel Kolomnaam/Index/Ignore Attribuut

  • Voorbeelden

12. Grote bestanden lezen met schijf-gebaseerde cache (Disk-Base Cache - SharedString)

Als de grootte van SharedStrings groter is dan 5 MB, gebruikt MiniExcel standaard lokale schijfcache, bijv. 10x100000.xlsx (een miljoen rijen data), wanneer schijfcache is uitgeschakeld is het maximale geheugengebruik 195MB, maar met schijfcache is slechts 65MB nodig. Let op, deze optimalisatie kost wat efficiëntie, dus in dit geval neemt de leestijd toe van 7,4 seconden naar 27,2 seconden. Als je het niet nodig hebt kun je schijfcache uitschakelen met onderstaande code:

var config = new OpenXmlConfiguration { EnableSharedStringCache = false };
MiniExcel.Query(path,configuration: config)

Je kunt SharedStringCacheSize gebruiken om de sharedString-bestandsgrootte te wijzigen waarna schijfcaching gebruikt wordt

var config = new OpenXmlConfiguration { SharedStringCacheSize=500*1024*1024 };
MiniExcel.Query(path, configuration: config);

public IActionResult DownloadExcel()
{
    var values = new[] {
        new { Column1 = "MiniExcel", Column2 = 1 },
        new { Column1 = "Github", Column2 = 2}
    };

    var memoryStream = new MemoryStream();
    memoryStream.SaveAs(values);
    memoryStream.Seek(0, SeekOrigin.Begin);
    return new FileStreamResult(memoryStream, "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet")
    {
        FileDownloadName = "demo.xlsx"
    };
}

private IEnumerable<Dictionary<string, object>> GetOrders(string lang, string role, Order[] orders)
{
    foreach (var order in orders)
    {
        var newOrder = new Dictionary<string, object>();

        if (lang == "zh-CN")
        {
            newOrder.Add("客户编号", order.CustomerID);
            newOrder.Add("订单编号", order.OrderNo);
            newOrder.Add("产品编号", order.ProductID);
            newOrder.Add("数量", order.Qty);
            if (role == "Sales")
                newOrder.Add("价格", order.Amt);
            yield return newOrder;
        }
        else if (lang == "en-US")
        {
            newOrder.Add("Customer ID", order.CustomerID);
            newOrder.Add("Order No", order.OrderNo);
            newOrder.Add("Product ID", order.ProductID);
            newOrder.Add("Quantity", order.Qty);
            if (role == "Sales")
                newOrder.Add("Amount", order.Amt);
            yield return newOrder;
        }
        else
        {
            throw new InvalidDataException($"lang {lang} wrong");
        }
    }
}

#### V. Hoe gebruikt Query gehele getal-indexen?

De standaardindex van Query is de string Key: A,B,C.... Als je wilt wijzigen naar een numerieke index, maak dan de volgende methode aan om te converteren

```csharp
void Main()
{
    var path = @"D:\git\MiniExcel\samples\xlsx\TestTypeMapping.xlsx";
    var rows = MiniExcel.Query(path,true);
    foreach (var r in ConvertToIntIndexRows(rows))

private IEnumerable<Dictionary<int, object>> ConvertToIntIndexRows(IEnumerable<object> rows)
{
    ICollection<string> keys = null;
    var isFirst = true;
    foreach (IDictionary<string,object> r in rows)
    {
        if(isFirst)
        {
            keys = r.Keys;
            isFirst = false;
        }

        var dic = new Dictionary<int, object>();
        var index = 0;
        foreach (var key in keys)
            dic[index++] = r[key];
        yield return dic;
    }
}

            {
                var column = new DataColumn(key, typeof(object)) { Caption = key };
                dt.Columns.Add(column);
            }

            dt.BeginLoadData();
            first = false;
        }

        var newRow = dt.NewRow();
        var isNull=true;
        foreach (var key in row.Keys)
        {
            var _v = row[key];
            if(_v!=null)
                isNull = false;
            newRow[key] = _v;
        }

        if(!isNull)
            dt.Rows.Add(newRow);
    }

    dt.EndLoadData();
    return dt;
}

V. Hoe SaveAs(path,value) gebruiken om een bestaand bestand te vervangen zonder de foutmelding "The file ...xlsx already exists error"

Gebruik de Stream-klasse om aangepaste logica voor het aanmaken van bestanden te implementeren, bijvoorbeeld:

    using (var stream = File.Create("Demo.xlsx"))
        MiniExcel.SaveAs(stream,value);

of, sinds V1.25.0, ondersteunt SaveAs de overwriteFile-parameter voor het inschakelen/uitschakelen van het overschrijven van bestaande bestanden

    MiniExcel.SaveAs(path, value, overwriteFile: true);
