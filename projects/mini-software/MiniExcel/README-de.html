Einführung

MiniExcel ist ein einfaches und effizientes Excel-Verarbeitungstool für .NET, das speziell darauf ausgelegt ist, den Speicherverbrauch zu minimieren.

Derzeit müssen die meisten gängigen Frameworks alle Daten aus einem Excel-Dokument in den Speicher laden, um Operationen zu ermöglichen, was zu Problemen mit dem Speicherverbrauch führen kann. MiniExcels Ansatz ist anders: Die Daten werden zeilenweise im Streaming-Verfahren verarbeitet, wodurch der ursprüngliche Verbrauch von potenziell mehreren hundert Megabyte auf nur wenige Megabyte reduziert wird. Dadurch werden Out-of-Memory(OOM)-Probleme effektiv verhindert.

Funktionen

  • Minimiert den Speicherverbrauch, verhindert Out-of-Memory-(OOM)-Fehler und vermeidet vollständige Garbage Collections
  • Ermöglicht Echtzeit-Datenoperationen auf Zeilenebene für bessere Leistung bei großen Datensätzen
  • Unterstützt LINQ mit verzögerter Ausführung und ermöglicht schnelles, speichereffizientes Paging und komplexe Abfragen
  • Leichtgewichtig, benötigt weder Microsoft Office noch COM+-Komponenten, und eine DLL-Größe unter 500KB
  • Einfacher und intuitiver API-Stil zum Lesen/Schreiben/Ausfüllen von Excel

12. Große Datei per Festplatten-Cache lesen (Disk-Base Cache - SharedString)

Wenn die Größe von SharedStrings 5 MB überschreitet, verwendet MiniExcel standardmäßig einen lokalen Festplatten-Cache, z.B. 10x100000.xlsx (eine Million Zeilen Daten). Wenn der Festplatten-Cache deaktiviert ist, beträgt die maximale Speichernutzung 195 MB, mit aktiviertem Cache werden nur 65 MB benötigt. Beachten Sie, dass diese Optimierung etwas Effizienz kostet, sodass sich die Lesezeit in diesem Fall von 7,4 Sekunden auf 27,2 Sekunden erhöht. Falls Sie das nicht benötigen, können Sie den Festplatten-Cache mit folgendem Code deaktivieren:

var config = new OpenXmlConfiguration { EnableSharedStringCache = false };
MiniExcel.Query(path,configuration: config)

Sie können SharedStringCacheSize verwenden, um die Dateigröße von sharedString festzulegen, ab der Festplatten-Caching verwendet wird.

var config = new OpenXmlConfiguration { SharedStringCacheSize=500*1024*1024 };
MiniExcel.Query(path, configuration: config);

public IActionResult DownloadExcel()
{
    var values = new[] {
        new { Column1 = "MiniExcel", Column2 = 1 },
        new { Column1 = "Github", Column2 = 2}
    };

    var memoryStream = new MemoryStream();
    memoryStream.SaveAs(values);
    memoryStream.Seek(0, SeekOrigin.Begin);
    return new FileStreamResult(memoryStream, "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet")
    {
        FileDownloadName = "demo.xlsx"
    };
}

#### 8. If/ElseIf/Else-Anweisungen in der Zelle

Regeln:
1. Unterstützt DateTime, Double, Int mit den Operatoren ==, !=, >, >=, <, <=.
2. Unterstützt String mit den Operatoren ==, !=.
3. Jede Anweisung sollte in einer neuen Zeile stehen.
4. Vor und nach den Operatoren sollte jeweils ein Leerzeichen stehen.
5. Es sollte keinen Zeilenumbruch innerhalb der Anweisungen geben.
6. Die Zelle muss exakt wie im folgenden Format aufgebaut sein.

5. Dynamisches i18n Mehrsprachigkeit und Rollenberechtigungsmanagement

Wie im Beispiel, erstellen Sie eine Methode zur Behandlung von i18n und Berechtigungsmanagement, und verwenden Sie yield return, um IEnumerable<Dictionary<string, object>> zurückzugeben, um dynamische und speicherschonende Verarbeitung zu erreichen.

void Main()
{
    var value = new Order[] {
        new Order(){OrderNo = "SO01",CustomerID="C001",ProductID="P001",Qty=100,Amt=500},
        new Order(){OrderNo = "SO02",CustomerID="C002",ProductID="P002",Qty=300,Amt=400},
    };

    Console.WriteLine("en-Us und Rolle Sales");
    {
        var path = Path.GetTempPath() + Guid.NewGuid() + ".xlsx";
        var lang = "en-US";
        var role = "Sales";
        MiniExcel.SaveAs(path, GetOrders(lang, role, value));

        MiniExcel.Query(path, true).Dump();
    }

    Console.WriteLine("zh-CN und PMC-Rolle");
    {
        var path = Path.GetTempPath() + Guid.NewGuid() + ".xlsx";
        var lang = "zh-CN";
        var role = "PMC";
        MiniExcel.SaveAs(path, GetOrders(lang, role, value));
        MiniExcel.Query(path, true).Dump();
    }
}

private IEnumerable<Dictionary<string, object>> GetOrders(string lang, string role, Order[] orders)
{
    foreach (var order in orders)
    {
        var newOrder = new Dictionary<string, object>();

        if (lang == "zh-CN")
        {
            newOrder.Add("客户编号", order.CustomerID);
            newOrder.Add("订单编号", order.OrderNo);
            newOrder.Add("产品编号", order.ProductID);
            newOrder.Add("数量", order.Qty);
            if (role == "Sales")
                newOrder.Add("价格", order.Amt);
            yield return newOrder;
        }
        else if (lang == "en-US")
        {
            newOrder.Add("Customer ID", order.CustomerID);
            newOrder.Add("Order No", order.OrderNo);
            newOrder.Add("Product ID", order.ProductID);
            newOrder.Add("Quantity", order.Qty);
            if (role == "Sales")
                newOrder.Add("Amount", order.Amt);
            yield return newOrder;
        }
        else
        {
            throw new InvalidDataException($"lang {lang} wrong");
        }
    }
}

#### F. Wie verwendet Query ganzzahlige Indizes?

Der Standardindex von Query ist der Zeichenfolgen-Key: A,B,C.... Wenn Sie auf numerische Indizes umstellen möchten, erstellen Sie bitte die folgende Methode zur Umwandlung:

```csharp
void Main()
{
    var path = @"D:\git\MiniExcel\samples\xlsx\TestTypeMapping.xlsx";
    var rows = MiniExcel.Query(path,true);
    foreach (var r in ConvertToIntIndexRows(rows))
    {
        Console.Write($"column 0 : {r[0]} ,column 1 : {r[1]}");
        Console.WriteLine();
    }
}

private IEnumerable<Dictionary<int, object>> ConvertToIntIndexRows(IEnumerable<object> rows)
{
    ICollection<string> keys = null;
    var isFirst = true;
    foreach (IDictionary<string,object> r in rows)
    {
        if(isFirst)
        {
            keys = r.Keys;
            isFirst = false;
        }

        var dic = new Dictionary<int, object>();
        var index = 0;
        foreach (var key in keys)
            dic[index++] = r[key];
        yield return dic;
    }
}

F. Es wird eine leere Excel-Datei ohne Titel generiert, wenn beim Exportieren von Excel der Wert leer ist

Da MiniExcel eine Logik ähnlich wie JSON.NET verwendet, um den Typ dynamisch aus den Werten zu ermitteln, um API-Operationen zu vereinfachen, kann der Typ ohne Daten nicht bestimmt werden. Siehe Issue #133 zum Verständnis.

Starker Typ & DataTable erzeugen Überschriften, aber Dictionary bleibt eine leere Excel-Datei

F. Wie kann man foreach beenden, wenn eine leere Zeile erscheint?

MiniExcel kann mit LINQ TakeWhile verwendet werden, um den foreach-Iterator zu stoppen.

F. Wie entfernt man leere Zeilen?

IEnumerable :

public static IEnumerable<dynamic> QueryWithoutEmptyRow(Stream stream, bool useHeaderRow, string sheetName, ExcelType excelType, string startCell, IConfiguration configuration)
{
    var rows = stream.Query(useHeaderRow,sheetName,excelType,startCell,configuration);
    foreach (IDictionary<string,object> row in rows)
    {
        if(row.Keys.Any(key=>row[key]!=null))
            yield return row;
    }
}

DataTable :

public static DataTable QueryAsDataTableWithoutEmptyRow(Stream stream, bool useHeaderRow, string sheetName, ExcelType excelType, string startCell, IConfiguration configuration)
{
    if (sheetName == null && excelType != ExcelType.CSV) /*Issue #279*/
        sheetName = stream.GetSheetNames().First();

    var dt = new DataTable(sheetName);
    var first = true;
    var rows = stream.Query(useHeaderRow,sheetName,excelType,startCell,configuration);
    foreach (IDictionary<string, object> row in rows)
    {
        if (first)
        {

            foreach (var key in row.Keys)
            {
                var column = new DataColumn(key, typeof(object)) { Caption = key };

                dt.Columns.Add(column);
            }

            dt.BeginLoadData();
            first = false;
        }

        var newRow = dt.NewRow();
        var isNull=true;
        foreach (var key in row.Keys)
        {
            var _v = row[key];
            if(_v!=null)
                isNull = false;
            newRow[key] = _v;
        }

        if(!isNull)
            dt.Rows.Add(newRow);
    }

    dt.EndLoadData();
    return dt;
}

F. Wie kann SaveAs(path, value) eine existierende Datei ersetzen, ohne den Fehler „Die Datei ...xlsx existiert bereits“ auszulösen?

Bitte verwenden Sie die Stream-Klasse, um eine eigene Logik zur Dateierstellung zu implementieren, z.B.:

    using (var stream = File.Create("Demo.xlsx"))
        MiniExcel.SaveAs(stream, value);

Oder, seit V1.25.0, unterstützt SaveAs den Parameter overwriteFile, um das Überschreiben einer existierenden Datei zu aktivieren/deaktivieren

    MiniExcel.SaveAs(path, value, overwriteFile: true);

  • xlsm unterstützt nur Query
