Khởi động nhanh

Có nhiều lựa chọn khởi động nhanh:

  • Tải bản phát hành mới nhất
  • Clone repo: git clone https://github.com/twbs/bootstrap.git
  • Cài đặt với npm: npm install bootstrap@v5.3.6
  • Cài đặt với yarn: yarn add bootstrap@v5.3.6
  • Cài đặt với Bun: bun add bootstrap@v5.3.6
  • Cài đặt với Composer: composer require twbs/bootstrap:5.3.6
  • Cài đặt với NuGet: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

Đọc Trang bắt đầu để biết thông tin về nội dung framework, mẫu, ví dụ và nhiều hơn nữa.

Source map (bootstrap.*.map) có sẵn để sử dụng với các công cụ dành cho nhà phát triển trên một số trình duyệt nhất định. Các tệp JS gói sẵn (bootstrap.bundle.js và bản rút gọn bootstrap.bundle.min.js) bao gồm cả Popper.

Báo lỗi và yêu cầu tính năng

Bạn gặp lỗi hoặc có yêu cầu tính năng? Vui lòng đọc trước hướng dẫn về issue và tìm kiếm các vấn đề đã tồn tại hoặc đã đóng. Nếu vấn đề hoặc ý tưởng của bạn chưa được đề cập, hãy tạo issue mới.

Tài liệu

Tài liệu của Bootstrap, được bao gồm trong repo này tại thư mục gốc, được xây dựng bằng Astro và được công khai trên GitHub Pages tại https://getbootstrap.com/. Tài liệu cũng có thể chạy cục bộ.

Chức năng tìm kiếm tài liệu được cung cấp bởi Algolia's DocSearch.

Chạy tài liệu cục bộ

  1. Chạy npm install để cài các phụ thuộc Node.js, bao gồm Astro (trình tạo trang).
  2. Chạy npm run test (hoặc script npm cụ thể) để biên dịch lại các tệp CSS và JavaScript phân phối, cũng như tài sản tài liệu.
  3. Từ thư mục gốc /bootstrap, chạy npm run docs-serve trong dòng lệnh.
  4. Mở http://localhost:9001/ trên trình duyệt, và xong.

Tìm hiểu thêm về cách sử dụng Astro bằng cách đọc tài liệu của nó.

Tài liệu cho các bản phát hành trước

Bạn có thể tìm tất cả tài liệu các bản phát hành trước tại https://getbootstrap.com/docs/versions/.

Các bản phát hành trước và tài liệu của chúng cũng có sẵn để tải về.

Đóng góp

Vui lòng đọc kỹ hướng dẫn đóng góp của chúng tôi. Trong đó bao gồm hướng dẫn mở issue, tiêu chuẩn mã hóa, và ghi chú phát triển.

Ngoài ra, nếu pull request của bạn chứa bản vá hoặc tính năng JavaScript, bạn phải đính kèm unit test liên quan. Tất cả HTML và CSS cần tuân thủ Code Guide, được duy trì bởi Mark Otto.

Tùy chỉnh trình soạn thảo có tại editor config để dễ dàng sử dụng trong các trình soạn thảo văn bản phổ biến. Đọc thêm và tải plugin tại https://editorconfig.org/.

Cộng đồng

Nhận cập nhật về quá trình phát triển của Bootstrap và trò chuyện với các quản trị viên dự án cũng như các thành viên cộng đồng.

  • Theo dõi @getbootstrap trên X.
  • Đọc và đăng ký Blog chính thức của Bootstrap.
  • Đặt câu hỏi và khám phá GitHub Discussions của chúng tôi.
  • Thảo luận, đặt câu hỏi và hơn thế nữa trên Discord cộng đồng hoặc Bootstrap subreddit.
  • Trò chuyện với các Bootstrappers khác trên IRC. Trên máy chủ irc.libera.chat, kênh #bootstrap.
  • Hỗ trợ triển khai có thể tìm thấy tại Stack Overflow (tag bootstrap-5).
  • Các nhà phát triển nên sử dụng từ khóa bootstrap trên các package mở rộng hoặc bổ sung chức năng cho Bootstrap khi phân phối qua npm hoặc các phương thức phân phối tương tự để tối đa hóa khả năng tìm kiếm.

Quản lý phiên bản

Để minh bạch về chu kỳ phát hành và duy trì khả năng tương thích ngược, Bootstrap được duy trì theo hướng dẫn Semantic Versioning. Đôi khi chúng tôi mắc lỗi, nhưng luôn cố gắng tuân thủ các quy tắc đó bất cứ khi nào có thể.

Xem phần Releases của dự án GitHub để xem nhật ký thay đổi của từng phiên bản Bootstrap. Các bài đăng thông báo phát hành trên blog chính thức của Bootstrap chứa tóm tắt các thay đổi nổi bật nhất trong mỗi bản phát hành.
