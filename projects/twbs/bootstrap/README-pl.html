Szybki start

Dostępnych jest kilka opcji szybkiego startu:

  • Pobierz najnowszą wersję
  • Sklonuj repozytorium: git clone https://github.com/twbs/bootstrap.git
  • Zainstaluj przez npm: npm install bootstrap@v5.3.6
  • Zainstaluj przez yarn: yarn add bootstrap@v5.3.6
  • Zainstaluj przez Bun: bun add bootstrap@v5.3.6
  • Zainstaluj przez Composer: composer require twbs/bootstrap:5.3.6
  • Zainstaluj przez NuGet: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

Przeczytaj stronę Pierwsze kroki, aby uzyskać informacje o zawartości frameworka, szablonach, przykładach i nie tylko.

Błędy i propozycje funkcji

Masz błąd lub propozycję funkcji? Najpierw przeczytaj wytyczne zgłaszania problemów i wyszukaj istniejące lub zamknięte zgłoszenia. Jeśli Twój problem lub pomysł nie został jeszcze poruszony, otwórz nowe zgłoszenie.

Dokumentacja

Dokumentacja Bootstrap, zawarta w tym repozytorium w katalogu głównym, jest budowana za pomocą Astro i publicznie hostowana na GitHub Pages pod adresem https://getbootstrap.com/. Dokumentację można również uruchomić lokalnie.

Wyszukiwanie w dokumentacji jest wspierane przez DocSearch firmy Algolia.

Uruchamianie dokumentacji lokalnie

  1. Uruchom npm install, aby zainstalować zależności Node.js, w tym Astro (generator strony).
  2. Uruchom npm run test (lub konkretny skrypt npm), aby odbudować dystrybuowane pliki CSS i JavaScript oraz zasoby dokumentacji.
  3. Z katalogu głównego /bootstrap uruchom w terminalu npm run docs-serve.
  4. Otwórz http://localhost:9001/ w przeglądarce i gotowe.

Dowiedz się więcej o korzystaniu z Astro, czytając jego dokumentację.

Dokumentacja dla poprzednich wersji

Wszystkie poprzednie wersje dokumentacji znajdziesz na https://getbootstrap.com/docs/versions/.

Poprzednie wydania oraz ich dokumentacja są również dostępne do pobrania.

Współtworzenie

Przeczytaj nasze wytyczne dla współtwórców. Znajdziesz tam instrukcje dotyczące zgłaszania problemów, standardów kodowania oraz uwagi dotyczące rozwoju.

Jeśli Twój pull request zawiera poprawki lub funkcje w JavaScript, musisz dołączyć odpowiednie testy jednostkowe. Cały kod HTML i CSS powinien być zgodny z Code Guide, utrzymywanym przez Marka Otto.

Preferencje edytora są dostępne w pliku editor config do łatwego wykorzystania w popularnych edytorach tekstowych. Przeczytaj więcej i pobierz wtyczki na https://editorconfig.org/.

Społeczność

Otrzymuj aktualizacje dotyczące rozwoju Bootstrap oraz rozmawiaj z opiekunami projektu i członkami społeczności.

  • Obserwuj @getbootstrap na X.
  • Czytaj i subskrybuj Oficjalny Blog Bootstrap.
  • Zadawaj pytania i przeglądaj nasze dyskusje GitHub.
  • Dyskutuj, zadawaj pytania i więcej na społecznościowym Discordzie lub subreddicie Bootstrap.
  • Rozmawiaj z innymi użytkownikami Bootstrap na IRC. Na serwerze irc.libera.chat, w kanale #bootstrap.
  • Pomoc w implementacji znajdziesz na Stack Overflow (tag bootstrap-5).
  • Programiści powinni używać słowa kluczowego bootstrap w paczkach, które modyfikują lub rozszerzają funkcjonalność Bootstrap przy dystrybucji przez npm lub podobne mechanizmy, aby zapewnić maksymalną wykrywalność.

Wersjonowanie

W trosce o przejrzystość cyklu wydawniczego oraz dążenie do zachowania kompatybilności wstecznej, Bootstrap jest utrzymywany zgodnie z wytycznymi Semantic Versioning. Czasem popełniamy błędy, ale stosujemy się do tych zasad, kiedy tylko to możliwe.

Zobacz sekcję Releases w naszym projekcie GitHub po changelogi dla każdej wersji Bootstrap. Wpisy ogłaszające wydania na oficjalnym blogu Bootstrap zawierają podsumowania najważniejszych zmian w każdej wersji.
