เริ่มต้นอย่างรวดเร็ว

มีตัวเลือกในการเริ่มต้นอย่างรวดเร็วหลายวิธี:

  • ดาวน์โหลดเวอร์ชันล่าสุด
  • โคลน repo: git clone https://github.com/twbs/bootstrap.git
  • ติดตั้งด้วย npm: npm install bootstrap@v5.3.6
  • ติดตั้งด้วย yarn: yarn add bootstrap@v5.3.6
  • ติดตั้งด้วย Bun: bun add bootstrap@v5.3.6
  • ติดตั้งด้วย Composer: composer require twbs/bootstrap:5.3.6
  • ติดตั้งด้วย NuGet: CSS: Install-Package bootstrap Sass: Install-Package bootstrap.sass

อ่าน หน้าการเริ่มต้นใช้งาน เพื่อดูข้อมูลเกี่ยวกับเนื้อหาเฟรมเวิร์ก เทมเพลต ตัวอย่าง และอื่น ๆ

รายงานบั๊กและขอฟีเจอร์

มีบั๊กหรือขอฟีเจอร์ใหม่? กรุณาอ่าน แนวทางการรายงานปัญหา และค้นหาปัญหาที่มีอยู่หรือปิดแล้วก่อน หากปัญหาหรือไอเดียของคุณยังไม่มี กรุณาเปิดประเด็นใหม่

เอกสาร

เอกสารของ Bootstrap ซึ่งรวมอยู่ใน repo นี้ในไดเรกทอรีหลัก ถูกสร้างด้วย Astro และเผยแพร่สาธารณะที่ GitHub Pages ที่ https://getbootstrap.com/ เอกสารสามารถรันแบบโลคัลได้เช่นกัน

ระบบค้นหาเอกสารขับเคลื่อนโดย DocSearch ของ Algolia

การรันเอกสารแบบโลคัล

  1. รัน npm install เพื่อติดตั้ง dependencies ของ Node.js รวมถึง Astro (ตัวสร้างเว็บไซต์)
  2. รัน npm run test (หรือ npm script อื่น) เพื่อสร้าง CSS และ JavaScript ที่แจกจ่ายใหม่ รวมถึงทรัพยากรของเอกสาร
  3. จากไดเรกทอรี /bootstrap หลัก รัน npm run docs-serve ใน command line
  4. เปิด http://localhost:9001/ ในเบราว์เซอร์ของคุณ

เรียนรู้เพิ่มเติมเกี่ยวกับการใช้งาน Astro ได้โดยอ่าน เอกสารของ Astro

เอกสารสำหรับเวอร์ชันก่อนหน้า

คุณสามารถหาเอกสารของเวอร์ชันก่อนหน้าทั้งหมดได้ที่ https://getbootstrap.com/docs/versions/

เวอร์ชันก่อนหน้า และเอกสารของแต่ละเวอร์ชันก็พร้อมดาวน์โหลด

การมีส่วนร่วม

กรุณาอ่าน แนวทางการมีส่วนร่วม ของเรา มีคำแนะนำเกี่ยวกับการเปิดประเด็น มาตรฐานการเขียนโค้ด และหมายเหตุเกี่ยวกับการพัฒนา

นอกจากนี้ หาก pull request ของคุณมีการแก้ไขหรือเพิ่มคุณสมบัติ JavaScript คุณต้องแนบ unit test ที่เกี่ยวข้อง ด้วย HTML และ CSS ทั้งหมดควรสอดคล้องกับ Code Guide ที่ดูแลโดย Mark Otto

การตั้งค่าของ editor มีอยู่ใน editor config เพื่อให้ง่ายต่อการใช้กับ editor ทั่วไป อ่านเพิ่มเติมและดาวน์โหลดปลั๊กอินได้ที่ https://editorconfig.org/

ชุมชน

รับข้อมูลอัปเดตเกี่ยวกับการพัฒนา Bootstrap และพูดคุยกับผู้ดูแลโปรเจกต์และสมาชิกชุมชน

  • ติดตาม @getbootstrap บน X
  • อ่านและติดตาม บล็อกทางการของ Bootstrap
  • ถามคำถามและสำรวจ GitHub Discussions ของเรา
  • พูดคุย ถามคำถาม และอื่น ๆ บน Discord ของชุมชน หรือ Bootstrap subreddit
  • สนทนากับชาว Bootstrappers ใน IRC ที่เซิร์ฟเวอร์ irc.libera.chat ในแชนแนล #bootstrap
  • ขอความช่วยเหลือเรื่องการใช้งานได้ที่ Stack Overflow (ติดแท็ก bootstrap-5)
  • นักพัฒนาควรใช้คีย์เวิร์ด bootstrap กับแพ็กเกจที่ปรับปรุงหรือเพิ่มฟังก์ชันการทำงานของ Bootstrap เมื่อเผยแพร่ผ่าน npm หรือเครื่องมือคล้ายกัน เพื่อให้ค้นหาได้ง่าย

การจัดการเวอร์ชัน

เพื่อความโปร่งใสในรอบการปล่อยเวอร์ชันและรักษาความเข้ากันได้ย้อนหลัง Bootstrap อยู่ภายใต้ แนวทาง Semantic Versioning บางครั้งเราอาจทำผิดพลาด แต่เราพยายามยึดตามกฎเหล่านั้นทุกครั้งที่ทำได้

ดู ส่วน Releases ของโปรเจกต์บน GitHub เพื่อดู changelog ของแต่ละเวอร์ชัน ประกาศออกเวอร์ชันใหม่ ๆ จะสรุปการเปลี่ยนแปลงสำคัญไว้ใน บล็อกทางการของ Bootstrap
