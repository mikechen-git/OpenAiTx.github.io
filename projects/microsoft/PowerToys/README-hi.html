परिचय

Microsoft PowerToys पावर यूज़र्स के लिए विंडोज़ अनुभव को ट्यून और स्ट्रीमलाइन करने के लिए यूटिलिटीज़ का एक सेट है, जिससे उत्पादकता बढ़ती है। PowerToys ओवरव्यू और यूटिलिटीज़ का उपयोग कैसे करें या Windows विकास परिवेश के लिए किसी अन्य टूल या संसाधन के लिए, learn.microsoft.com पर जाएं!

Microsoft PowerToys को इंस्टॉल और चलाना

आवश्यकताएँ

  • Windows 11 या Windows 10 संस्करण 2004 (कोड नाम 20H1 / बिल्ड नंबर 19041) या नया।
  • x64 या ARM64 प्रोसेसर
  • हमारा इंस्टॉलर निम्नलिखित आइटम इंस्टॉल करेगा:
    • Microsoft Edge WebView2 Runtime बूटस्ट्रैपर। यह नवीनतम संस्करण इंस्टॉल करेगा।

Microsoft Store के माध्यम से

Microsoft Store के PowerToys पेज से इंस्टॉल करें। आपको नया Microsoft Store उपयोग करना होगा, जो Windows 11 और Windows 10 दोनों के लिए उपलब्ध है।

WinGet के माध्यम से

WinGet से PowerToys डाउनलोड करें। WinGet के माध्यम से PowerToys को अपडेट करने से वर्तमान इंस्टॉलेशन स्कोप बरकरार रहेगा। PowerToys इंस्टॉल करने के लिए, कमांड लाइन / PowerShell से निम्नलिखित कमांड चलाएँ:

यूज़र स्कोप इंस्टॉलर [डिफ़ॉल्ट]

winget install Microsoft.PowerToys -s winget

मशीन-वाइड स्कोप इंस्टॉलर

winget install --scope machine Microsoft.PowerToys -s winget

अन्य इंस्टॉल विधियाँ

कम्युनिटी द्वारा संचालित इंस्टॉल विधियाँ जैसे Chocolatey और Scoop उपलब्ध हैं। यदि ये आपकी पसंदीदा इंस्टॉल सॉल्यूशन हैं, तो आप वहाँ इंस्टॉल निर्देश पा सकते हैं।

थर्ड-पार्टी रन प्लगइन्स

कम्युनिटी द्वारा बनाए गए थर्ड-पार्टी प्लगइन्स का एक संग्रह है, जो PowerToys के साथ वितरित नहीं होते हैं।

योगदान

यह प्रोजेक्ट सभी प्रकार के योगदान का स्वागत करता है। कोडिंग फीचर्स / बग फिक्स के अलावा, अन्य मदद के तरीके हैं: स्पेक लेखन, डिजाइन, दस्तावेजीकरण, और बग ढूंढना। हम पावर यूज़र कम्युनिटी के साथ मिलकर ऐसे टूल्स का सेट बनाना चाहते हैं, जो विंडोज़ का अधिकतम लाभ उठाने में आपकी मदद करें।

हम अनुरोध करते हैं कि किसी फीचर पर काम शुरू करने से पहले जिसे आप योगदान देना चाहते हैं, कृपया हमारे Contributor's Guide को पढ़ें। हम आपके साथ मिलकर सर्वोत्तम दृष्टिकोण तय करने, मार्गदर्शन और मेंटॉरशिप देने, और अनावश्यक या डुप्लिकेट प्रयास से बचने के लिए तैयार हैं।

अधिकांश योगदान के लिए आपको Contributor License Agreement (CLA) से सहमत होना आवश्यक है, जिसमें आप हमें अपने योगदान के उपयोग का अधिकार प्रदान करते हैं और आपके पास ऐसा करने की अनुमति है।

PowerToys के लिए डेवलपमेंट संबंधी मार्गदर्शन के लिए, कृपया डेवलपर डॉक्युमेंटेशन पढ़ें, जिसमें सेटअप और कंपाइलिंग का विस्तृत विवरण है।

क्या हो रहा है

PowerToys रोडमैप

हमारी प्राथमिकता वाली रोडमैप जिसमें वे फीचर्स और यूटिलिटीज़ हैं, जिन पर कोर टीम ध्यान केंद्रित कर रही है।

0.91 - मई 2025 अपडेट

इस रिलीज़ में हमने नई सुविधाओं, स्थिरता और ऑटोमेशन पर ध्यान केंद्रित किया।

✨ मुख्य बिंदु

  • हमने कमांड पैलेट के प्रदर्शन को बहुत बेहतर किया और कई बग्स को ठीक किया। कुछ नई सुविधाएँ जो जोड़ी गईं:
  • कमांड पैलेट में किसी भी फाइल को सर्च करने की क्षमता जोड़ी गई।
  • कमांड पैलेट के ग्लोबल हॉटकी को लो-लेवल कीबोर्ड हुक बनाने की क्षमता जोड़ी गई।
  • वेब सर्च एक्सटेंशन के लिए ओपन URL फॉलबैक कमांड जोड़ा गया, जिससे उपयोगकर्ता सीधे ब्राउज़र में URL खोल सकते हैं।
  • अब आप PT Run और Command Palette के Date and Time प्लगइन्स में कस्टम फॉर्मेट परिभाषित कर सकते हैं। धन्यवाद @htcfreek!

एडवांस्ड पेस्ट

  • एक समस्या को ठीक किया गया जहाँ एडवांस्ड पेस्ट कुछ इंग्लिश लैंग्वेज टैग्स (जैसे en-CA) के लिए OCR इंजन नहीं बना पा रहा था, अब यह यूज़र प्रोफाइल भाषा से OCR इंजन इनिशियलाइज़ करता है। धन्यवाद @cryolithic!

कलर पिकर

  • एक समस्या को ठीक किया गया जहाँ रिसोर्स लीक के कारण हैंग या क्रैश हो रहा था, अब ग्राफिक्स ऑब्जेक्ट को सही तरीके से डिस्पोज़ किया गया। धन्यवाद @dcog989!
  • एक समस्या को ठीक किया गया जहाँ बैकस्पेस की दबाने पर कलर पिकर बंद हो जाता था, अब केवल फोकस होने पर ही बंद होगा और Escape/Backspace व्यवहार को संरेखित किया गया। धन्यवाद @PesBandi!
  • कलर पिकर में Oklab और Oklch कलर फॉर्मेट के लिए सपोर्ट जोड़ा गया। धन्यवाद @lemonyte!

कमांड नॉट फाउंड

  • WinGet Command Not Found स्क्रिप्ट को अपडेट किया गया ताकि केवल उन्हीं एक्सपेरिमेंटल फीचर्स को सक्षम किया जाए जो वास्तव में मौजूद हैं।

  • बग टेम्पलेट को अपडेट किया गया ताकि कमांड पैलेट मॉड्यूल शामिल हो।
  • एक समस्या को ठीक किया गया जहाँ टोस्ट विंडो DPI के अनुसार स्केल नहीं होती थी, जिससे डिस्प्ले स्केलिंग के तहत लेआउट समस्याएँ होती थीं।
  • एक समस्या को ठीक किया गया जहाँ Up/Down कीबोर्ड नेविगेशन सेलेक्शन को नहीं बदलता था जब कैरेट पोजीशन 0 पर था, और PT Run v1 जैसी लगातार नेविगेशन जोड़ी गई। धन्यवाद @davidegiacometti!
