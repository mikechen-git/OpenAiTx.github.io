protobuf-net.Grpc

protobuf-net.Grpc voegt code-first ondersteuning toe voor services over gRPC met gebruik van ofwel de native Grpc.Core API, of de volledig beheerde Grpc.Net.Client / Grpc.AspNetCore.Server API.

Het zou moeten werken op alle .NET-talen die iets kunnen genereren dat ook maar enigszins lijkt op een regulier .NET-type model.

  • Aan de slag
  • Alle documentatie
  • Bouwen/gebruiken beschikbaar via protobuf-net.BuildTools

Het gebruik is net zo eenvoudig als het declareren van een interface voor je servicecontract:

[ServiceContract]
public interface IMyAmazingService {
    ValueTask<SearchResponse> SearchAsync(SearchRequest request);
    // ...
}

daarna kun je deze interface implementeren voor een server:

public class MyServer : IMyAmazingService {
    // ...
}

of het systeem vragen om een client:

var client = http.CreateGrpcService<IMyAmazingService>();
var results = await client.SearchAsync(request);

Dit zou gelijkwaardig zijn aan de service in .proto:

service MyAmazingService {
    rpc Search (SearchRequest) returns (SearchResponse) {}
	// ...
}

Uiteraard moet je de uri enz. opgeven - zie Aan de slag. Meestal is de configuratie conventie-gebaseerd, maar als je dat liever hebt: er zijn verschillende configuratieopties.

Hoe kom je eraan

Alles is beschikbaar als vooraf gebouwde pakketten op nuget; in het bijzonder wil je waarschijnlijk een van de volgende:

  • protobuf-net.Grpc.AspNetCore voor servers die ASP.NET Core 3.1 gebruiken
  • protobuf-net.Grpc.Native voor clients of servers die de native/unmanaged API gebruiken
  • protobuf-net.Grpc en Grpc.Net.Client voor clients die HttpClient gebruiken op .NET Core 3.1

Voorbeelden van gebruik zijn beschikbaar in C#, VB en F#.

Nog iets anders?

protobuf-net.Grpc is gemaakt en wordt onderhouden door Marc Gravell (@marcgravell), de auteur van protobuf-net.

Het maakt gebruik van tools van grpc, maar is niet officieel geassocieerd met, verbonden aan of goedgekeurd door dat project.
