protobuf-net.Grpc

protobuf-net.Grpc เพิ่มการรองรับแบบ code-first สำหรับบริการผ่าน gRPC โดยใช้ทั้ง Grpc.Core API แบบ native หรือ Grpc.Net.Client / Grpc.AspNetCore.Server API ที่เป็น fully-managed

สามารถทำงานได้กับภาษา .NET ทุกภาษา ที่สามารถสร้าง type model ของ .NET ได้ แม้จะคล้ายคลึงเพียงเล็กน้อยก็ตาม

  • เริ่มต้นใช้งาน
  • เอกสารทั้งหมด
  • การ build/ใช้งานผ่าน protobuf-net.BuildTools

การใช้งานง่ายเพียงแค่ประกาศ interface สำหรับ service-contract ของคุณ:

[ServiceContract]
public interface IMyAmazingService {
    ValueTask<SearchResponse> SearchAsync(SearchRequest request);
    // ...
}

จากนั้นสามารถ implement interface นี้สำหรับ server ได้:

public class MyServer : IMyAmazingService {
    // ...
}

หรือขอ client จากระบบ:

var client = http.CreateGrpcService<IMyAmazingService>();
var results = await client.SearchAsync(request);

ซึ่งจะเทียบเท่ากับบริการใน .proto:

service MyAmazingService {
    rpc Search (SearchRequest) returns (SearchResponse) {}
	// ...
}

แน่นอนว่าคุณต้องระบุ uri เป็นต้น - ดูได้ที่ เริ่มต้นใช้งาน โดยปกติการตั้งค่าจะใช้ตาม convention แต่ หากคุณต้องการ: มี ตัวเลือกการตั้งค่าหลากหลาย

การติดตั้ง

ทุกอย่างมีให้ในรูปแบบแพ็คเกจสำเร็จรูปบน nuget; โดยเฉพาะ คุณอาจต้องการตัวใดตัวหนึ่งต่อไปนี้:

  • protobuf-net.Grpc.AspNetCore สำหรับเซิร์ฟเวอร์ที่ใช้ ASP.NET Core 3.1
  • protobuf-net.Grpc.Native สำหรับ client หรือ server ที่ใช้ native/unmanaged API
  • protobuf-net.Grpc และ Grpc.Net.Client สำหรับ client ที่ใช้ HttpClient บน .NET Core 3.1

ตัวอย่างการใช้งานมีให้ในภาษา C#, VB และ F#

มีอะไรอีกไหม?

protobuf-net.Grpc ถูกสร้างและดูแลโดย Marc Gravell (@marcgravell) ผู้เขียน protobuf-net

มีการใช้เครื่องมือจาก grpc, แต่ไม่ได้เป็นโครงการอย่างเป็นทางการ, ไม่ได้มีความเกี่ยวข้องหรือได้รับการรับรองโดยโครงการนั้น

ผมรอคอยความคิดเห็นจากคุณ และหากสิ่งนี้ช่วยประหยัดเวลาให้คุณได้มาก คุณยินดีที่จะ
