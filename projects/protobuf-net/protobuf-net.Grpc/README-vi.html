protobuf-net.Grpc

protobuf-net.Grpc bổ sung hỗ trợ code-first cho các dịch vụ trên gRPC sử dụng API gốc Grpc.Core, hoặc API hoàn toàn managed Grpc.Net.Client / Grpc.AspNetCore.Server.

Nó có thể hoạt động trên tất cả các ngôn ngữ .NET có thể sinh ra thứ gì đó tương tự như một mô hình kiểu .NET thông thường.

  • Bắt đầu sử dụng
  • Tất cả tài liệu
  • Build/sử dụng qua protobuf-net.BuildTools

Cách sử dụng đơn giản như khai báo một interface cho hợp đồng dịch vụ của bạn:

[ServiceContract]
public interface IMyAmazingService {
    ValueTask<SearchResponse> SearchAsync(SearchRequest request);
    // ...
}

sau đó hoặc hiện thực interface đó cho server:

public class MyServer : IMyAmazingService {
    // ...
}

hoặc yêu cầu hệ thống tạo client:

var client = http.CreateGrpcService<IMyAmazingService>();
var results = await client.SearchAsync(request);

Điều này tương đương với dịch vụ trong .proto:

service MyAmazingService {
    rpc Search (SearchRequest) returns (SearchResponse) {}
	// ...
}

Rõ ràng bạn cần chỉ định uri, v.v. - xem Bắt đầu sử dụng. Thông thường cấu hình sẽ dựa trên quy ước, nhưng nếu bạn muốn: có nhiều tùy chọn cấu hình.

Làm thế nào để lấy nó

Tất cả đều có sẵn dưới dạng các gói pre-built trên nuget; cụ thể, bạn có thể sẽ muốn một trong số:

  • protobuf-net.Grpc.AspNetCore cho server sử dụng ASP.NET Core 3.1
  • protobuf-net.Grpc.Native cho client hoặc server sử dụng API gốc/unmanaged
  • protobuf-net.Grpc và Grpc.Net.Client cho client sử dụng HttpClient trên .NET Core 3.1

Ví dụ sử dụng có sẵn bằng C#, VB và F#.

Còn gì nữa không?

protobuf-net.Grpc được tạo và duy trì bởi Marc Gravell (@marcgravell), tác giả của protobuf-net.

Nó sử dụng các công cụ từ grpc, nhưng không chính thức liên kết, liên quan hay được xác nhận bởi dự án đó.

Tôi rất mong nhận được phản hồi của bạn, và nếu công cụ này giúp bạn tiết kiệm được rất nhiều thời gian, bạn luôn được chào đón
