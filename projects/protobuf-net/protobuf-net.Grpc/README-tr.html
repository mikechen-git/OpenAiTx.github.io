protobuf-net.Grpc

protobuf-net.Grpc, ister yerel Grpc.Core API'sini, ister tamamen yönetilen Grpc.Net.Client / Grpc.AspNetCore.Server API'sini kullanarak gRPC üzerinden hizmetler için kod-öncelikli destek ekler.

Düzenli bir .NET tür modeli gibi bir şey üretebilen tüm .NET dilleriyle çalışmalıdır.

  • Başlarken
  • Tüm Dokümantasyon
  • Yapı/kullanım protobuf-net.BuildTools ile erişilebilir

Kullanımı, hizmet sözleşmeniz için bir arayüz tanımlamak kadar basittir:

[ServiceContract]
public interface IMyAmazingService {
    ValueTask<SearchResponse> SearchAsync(SearchRequest request);
    // ...
}

sonra ya bu arayüzü bir sunucu için uygularsınız:

public class MyServer : IMyAmazingService {
    // ...
}

ya da istemci için sistemden bir örnek istersiniz:

var client = http.CreateGrpcService<IMyAmazingService>();
var results = await client.SearchAsync(request);

Bu, .proto'daki servise eşdeğer olurdu:

service MyAmazingService {
    rpc Search (SearchRequest) returns (SearchResponse) {}
	// ...
}

Elbette, ona uri vb. bilgileri vermeniz gerekir - bkz. Başlarken. Genellikle yapılandırma geleneklere dayalıdır, ancak isterseniz: çeşitli yapılandırma seçenekleri mevcuttur.

Nasıl edinebilirim?

Her şey nuget'te hazır paketler olarak mevcuttur; özellikle, muhtemelen aşağıdakilerden birini isteyeceksiniz:

  • ASP.NET Core 3.1 kullanan sunucular için protobuf-net.Grpc.AspNetCore
  • Yerel/yönetilmeyen API kullanan istemci veya sunucular için protobuf-net.Grpc.Native
  • .NET Core 3.1 üzerinde HttpClient kullanan istemciler için protobuf-net.Grpc ve Grpc.Net.Client

Kullanım örnekleri C#, VB ve F# dillerinde mevcuttur.

Başka bir şey var mı?

protobuf-net.Grpc, protobuf-net yazarı Marc Gravell (@marcgravell) tarafından oluşturulmuş ve bakımına devam edilmektedir.

grpc projesinden araçlar kullanır, ancak bu proje ile resmî olarak ilişkili, bağlantılı veya onaylı değildir.

Geri bildiriminizi bekliyorum ve eğer bu size çok zaman kazandıracaksa, her zaman ile destek olabilirsiniz.
