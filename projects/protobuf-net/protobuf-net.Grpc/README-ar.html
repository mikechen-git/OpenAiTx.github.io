protobuf-net.Grpc

protobuf-net.Grpc يضيف دعم تصميم الكود أولاً للخدمات عبر gRPC باستخدام إما واجهة برمجة التطبيقات الأصلية Grpc.Core، أو واجهة برمجة التطبيقات المدارة بالكامل Grpc.Net.Client / Grpc.AspNetCore.Server.

ينبغي أن يعمل على جميع لغات .NET التي يمكنها توليد شيء حتى وإن كان شبيهاً بنموذج نوع .NET عادي.

  • البدء السريع
  • كل الوثائق
  • البناء/الاستخدام متوفر عبر protobuf-net.BuildTools

الاستخدام ببساطة يكون عبر إعلان واجهة لعقد الخدمة الخاصة بك:

[ServiceContract]
public interface IMyAmazingService {
    ValueTask<SearchResponse> SearchAsync(SearchRequest request);
    // ...
}

ثم إما تنفيذ تلك الواجهة للخادم:

public class MyServer : IMyAmazingService {
    // ...
}

أو الطلب من النظام إنشاء عميل:

var client = http.CreateGrpcService<IMyAmazingService>();
var results = await client.SearchAsync(request);

هذا سيكون مكافئاً للخدمة في ملف .proto:

service MyAmazingService {
    rpc Search (SearchRequest) returns (SearchResponse) {}
	// ...
}

من الواضح أنك بحاجة لإعلامه بعنوان uri وما إلى ذلك - راجع البدء السريع. عادةً ما يكون التكوين قائماً على القواعد، ولكن إذا كنت تفضل: هناك خيارات تكوين متعددة.

كيفية الحصول عليه

كل شيء متوفر كحزم جاهزة البناء على nuget؛ على وجه الخصوص، ربما ترغب في أحد الخيارات التالية:

  • protobuf-net.Grpc.AspNetCore للخوادم التي تستخدم ASP.NET Core 3.1
  • protobuf-net.Grpc.Native للعملاء أو الخوادم التي تستخدم الواجهة البرمجية الأصلية/غير المدارة
  • protobuf-net.Grpc و Grpc.Net.Client للعملاء الذين يستخدمون HttpClient على .NET Core 3.1

أمثلة الاستخدام متوفرة بـ C# وVB وF#.

أي شيء آخر؟

protobuf-net.Grpc تم إنشاؤه وصيانته بواسطة Marc Gravell (@marcgravell)، مؤلف protobuf-net.

يستخدم أدوات من grpc، لكنه ليس مرتبطاً رسمياً أو معتمداً أو موصى به من قبل هذا المشروع.

أتطلع إلى ملاحظاتكم، وإذا كان هذا بإمكانه توفير الكثير من الوقت لكم، فأنتم دائماً مرحب بكم
