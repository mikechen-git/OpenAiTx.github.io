protobuf-net.Grpc

protobuf-net.Grpc gRPC पर सेवाओं के लिए कोड-फर्स्ट सपोर्ट जोड़ता है, चाहे वह नेटिव Grpc.Core API हो या पूरी तरह से मैनेज्ड Grpc.Net.Client / Grpc.AspNetCore.Server API।

यह उन सभी .NET भाषाओं पर काम करना चाहिए जो कुछ हद तक सामान्य .NET टाइप मॉडल उत्पन्न कर सकती हैं।

  • शुरुआत करें
  • संपूर्ण प्रलेखन
  • निर्माण/उपयोग protobuf-net.BuildTools के माध्यम से उपलब्ध

इस्तेमाल करना उतना ही सरल है जितना कि अपनी सेवा-संविदा के लिए एक इंटरफ़ेस घोषित करना:

[ServiceContract]
public interface IMyAmazingService {
    ValueTask<SearchResponse> SearchAsync(SearchRequest request);
    // ...
}

फिर या तो उस इंटरफ़ेस को सर्वर के लिए लागू करें:

public class MyServer : IMyAmazingService {
    // ...
}

या सिस्टम से क्लाइंट के लिए पूछें:

var client = http.CreateGrpcService<IMyAmazingService>();
var results = await client.SearchAsync(request);

यह .proto में सेवा के समकक्ष होगा:

service MyAmazingService {
    rpc Search (SearchRequest) returns (SearchResponse) {}
	// ...
}

स्पष्ट रूप से आपको इसे uri आदि बताना होगा - देखें शुरुआत करें। आमतौर पर कॉन्फ़िगरेशन कन्वेंशन-आधारित होती है, लेकिन अगर आप चाहें: विभिन्न कॉन्फ़िगरेशन विकल्प उपलब्ध हैं।

इसे प्राप्त करना

सभी चीजें nuget पर प्री-बिल्ट पैकेज के रूप में उपलब्ध हैं; विशेष रूप से, शायद आप इनमें से एक चाहेंगे:

  • protobuf-net.Grpc.AspNetCore उन सर्वरों के लिए जो ASP.NET Core 3.1 का उपयोग करते हैं
  • protobuf-net.Grpc.Native उन क्लाइंट या सर्वरों के लिए जो नेटिव/अनमैनेज्ड API का उपयोग करते हैं
  • protobuf-net.Grpc और Grpc.Net.Client उन क्लाइंट के लिए जो .NET Core 3.1 पर HttpClient का उपयोग करते हैं

उदाहरण C#, VB और F# में उपलब्ध हैं।

और कुछ?

protobuf-net.Grpc Marc Gravell (@marcgravell) द्वारा बनाया और अनुरक्षित किया गया है, जो protobuf-net के लेखक हैं।

यह grpc के टूल्स का उपयोग करता है, लेकिन उस प्रोजेक्ट के साथ औपचारिक रूप से संबद्ध, सम्बद्ध या अनुमोदित नहीं है।

मैं आपकी प्रतिक्रिया की प्रतीक्षा कर रहा हूँ, और यदि यह आपके बहुत समय की बचत कर सकता है, तो आप हमेशा स्वागत हैं
