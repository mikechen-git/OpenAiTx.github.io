ComfyUI FLOAT

Questo progetto fornisce un wrapper ComfyUI di FLOAT per Generative Motion Latent Flow Matching per Talking Portrait guidato dall’audio

Per una versione più avanzata e mantenuta, consulta: ComfyUI-FLOAT_Optimized

⭐ Supporto

Se ti piacciono i miei progetti e desideri vedere aggiornamenti e nuove funzionalità, considera di supportarmi. Aiuta molto!

🚀 Installazione

git clone https://github.com/yuvraj108c/ComfyUI-FLOAT.git
cd ./ComfyUI-FLOAT
pip install -r requirements.txt

☀️ Utilizzo

  • Carica workflow di esempio
  • Carica immagine e audio di guida, clicca su queue
  • Modelli scaricati automaticamente in /ComfyUI/models/float
  • I modelli sono organizzati come segue:
``.bash |-- float.pth # modello principale |-- wav2vec2-base-960h/ # audio encoder | |-- config.json | |-- model.safetensors | |-- preprocessor_config.json |-- wav2vec-english-speech-emotion-recognition/ # emotion encoder |-- config.json |-- preprocessor_config.json |-- pytorch_model.bin

🛠️ Parametri

  • ref_image: Immagine di riferimento con un volto (deve avere batch size 1)
  • ref_audio: Audio di riferimento (Per audio lunghi (es. 3+ minuti), assicurati di avere abbastanza ram/vram)
  • a_cfg_scale: Scala di guidance audio classifier-free (default:2)
  • r_cfg_scale: Scala di guidance reference classifier-free (default:1)
  • emotion: none, angry, disgust, fear, happy, neutral, sad, surprise (default:none)
  • e_cfg_scale: Intensità dell’emozione (default:1). Per video con emozione più intensa, prova un valore elevato da 5 a 10
  • crop: Abilita solo se l’immagine di riferimento non ha un volto centrato
  • fps`: Framerate del video di output (default:25)
