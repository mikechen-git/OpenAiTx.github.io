ML Training Reference Architectures & Tests

Peringatan
Kami saat ini sedang melakukan refactoring besar pada repositori ini, terutama fokus pada bagian test cases. Jika Anda lebih memilih menggunakan struktur direktori sebelumnya dan test cases yang sudah tidak digunakan, silakan lihat v1.1.0.

Repositori ini berisi arsitektur referensi dan test cases untuk pelatihan model terdistribusi dengan Amazon SageMaker Hyperpod, AWS ParallelCluster, AWS Batch, dan Amazon EKS. Test cases mencakup berbagai tipe dan ukuran model serta berbagai framework dan optimasi paralel (Pytorch DDP/FSDP, MegatronLM, NemoMegatron...).

Komponen utama dari direktori ini adalah:

reference-architectures/
|-- 1.architectures/               # Template CloudFormation untuk arsitektur referensi
|-- 2.ami_and_containers/          # Skrip untuk membuat AMI dan image container
|-- 3.test_cases/                  # Test case referensi dan/atau skrip benchmark
|-- 4.validation_observability/    # Alat untuk mengukur performa atau troubleshooting
`-- ...

CATATAN: arsitektur dirancang untuk bekerja dengan bucket S3 dan VPC yang dibuat menggunakan template referensi 1.architectures/0.s3/ dan 1.architectures/1.vpc_network/. Sangat disarankan untuk menerapkan kedua template ini sebelum menerapkan arsitektur referensi manapun.

0. Workshop

Anda dapat mengikuti workshop berikut untuk melatih model di AWS. Masing-masing berisi contoh untuk beberapa test case serta informasi penting tentang mengoperasikan cluster untuk pelatihan LLM.

Nama Komentar
Amazon SageMaker HyperPod Workshop untuk SageMaker HyperPod, menunjukkan cara deploy dan memonitor
AWS ParallelCluster Workshop serupa seperti HyperPod tapi di ParallelCluster
Amazon SageMaker HyperPod EKS Workshop untuk SageMaker HyperPod EKS, menunjukkan cara deploy dan memonitor

1. Arsitektur

Arsitektur berada di 1.architectures dan terdiri dari utilitas serta arsitektur terkait layanan.

Nama Kategori Penggunaan
0.s3 Storage Membuat bucket S3
1.vpc_network Network Membuat VPC dengan subnet dan sumber daya yang diperlukan
2.aws-parallelcluster Compute Template cluster untuk pelatihan GPU & custom silicon
3.aws-batch Compute Template AWS Batch untuk pelatihan terdistribusi
4.amazon-eks Compute File manifest untuk pelatihan dengan Amazon EKS
5.sagemaker-hyperpod Compute Template SageMaker HyperPod untuk pelatihan terdistribusi

Akan ada lebih banyak lagi, silakan tambahkan yang baru (misal Ray?). Anda juga akan menemukan dokumentasi untuk EFA dan variabel lingkungan yang direkomendasikan.

2. Custom Amazon Machine Images

Image mesin kustom dapat dibangun menggunakan Packer untuk AWS ParallelCluster, Amazon EKS dan EC2 biasa. Image ini berbasis pada peran dan playbook Ansible.

3. Test cases

Test cases diorganisasi berdasarkan framework dan mencakup berbagai skenario pelatihan terdistribusi. Setiap test case mencakup skrip dan konfigurasi yang diperlukan untuk menjalankan pekerjaan pelatihan terdistribusi.

Test Cases PyTorch

  • FSDP/ - Contoh pelatihan Fully Sharded Data Parallel
  • megatron-lm/ - Contoh pelatihan terdistribusi Megatron-LM
  • nemo-launcher/ - Contoh NeMo Launcher untuk pelatihan terdistribusi. Test case ini untuk NeMo versi 1.0 saja.
  • nemo-run/ - Contoh pelatihan terdistribusi framework NeMo. Test case ini untuk NeMo versi 2.0+.
  • neuronx-distributed/ - Contoh pelatihan terdistribusi AWS Trainium
  • mosaicml-composer/ - Contoh MosaicML Composer
  • picotron/ - Contoh pelatihan terdistribusi PicoTron
  • torchtitan/ - Contoh TorchTitan
  • cpu-ddp/ - Contoh Distributed Data Parallel berbasis CPU
  • bionemo/ - Contoh pelatihan terdistribusi BioNeMo

Test Cases JAX

  • jax/ - Contoh pelatihan terdistribusi berbasis JAX menggunakan PaxML

Setiap test case mencakup:

  • Skrip dan konfigurasi pelatihan
  • Definisi container (jika ada)
  • Skrip peluncuran untuk berbagai jenis cluster
  • Alat pemantauan performa dan validasi

5. CI

Tes integrasi ditulis dalam pytest. Jalankan saja:

pytest .

Alternatifnya, Anda dapat menjalankan tes tanpa menangkap stdout dan mempertahankan semua image docker dan artefak lainnya.

pytest -s --keep-artifacts=t
