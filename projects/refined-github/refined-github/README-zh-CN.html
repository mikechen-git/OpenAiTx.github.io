Refined GitHub

简化 GitHub 界面并增加实用功能的浏览器扩展

我们频繁使用 GitHub，发现了许多想要修复的不便之处。所以，这里有很多“龙”。

我们希望 GitHub 能注意到并实现这些亟需的改进。如果你喜欢其中任何一项功能，欢迎在 GitHub feedback 发起讨论，或联系 GitHub support 反馈建议。

仓库

  • 🔥 在仓库名称旁添加构建/CI 状态图标。
  • 为仓库导航下拉菜单增加实用链接
  • 在分支对比视图中添加切换分支的链接。
  • 在侧边栏显示仓库创建时间。
  • 在你的 fork 仓库中，显示你向原仓库发起的未关闭 PR 数量。
  • 让仓库文件列表中的部分按钮更紧凑，为其他功能腾出空间。
  • 创建仓库时自动禁用 Projects 和 Wiki。
  • 在仓库、Issue 和 PR 页面使侧边栏置顶（如果适配视口）。
  • 在个人/组织仓库中添加 github.io 主页访问链接。
  • 在工作流侧边栏显示可手动触发的指示器及下次定时运行时间（如适用）。
  • 简化仓库删除流程。为 0-star 仓库增加“删除 Fork”按钮，支持 shift alt 快捷点击两步删除。
  • 帮助你在归档仓库中查找 fork。
  • 将“Security”和“Insights”移动到仓库导航下拉菜单。未使用时也移动“Projects”、“Actions”和“Wiki”标签。
  • 在公共仓库头部显示用户头像。
  • 在仓库任意位置快速创建 Issue 的链接。
  • 在 Issue/PR 列表和 @提及中显示小头像。
  • 🔥 提示你当前是否查看的是仓库最新版本，或是否有未发布的提交。
  • 🔥 为工作流页面添加回溯到触发该工作流的 PR 的链接。
  • 移动端标签更紧凑，显示更多内容。
  • 显示仓库是否为 Fork，并在头部显示 Star 数。
  • 在浏览指定 tag 下的文件时，增加跳转至该 release/tag 的链接。

文件管理

  • 添加下载整个文件夹的按钮，通过 https://download-directory.github.io 实现。
  • 在仓库文件列表中添加文件编辑按钮。
  • 在整个仓库启用文件查找快捷键 (t)。
  • 在 fork 仓库的分支上显示关联的 PR。
  • 为 HTML 文件添加预览链接。
  • 高亮显示文件列表中最近修改的项目。
  • 一键查看文件的上一个版本。

代码

  • 在代码和 Issue/PR 标题中将引用和 URL 自动转为链接。
  • 增强 y 快捷键，同时复制永久链接。
  • 为符号链接文件添加跳转链接。
  • 当前文件被未关闭 PR 修改时给出提醒。
  • 将制表符宽度由 8 个空格缩小为 4 个空格（GitHub 未遵循用户偏好时），参见 用户设置
  • 增加取消当前行选择的快捷键：esc。
  • 将 Markdown front matter 显示为竖向表格。
  • 在分支提交列表中显示涉及当前分支的 PR。

评论编写

  • 🔥 在评论输入框中启用 tab 和 shift tab 缩进。
  • 文本编辑器内增加插入可折叠内容（<details>）的按钮。
  • 🔥 自动调整评论输入区高度以适应内容，无需滚动条。
  • 一键编辑任意评论，无需打开下拉菜单。
  • 按下 Markdown 符号键时包裹选中文本而非替换： [ ` ' " * ~ _
  • 隐藏不必要的评论工具栏提示和按钮（每个都有快捷键）
  • 在 Issue 和 PR 中添加 @mention 用户的按钮。
  • 文本编辑器内增加快速插入简化 HTML 表格的按钮。
  • 在隐藏标签页中有未完成评论时提醒用户。
  • 编辑 Review 评论时，一键删除评论。
  • 在部分提交/PR/Issue 标题输入框禁用 enter 直接提交，防止误操作。请使用 ctrl enter 提交。

评论阅读

  • 🔥 在评论反应处显示是谁做出的反应。
  • 评论中链接的 gist 可内嵌预览。
  • 增加 浏览仓库 和 相关文件 在评论时点的快照链接。
  • 在用户名旁显示用户真实姓名。
  • 将 URL 和仓库链接缩短为更易读的引用格式，如 "_user/repo/.file@d71718d"。
  • 🔥 内联预览被隐藏的评论。
  • 🔥 高亮显示 Issue 中最有用的评论。
  • 隐藏反应类评论（如 "+1", "👍" 等）（维护者除外）但依然可展开查看。
  • 限制超长代码块和引用的高度。
  • 简化评论隐藏的界面。
  • Issue 列表中的“评论”图标直接跳转到最新评论。
  • 在长会话中，GitHub 会将评论折叠到“已隐藏 N 项，加载更多...”下，按下 alt 点击可一次性加载最多 200 条评论（默认 60 条）。
  • 为 Issue、PR 会话和 PR 文件列表增加快捷键：j 聚焦下一个评论/文件，k 聚焦上一个。
  • 在长 Issue 页眉提示有隐藏评论，按 Cmd+F 或 Ctrl+F 时自动跳转到隐藏评论处。

会话

  • 一键打开所有可见的 Issue/PR。
  • 让 Issue/PR 列表的筛选工具栏置顶。
  • 高亮显示由你或当前仓库协作者创建的 Issue/PR。
  • 在 Issue/PR 列表中，将标签左对齐并显示在标题下方。
  • 🔥 将 Issue/PR 默认排序更改为“最近更新”。
  • 在搜索下拉菜单中增加“你参与的所有内容”和“你订阅的所有内容”筛选项。
  • 在全局 Issue/PR 搜索中添加“你拥有仓库的 Issue/PR”和“你评论过的 Issue/PR”筛选项。
  • 🔥 隐藏 Issue/PR 侧边栏中为空的分区（或仅隐藏“空”标签）。
  • 当议题/PR列表中的 Projects 和 Milestones 过滤器为空时隐藏它们。
  • 添加快捷方式，可以一次性切换所有相似项（最小化评论、延迟差异等）：在每个按钮或复选框上 alt 点击。
  • 允许在搜索中切换 is:open/is:closed/is:merged 过滤器。
  • 如果存在带有 "bug" 标签的未关闭议题，则在仓库中添加一个 "Bugs" 选项卡。
  • 将置顶议题的 "opened" 时间替换为 "updated" 时间。
  • 将置顶议题的布局从并排显示更改为标准列表。
  • 为议题/PR 中的标签添加一键移除按钮。
  • 移除议题和 PR 头部的重复信息（如 "User wants to merge X commits from Y into Z"）。
  • 使机器人提交和 PR 显得更暗淡以减少干扰。
  • 为编辑议题/PR 标题添加取消快捷键：esc。
  • 当议题/PR 的更新时间与打开/关闭/合并时间相同时，在列表中隐藏更新时间。
  • 将评论中的 "Contributor" 和 "Member" 标签链接到作者在该仓库的提交。
  • 为议题/PR 添加跳转到最新关闭事件的链接。
  • 可一键将议题 "关闭为未计划"，无需三步操作。
  • 在锁定的议题和 PR 上显示标签。

查看拉取请求

  • 在访问 PR 提交时，添加链接指向非 PR 的提交。
  • 在 PR 列表中添加检查和草稿 PR 下拉过滤器。
  • 在 PR 列表中显示颜色区分的审核数。
  • 如果 PR 基准分支不是默认分支，则在 PR 列表中显示基准分支。
  • 在 PR 中隐藏非活跃部署。
  • 在 Commits 选项卡页面底部添加重复的提交导航按钮。
  • 在 PR 审查中，当评论被隐藏时显示评论指示器。
  • 在 PR 列表中显示哪些 PR 存在冲突。
  • 在 PR 提交中显示差异统计信息。
  • 在 PR 中已删除的分支上添加删除线。
  • 在 PR 文件选项卡中，批量标记/取消标记多个文件为“已查看”。点击你想要标记/取消的第一个复选框，然后 shift-点击另一个；两者之间的所有文件都将被标记/取消为“已查看”。
  • 🔥 显示合并 PR 首次被包含的 Git 标签，或建议创建新版本（如果尚未发布）。
  • 点击进度条时，跳转到 PR 中第一个未查看的文件。
  • 添加跳转到最新的变更请求评论的链接。
  • 在 PR 页眉添加链接，以打开最新部署。
  • 在拆分视图无用时，始终使用统一差异视图。
  • 更容易区分列表中的草稿 PR。

编辑拉取请求

  • 🔥 将 PR 标题用作默认 squash 提交标题 并在更改时同步 PR 标题与提交标题。](https://github-production-user-asset-6210df.s3.amazonaws.com/140871606/257011579-25332762-b25f-407b-b6d2-bbfc13de2be7.png)
  • 为每个 PR 添加“更新分支”按钮。 GitHub 也有同样的功能，但必须手动配置受保护分支。
  • 简化 PR 审查表单：通过一键式审核类型按钮更快地批准或拒绝审查。
  • 为 PR 选项卡添加键盘快捷键：g ← 和 g →，或 g 1、g 2、g 3、g 4。
  • 合并 PR 后自动删除分支（如有可能）。 已知长期存在的常用分支名会被排除（如 develop、release/* 等）
  • 支持一键创建草稿 PR 和公开 gist。
  • 清理 PR 合并提交信息，仅保留去重后的联合作者。
  • 为 PR 侧边栏添加快速审查按钮，自动聚焦审查文本框，并添加打开审查弹窗的快捷键：v。
  • 使用新 PR 第一个提交的信息作为标题和描述。

提交

  • 为提交添加 .patch 和 .diff 文件链接。
  • 为 PR 和提交中的文件添加查看原始、追溯（blame）、历史等链接。
  • 为更改差异样式、忽略空白添加一键按钮，并添加忽略空白的快捷键：d w。
  • 将 “展开差异” 按钮扩宽，使其在整屏内都可点击。
  • 隐藏差异符号，因为差异已经用颜色区分。
  • 建议将提交和 PR 标题限制在 72 个字符内。
  • 在提交旁显示其对应的标签。
  • 在提交列表中标记合并提交。
  • 浏览追溯时，Alt-点击 “Reblame” 按钮将优先提取相关 PR 的所有提交，而不是将该提交视为单次更改。
  • 用图标指示提交和 PR 中的文件是新增还是删除。
  • 点击文件头部栏即可切换显示/隐藏文件差异。
  • 在查看作者所有提交时，保留当前分支和路径。
  • 点击提交框即可切换显示/隐藏提交信息。
  • 将对比页面的 "X files changed" 文本变为可点击链接，跳转到差异。
  • 在提交信息前显示 conventional commit 类型标签。

标签与发布

  • 在发布资源旁显示下载次数。
  • 添加 Releases 选项卡及其键盘快捷键：g r。
  • 在标签/发布页面添加标签下拉/搜索功能。
  • 在 Releases 页面添加创建新发布的快捷键：c。
  • 🔥 为每个标签/发布添加链接，可查看自上一个标签/发布以来的更改。
  • 添加将发布转换为草稿的按钮。
  • 在发布页面添加按钮，可查看更新日志文件。

个人主页

  • 在个人主页上显示该用户是否关注了你。
  • 在个人主页添加用户公开 gist 的链接。
  • 在你的个人主页上标记私有组织。
  • 添加访问你自己主页的快捷键：g m。
  • 添加用户最受欢迎仓库的链接。
  • 默认在个人主页中过滤掉 fork 和已归档仓库，仅显示源仓库。 (但仍可手动显示。)
  • 将用户 hovercard 和主页中的位置信息变为可点击链接。
  • 在用户 hovercard 上显示其本地时间（基于其最近一次提交）。
  • 在用户主页的仓库选项卡和全局搜索中，添加指向议题和拉取请求的链接。

通知

  • 添加按钮，可一次性打开所有未读通知。
  • 🔥 添加全局快捷键，可一次性打开所有未读通知：g u
  • 添加选择所有可见通知的快捷键：a。
  • 按住 Alt 时，阻止通知栏动作重定向到通知收件箱。
  • 添加跳转到通知最后一页的链接。
  • 使 PR 通知指向 “会话” 选项卡，而不是可能为 404 的提交页面。
  • 让通知操作栏固定显示。

全局

  • 🔥 在 404 页面上添加可能相关的页面和替代方案。
  • 在通过 j 和 k 导航时，添加快捷键 shift o 可在新标签页打开所选项。
  • 当下拉菜单不再可见时自动关闭。
  • GitHub 在某些地方会渲染 `反引号中的文本`，但在其他地方不会；该功能补齐遗漏的情况。
  • 添加分页导航快捷键：← 和 →。
  • 可在 Marketplace 查看当前 Action 的实际用法。
  • 在帮助弹窗（? 快捷键）中显示 Refined GitHub 新增的所有快捷键。

网络礼仪

  • 添加不显眼的网络礼仪提示（如旧已关闭议题、高活跃议题、草稿 PR 等）。
  • 在从默认分支创建 PR 时发出警告，因为这是一种反模式。
  • 在取消勾选 允许维护者编辑 时发出警告，因为这对维护者不友好。

修复 GitHub 不足

  • 移除仓库文件浏览器中的悬停高亮效果。
  • 移除仓库侧边栏中的不必要或冗余信息。
  • 在 “快速 PR” 页面中将分支引用变为链接。
  • 移除搜索字段的自动补全。
  • 将 PR 页面中的 "查看文件" 链接指向分支，而不是提交，这样在 "查看文件" 页面上可用编辑/删除按钮。
  • 重试因 GitHub 代理限制导致下载失败的图片。
  • 让部分下拉菜单变为一键操作，避免多余的两步点击。
  • 建议修复被 GitHub 错误截断的链接。
  • 将编辑历史弹窗中的用户名变为可点击链接。
  • 防止误点多次 "创建拉取请求" 时生成重复 PR。
