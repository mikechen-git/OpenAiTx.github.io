Refined GitHub

ส่วนขยายเบราว์เซอร์ที่ทำให้หน้าตา GitHub เรียบง่ายขึ้นและเพิ่มฟีเจอร์ที่มีประโยชน์

พวกเราใช้ GitHub เป็นประจำและพบว่ามีจุดที่น่ารำคาญหลายอย่างที่อยากจะแก้ไข ดังนั้นนี่คือจุดเริ่มต้นของเรา

เราหวังว่า GitHub จะเห็นและนำการปรับปรุงเหล่านี้ไปใช้ หากคุณชอบการปรับปรุงเหล่านี้ โปรดเปิดการสนทนาใน GitHub feedback หรือ ติดต่อ GitHub support เพื่อแจ้งให้ทราบ

GitHub Enterprise ก็รองรับเช่นกัน: วิธีเปิดใช้งาน

เครื่องหมายการค้า GITHUB และ REFINED GITHUB เป็นของ GitHub, Inc. และได้รับอนุญาตให้ใช้

เฟรมเวิร์กเว็บ low-code ที่ครบเครื่องและซอฟต์แวร์โอเพนซอร์สระดับโลกอีกมากมาย

จุดเด่น 🔥

ทำให้เห็นอักขระช่องว่าง

เพิ่มตัวแก้ไข conflict แบบคลิกเดียว

แสดงว่าหัว branch ของ PR ตามหลังแค่ไหน + บอก base commit ของมัน

ให้คุณซ่อนทุก event ยกเว้นคอมเมนต์หรือคอมเมนต์ที่ยังไม่ได้แก้ใน issues และ PRs

ให้คุณ subscribe กับ event การเปิด/ปิด issue เพียงคลิกเดียว

เพิ่มลิงก์ไปที่ default branch ในหน้ารายการไดเรกทอรีและไฟล์

เพิ่มปุ่มสำหรับ discard การเปลี่ยนแปลงทั้งหมดของไฟล์ใน PR

เลือกการแจ้งเตือนตามประเภทและสถานะ

รีโพสิทอรี

  • 🔥 เพิ่มไอคอนสถานะ build/CI ข้างชื่อ repo
  • เพิ่มลิงก์ที่เป็นประโยชน์ในเมนูดรอปดาวน์นำทางของ repo
  • เพิ่มลิงก์สลับ branch ในหน้าเปรียบเทียบ branch
  • แสดงอายุของ repo ใน sidebar
  • ใน repo ที่ fork จะแสดงจำนวน PR ที่เปิดไปยัง repo ต้นทาง
  • ทำให้ปุ่มบางปุ่มในรายชื่อ repo กะทัดรัดขึ้น เพื่อให้พื้นที่กับฟีเจอร์อื่น
  • ปิดการใช้งาน projects และ wikis อัตโนมัติเมื่อสร้าง repo ใหม่
  • ทำให้ sidebar ใน repo, issue, และ PR ติดขอบหน้าจอถ้าแสดงผลได้
  • เพิ่มลิงก์ไปยังเว็บไซต์ github.io ของผู้ใช้จาก repo นั้น
  • ใน sidebar workflow แสดงตัวบ่งชี้ว่าทำงานด้วยตนเองได้ และเวลานัดถัดไปถ้ามี
  • ลบ repo ได้ง่ายขึ้น เพิ่มปุ่ม "Delete fork" ใน repo ที่ไม่มีดาว และใช้ shift alt ลบ repo ใน 2 คลิก
  • ช่วยค้นหา fork ใน repo ที่ถูก archive
  • ย้าย "Security" และ "Insights" ไปไว้ในเมนูนำทางดรอปดาวน์ของ repo รวมถึง "Projects", "Actions" และ "Wiki" หากว่าง/ไม่ใช้
  • เพิ่มรูปโปรไฟล์ที่หัวของ repo สาธารณะ
  • เพิ่มลิงก์สร้าง issue จากทุกที่ใน repo
  • แสดง avatar เล็กข้างชื่อผู้ใช้ในรายการ issue/PR และการ mention
  • 🔥 แจ้งว่าคุณกำลังดู repo เวอร์ชันล่าสุดหรือมี commit ที่ยังไม่ release
  • 🔥 เพิ่มลิงก์กลับไปยัง PR ที่รัน workflow
  • ทำให้แท็บกะทัดรัดขึ้นในมือถือเพื่อให้เห็นหลายแท็บมากขึ้น
  • แสดงว่า repo นี้เป็น fork และเพิ่มจำนวนดาวที่หัวข้อ
  • เมื่อเรียกดูไฟล์ใน repo บน tag เฉพาะ จะเพิ่มลิงก์ไปยัง release/tag นั้น

การจัดการไฟล์

  • เพิ่มปุ่มดาวน์โหลดโฟลเดอร์ทั้งโฟลเดอร์, ผ่าน https://download-directory.github.io
  • เพิ่มปุ่มแก้ไขไฟล์จากรายการไฟล์ใน repo
  • เปิดใช้คีย์ลัด File Finder (t) ใน repo ทั้งหมด
  • แสดง PR ที่เกี่ยวข้องกับ branch ใน repo ที่ fork
  • เพิ่มลิงก์เพื่อดูตัวอย่างไฟล์ HTML
  • ไฮไลต์รายการไฟล์ที่เพิ่งแก้ไขล่าสุดในรายการไฟล์
  • ให้คุณดูเวอร์ชันก่อนหน้าของไฟล์ได้ในคลิกเดียว

โค้ด

  • เปลี่ยนการอ้างอิง issue/PR และ URL ในโค้ดและชื่อเรื่อง issue/PR ให้เป็นลิงก์
  • ปรับปรุง คีย์ลัด y ให้คัดลอก permalink ด้วย
  • เปลี่ยนไฟล์ symbolic link ให้เป็นลิงก์
  • แจ้งเตือนถ้าไฟล์ปัจจุบันถูกแก้ไขโดย PR ที่เปิดอยู่
  • ลดขนาด tab เหลือ 4 ช่องว่างแทน 8 ช่อง ในกรณีที่ GitHub ไม่ทำตาม ค่าตั้งของผู้ใช้
  • เพิ่มคีย์ลัดสำหรับยกเลิกการเลือกบรรทัดปัจจุบัน: esc
  • แสดง front matter ของ Markdown เป็นตารางแนวตั้ง
  • ในรายการ commit ของ branch แสดง PR ที่เกี่ยวข้องกับ branch ปัจจุบัน

การเขียนคอมเมนต์

  • 🔥 เปิดให้ใช้ tab และ shift tab สำหรับเยื้องในฟิลด์คอมเมนต์
  • เพิ่มปุ่มใน text editor เพื่อแทรกเนื้อหาที่พับได้ (ผ่าน <details>)
  • 🔥 ปรับขนาดฟิลด์คอมเมนต์อัตโนมัติตามเนื้อหาและไม่แสดง scroll bar
  • แก้ไขคอมเมนต์ใดก็ได้ในคลิกเดียวโดยไม่ต้องเปิดดรอปดาวน์
  • ครอบข้อความที่เลือกด้วยสัญลักษณ์ Markdown เมื่อกดปุ่มเดียว แทนที่จะแทนที่ข้อความ: [ ` ' " * ~ _
  • ซ่อน tooltip และไอเท็ม toolbar ที่ไม่จำเป็นในฟิลด์คอมเมนต์ (แต่ละอันมีคีย์ลัดของตัวเอง)
  • เพิ่มปุ่ม @mention ผู้ใช้ใน issue และ PR
  • เพิ่มปุ่มใน text editor สำหรับแทรกตาราง HTML แบบง่ายอย่างรวดเร็ว
  • แจ้งเตือนผู้ใช้หากมีคอมเมนต์ที่ยังไม่เสร็จในแท็บที่ซ่อนอยู่
  • เพิ่มปุ่มลบคอมเมนต์รีวิวในคลิกเดียวขณะกำลังแก้ไข
  • ปิดใช้งานคีย์ลัด enter เพื่อส่งในบางฟิลด์ชื่อ commit/PR/issue เพื่อป้องกันการส่งโดยไม่ตั้งใจ ให้ใช้ ctrl enter แทน

การอ่านคอมเมนต์

  • 🔥 เพิ่ม avatar ของผู้ที่กด reaction กับคอมเมนต์นั้น
  • ฝัง gist สั้นๆ เมื่อถูกลิงก์ในคอมเมนต์แยกบรรทัดเอง
  • เพิ่มลิงก์เพื่อ เรียกดู repo และ ไฟล์ที่ลิงก์ไว้ ณ เวลาที่คอมเมนต์แต่ละอัน
  • เพิ่มชื่อจริงของผู้ใช้ข้างชื่อผู้ใช้
  • ย่อ URL และ repo URL ให้เป็นแบบอ่านง่าย เช่น "_user/repo/.file@d71718d"
  • 🔥 แสดงตัวอย่างคอมเมนต์ที่ถูกซ่อนแบบ inline
  • 🔥 ไฮไลต์คอมเมนต์ที่มีประโยชน์มากที่สุดใน issue
  • ซ่อนคอมเมนต์ reaction ("+1", "👍", …) (ยกเว้นของผู้ดูแลระบบ) แต่ยังสามารถแสดงได้
  • จำกัดความสูงของ code block และ quote ที่สูงมาก
  • ทำให้ UI สำหรับซ่อนคอมเมนต์ง่ายขึ้น
  • ทำให้ไอคอน "comment" ในรายการ issue ลิงก์ไปยังคอมเมนต์ล่าสุดของ issue นั้น
  • ในการสนทนายาวที่ GitHub ซ่อนคอมเมนต์ไว้ใต้ "N hidden items. Load more..." เมื่อ alt-คลิกจะโหลดได้สูงสุด 200 คอมเมนต์ในครั้งเดียวแทน 60
  • เพิ่มคีย์ลัดใน issue, PR, และรายการไฟล์ PR: j โฟกัสคอมเมนต์/ไฟล์ถัดไป; k โฟกัสคอมเมนต์/ไฟล์ก่อนหน้า
  • แจ้งว่ามีคอมเมนต์ถูกซ่อนใน header ของ issue ยาวๆ และเลื่อนลงไปยังคอมเมนต์ที่ซ่อนเมื่อกด Cmd+F หรือ Ctrl+F

การสนทนา

  • ให้คุณเปิด issue/PR ที่แสดงอยู่ทั้งหมดในครั้งเดียว
  • ทำให้แถบเครื่องมือฟิลเตอร์ของรายการ issue/PR ติดขอบหน้าจอ
  • ไฮไลต์ issue/PR ที่เปิดโดยคุณหรือผู้ร่วมงาน repo ปัจจุบัน
  • ในรายการ issue/PR จัดตำแหน่ง label ไปทางซ้ายใต้แต่ละหัวข้อ
  • 🔥 เปลี่ยนการเรียงลำดับเริ่มต้นของ issue/PR เป็น Recently updated
  • เพิ่ม filter Everything you’re involved in และ Everything you subscribed to ในเมนูค้นหาแบบดรอปดาวน์
  • เพิ่ม filter สำหรับ issue/PR ใน repo ของคุณ และ ที่คุณคอมเมนต์ ในการค้นหา issue/PR ทั่วโลก
  • 🔥 ซ่อนส่วนว่าง (หรือเฉพาะ label "ว่าง") ใน sidebar ของ issue/PR
  • ซ่อนตัวกรอง Projects และ Milestones ในรายการ issue/PR หากว่างเปล่า
  • เพิ่มคีย์ลัดเพื่อสลับรายการที่คล้ายกันทั้งหมด (คอมเมนต์ที่ย่อไว้, diff ที่ถูกเลื่อน ฯลฯ) ในครั้งเดียว: alt คลิก ที่แต่ละปุ่มหรือเช็คบ็อกซ์
  • ให้คุณสลับระหว่างตัวกรอง is:open/is:closed/is:merged ในการค้นหา
  • เพิ่มแท็บ "Bugs" ให้กับ repo หากมี issue ที่เปิดอยู่พร้อม label "bug"
  • แทนที่เวลาที่ "opened" ด้วยเวลาที่ "updated" บน pinned issues
  • เปลี่ยนเลย์เอาต์ของ pinned issues จากแบบข้างกันเป็นแบบรายการมาตรฐาน
  • เพิ่มปุ่มคลิกเดียวเพื่อลบ label ใน issues/PRs
  • ลบข้อมูลซ้ำใน header ของ issues และ PRs ("User wants to merge X commits from Y into Z")
  • ทำให้ commit และ PR จากบอทจางลงเพื่อลดเสียงรบกวน
  • เพิ่มคีย์ลัดเพื่อยกเลิกการแก้ไขชื่อ issue/PR: esc
  • ซ่อนเวลาการอัปเดตของ issues/PRs ในรายการเมื่อมันตรงกับเวลาเปิด/ปิด/merged
  • ลิงก์ label "Contributor" และ "Member" ในคอมเมนต์ไปยัง commit ของผู้เขียนใน repo
  • เพิ่มลิงก์เพื่อข้ามไปยัง event ปิดล่าสุดของ issue/PR
  • ให้คุณ "ปิด issue เป็น unplanned" ได้ด้วยคลิกเดียวแทนที่จะต้องสามคลิก
  • แสดง label บน issue และ PR ที่ล็อกแล้ว

การดู pull requests

  • เพิ่มลิงก์ไปยัง commit ปกติเมื่อเยี่ยมชม commit ใน PR
  • เพิ่มตัวกรอง Checks และ Draft PR แบบดรอปดาวน์ในรายการ PR
  • แสดงจำนวนรีวิวแบบมีรหัสสีในรายการ PR
  • แสดง base branch ในรายการ PR หากไม่ใช่ branch เริ่มต้น
  • ซ่อน deployment ที่ไม่ได้ใช้งานใน PR
  • เพิ่มปุ่มนำทาง commit ซ้ำที่ด้านล่างของหน้าแท็บ Commits
  • เพิ่มตัวบ่งชี้คอมเมนต์เมื่อมีคอมเมนต์ที่ถูกซ่อนในการรีวิว PR
  • แสดงว่า PR ไหนมี conflict ในรายการ PR
  • เพิ่มสถิติ diff ใน commit ของ PR
  • ขีดฆ่า branch ที่ถูกลบใน PR
  • ทำเครื่องหมาย/ยกเลิกเครื่องหมายไฟล์หลายไฟล์ว่า “Viewed” ในแท็บ Files ของ PR โดยคลิกเช็คบ็อกซ์แรกที่ต้องการ จากนั้น shift-คลิกอีกอัน; ไฟล์ทั้งหมดระหว่างสองเช็คบ็อกซ์จะถูกทำเครื่องหมาย/ยกเลิก
  • 🔥 แสดง Git tag แรกที่ PR ที่ merge แล้วถูกใส่ไว้ หรือแนะนำให้สร้าง release หากยังไม่มี release
  • ข้ามไปยังไฟล์ที่ยังไม่ได้ดูไฟล์แรกใน PR เมื่อคลิกที่ progress bar
  • เพิ่มลิงก์เพื่อข้ามไปยังคอมเมนต์ change request ล่าสุด
  • เพิ่มลิงก์เพื่อเปิด deployment ล่าสุดจาก header ของ PR
  • ใช้ unified diff เสมอในไฟล์ที่ split diff ไม่เป็นประโยชน์
  • ทำให้แยกแยะ draft PR ในรายการได้ง่ายขึ้น

การแก้ไข pull requests

  • 🔥 ใช้ชื่อ PR เป็นชื่อ commit squash โดยดีฟอลต์ และ อัปเดตชื่อ PR ให้ตรงกับชื่อ commit หากมีการเปลี่ยนแปลง
  • เพิ่มปุ่ม "Update branch" ให้กับทุก PR GitHub มีฟีเจอร์เดียวกันแต่ต้องตั้งค่าด้วย branch ที่ได้รับการปกป้อง
  • ทำให้ฟอร์มรีวิว PR ง่ายขึ้น: อนุมัติหรือปฏิเสธรีวิวได้เร็วขึ้นด้วยปุ่มรีวิวแบบคลิกเดียว
  • เพิ่มคีย์ลัดสำหรับเปลี่ยนแท็บ PR: g ← และ g → หรือ g 1, g 2, g 3, และ g 4
  • ลบ branch อัตโนมัติหลังจาก merge PR ถ้าเป็นไปได้ โดยชื่อ branch ที่เป็นที่รู้จักว่าใช้งานระยะยาวจะไม่ถูกลบ (develop, release/* เป็นต้น)
  • ให้คุณสร้าง draft PR และ public gist ได้ในคลิกเดียว
  • ลบข้อความ commit merge ของ PR ที่รกออก เหลือแต่ co-author ที่ไม่ซ้ำกัน
  • เพิ่มปุ่ม quick-review ใน sidebar ของ PR, โฟกัส textarea รีวิวโดยอัตโนมัติ, และเพิ่มคีย์ลัดเพื่อเปิด popup รีวิว: v
  • ใช้ commit แรกเป็นชื่อและคำอธิบายของ PR ใหม่

Commits

  • เพิ่มลิงก์ไปยังไฟล์ .patch และ .diff ใน commit
  • เพิ่มลิงก์เพื่อดูไฟล์ raw, blame และ history ของไฟล์ใน PR และ commit
  • เพิ่มปุ่มคลิกเดียวเพื่อเปลี่ยนรูปแบบ diff และข้ามช่องว่าง พร้อมคีย์ลัด: d w
  • ขยายปุ่ม Expand diff ให้คลิกได้ทั่วหน้าจอ
  • ซ่อนสัญลักษณ์ diff เพราะมีการใช้สีอยู่แล้ว
  • แนะนำให้จำกัดชื่อ commit และ PR ไม่เกิน 72 ตัวอักษร
  • แสดงแท็กที่เกี่ยวข้องถัดจาก commit
  • ทำเครื่องหมาย merge commit ในรายการ commit
  • เมื่อดู blame, Alt-คลิกที่ปุ่ม “Reblame” จะดึง commit ที่เกี่ยวข้องกับ PR ก่อน แทนที่จะมอง commit นั้นเป็นการเปลี่ยนแปลงเดียว
  • แสดงไอคอนว่ากำลังเพิ่มหรือลบไฟล์ใน commit และ PR
  • สลับการแสดงผล diff ของไฟล์โดยคลิกที่แถบหัวเรื่องของไฟล์
  • คง branch และ path ปัจจุบันไว้เมื่อดู commit ทั้งหมดของผู้เขียน
  • สลับแสดง/ซ่อนข้อความ commit โดยคลิกที่กล่อง commit
  • ลิงก์ข้อความ "X files changed" บนหน้า compare เพื่อข้ามไปยัง diff ได้
  • แสดงประเภท conventional commit เป็น label หน้าข้อความ commit

แท็กและรีลีส

  • เพิ่มจำนวนการดาวน์โหลดถัดจาก asset ของ release
  • เพิ่มแท็บ Releases และคีย์ลัด: g r
  • เพิ่มดรอปดาวน์/ค้นหาแท็กในหน้า tag/release
  • เพิ่มคีย์ลัดเพื่อสร้าง release ใหม่ในหน้า Releases: c
  • 🔥 เพิ่มลิงก์ไปยังการเปลี่ยนแปลงตั้งแต่แท็ก/รีลีสก่อนหน้าสำหรับแต่ละแท็ก/รีลีส
  • เพิ่มปุ่มเพื่อเปลี่ยน release เป็น draft
  • เพิ่มปุ่มเพื่อดูไฟล์ changelog จากหน้า releases

โปรไฟล์

  • ในหน้าโปรไฟล์ จะแสดงว่าผู้ใช้นั้นติดตามคุณหรือไม่
  • เพิ่มลิงก์ไปยัง public gist ของผู้ใช้ในโปรไฟล์
  • ทำเครื่องหมายองค์กรส่วนตัวในโปรไฟล์ของคุณเอง
  • เพิ่มคีย์ลัดเพื่อเข้าชมโปรไฟล์ของคุณเอง: g m
  • เพิ่มลิงก์ไปยัง repository ที่มีดาวสูงสุดของผู้ใช้
  • กรอง fork และ repo ที่ archive ออกจากโปรไฟล์โดยดีฟอลต์ เหลือแต่ source (แต่ยังสามารถแสดงได้)
  • ลิงก์ตำแหน่งผู้ใช้ใน hovercard และหน้าโปรไฟล์
  • แสดงเวลาท้องถิ่นของผู้ใช้ใน hovercard (อิงจาก commit ล่าสุด)
  • เพิ่มลิงก์ไปยัง issues และ pulls บนแท็บ repository ของโปรไฟล์ผู้ใช้และในการค้นหาทั่วโลก

การแจ้งเตือน

  • เพิ่มปุ่มเพื่อเปิดการแจ้งเตือนที่ยังไม่ได้อ่านทั้งหมดในครั้งเดียว
  • 🔥 เพิ่มคีย์ลัดทั่วโลกเพื่อเปิดการแจ้งเตือนที่ยังไม่ได้อ่านทั้งหมด: g u
  • เพิ่มคีย์ลัดเพื่อเลือกการแจ้งเตือนทั้งหมดที่แสดงอยู่: a
  • หยุด redirect ไปที่กล่องจดหมายแจ้งเตือนจากแถบ notification ขณะกด Alt
  • เพิ่มลิงก์ไปยังหน้าสุดท้ายของการแจ้งเตือน
  • ชี้ลิงก์การแจ้งเตือน PR ไปที่แท็บ Conversation แทนหน้า commits ซึ่งอาจเป็น 404
  • ทำให้แถบการกระทำของการแจ้งเตือนอยู่กับที่ (sticky)

ทั่วไป

  • 🔥 เพิ่มหน้าที่เกี่ยวข้องหรือทางเลือกในหน้า 404
  • เพิ่มคีย์ลัดเพื่อเปิดรายการที่เลือกในแท็บใหม่ขณะนำทางด้วย j และ k: shift o
  • ปิดเมนูดรอปดาวน์อัตโนมัติเมื่อไม่อยู่ในมุมมอง
  • GitHub แสดงผล `ข้อความใน backticks` บางที่แต่ไม่แสดงบางที่; ฟีเจอร์นี้เติมเต็มส่วนที่ลืมไว้
  • เพิ่มคีย์ลัดสำหรับนำทางหน้าด้วยปุ่มเปลี่ยนหน้า: ← และ →
  • ให้คุณเห็นว่าคนอื่นใช้ Action นี้ใน Marketplace อย่างไร
  • แสดงคีย์ลัดใหม่ทั้งหมดของ Refined GitHub ใน help modal (? คีย์ลัด)

มารยาทออนไลน์

  • เพิ่มการแจ้งเตือนมารยาทออนไลน์แบบไม่รบกวน (issue ที่ปิดมานาน, issue ที่มีการเคลื่อนไหวสูง, draft PRs ฯลฯ)
  • เตือนเมื่อสร้าง PR จาก branch เริ่มต้น เนื่องจากเป็น anti-pattern
  • เตือนเมื่อยกเลิกเลือก Allow edits from maintainers เพราะเป็นมิตรกับ maintainer น้อยลง

การแก้ไขข้อบกพร่องของ GitHub

  • ลบเอฟเฟกต์ hover ใน repo file browser
  • ลบข้อมูลที่ไม่จำเป็นหรือซ้ำซ้อนจาก sidebar ของ repository
  • ลิงก์อ้างอิง branch ในหน้า "Quick PR"
  • ลบ autocomplete ในช่องค้นหา
  • ชี้ปุ่ม "View file" ใน PR ไปที่ branch แทน commit เพื่อให้ปุ่ม Edit/Delete ใช้งานได้ในหน้า "View file"
  • ลองดาวน์โหลดภาพใหม่ที่โหลดไม่สำเร็จเพราะ proxy ของ GitHub จำกัด
  • ทำให้ดรอปดาวน์บางอันคลิกเดียวแทนที่จะต้องสองคลิกโดยไม่จำเป็น
  • แนะนำการแก้ไขลิงก์ที่ถูกย่อโดย GitHub ผิดพลาด
  • ลิงก์ชื่อผู้ใช้ใน popup ประวัติการแก้ไข
  • ป้องกันการสร้าง PR ซ้ำเมื่อคลิก "Create pull request" หลายครั้งโดยไม่ตั้งใจ
  • ย้ายปุ่ม "Convert PR to Draft" ไปที่กล่อง mergeability และเพิ่ม feedback ทางสายตาให้ปุ่ม confirm
  • ป้องกันไม่ให้ merge panel เปิดอัตโนมัติในทุกการโหลดหน้าหลังจากที่เคยเปิดไปแล้ว
  • เพิ่มคีย์ลัดสำหรับเลือกไอเท็มใน command palette ด้วย ctrl n และ ctrl p (macOS เท่านั้น)
  • ขณะเขียน/แก้ไขคอมเมนต์ เปิดลิงก์ preview ในแท็บใหม่แทนที่จะนำทางออกจากหน้า
  • เปลี่ยนเส้นทางไปยังรายการ issue ของ repo เมื่อการค้นหาไม่มี is:pr
  • ลบพารามิเตอร์ "readme-ov-file" ที่ซ้ำซ้อนออกจาก URL ของ repo
  • ปิด checks list เมื่อคลิกนอก modal
  • ลิงก์หมายเลขบรรทัดในที่ที่ GitHub ลืมใส่ลิงก์
  • เลื่อน file tree ไปยังไฟล์ปัจจุบัน
  • ปิด modal ที่ลดประสบการณ์ผู้ใช้แทนที่จะเพิ่ม
  • ป้องกันไม่ให้ลิงก์เปิดในแท็บใหม่โดยไม่จำเป็น "Open in new tab" ควรเป็นตัวเลือกของผู้ใช้ ไม่ใช่ค่าดีฟอลต์

เคยเป็นส่วนหนึ่งของ Refined GitHub

GitHub ได้แรงบันดาลใจจาก Refined GitHub และได้เพิ่มฟีเจอร์ของมันเข้าไปในระบบเองหลายสิบรายการ 🎉 ดังนั้นจึงได้ นำออกจากส่วนขยายนี้แล้ว คุณยังสามารถดู ฟีเจอร์เก่าทั้งหมด ของ Refined GitHub ได้ในรายการเดียว

การปรับแต่ง

ฟีเจอร์ส่วนใหญ่สามารถปิดการใช้งานได้หากเป็น JavaScript-based และคุณสามารถ override CSS ของเราได้เองในตัวเลือกของส่วนขยาย

เรายินดีรับข้อเสนอแนะและการมีส่วนร่วม แต่โปรดทราบว่านี่เป็นโปรเจกต์ที่มีความคิดเห็นเฉพาะทางสูง มีเกณฑ์สูงสำหรับการเพิ่มฟีเจอร์ ผู้ใช้จะมีความเห็นไม่ตรงกันเสมอ อย่างไรก็ตาม เรายินดีที่จะพูดคุย
