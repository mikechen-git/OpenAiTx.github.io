什麼是 json-everything？

json-everything 背後的主要理念是確保常見的 JSON 功能在 System.Text.Json 生態中獲得良好支援。

第一步是檢查是否已經有其他專案支援某個功能。

如果有，這裡通常不會重複實作。稍後你會看到一些其他專案的列表，它們對 JSON 做了很酷的事情。

否則，這個專案會考慮納入該功能。

內容介紹

其實有多個「盒子」。每一項功能都被拆分為獨立的函式庫，你可以按需選擇。

所有函式庫皆支援 .Net Standard 2.0 及 .Net 8。

JsonSchema.Net
 JSON Schema，支援第 6 版及以上
JsonSchema.Net.Generation
 從 .Net 類型自動產生 Schema
JsonSchema.Net.Generation.DataAnnotations
 支援 System.ComponentModel.DataAnnotations 的 Schema 產生。
JsonSchema.Net.DataGeneration
 隨機資料產生（由 Bogus 驅動）
JsonSchema.Net.Data
 用於存取實例與外部資料的詞彙
JsonSchema.Net.ArrayExt
 用於擴展陣列驗證的詞彙
JsonSchema.Net.OpenApi
 OpenApi 3.1 詞彙擴充（由 Graeae 使用）
JsonPath.Net
 JSON Path (RFC 9535)
JsonPatch.Net
 JSON Patch (RFC 6902)
JsonPointer.Net
 JSON Pointer (RFC 6901) 及 Relative JSON Pointer (規範)
JsonLogic
 JsonLogic (官方網站)
JsonE.Net
 JSON-e (官方網站)
Json.More.Net
 通用擴充功能，原應包含於 System.Text.Json[.Nodes]，但尚未納入
Yaml2JsonNode
 YamlDotNet 的 YAML 文件模型與 JsonNode 之間的轉換（雙向支援）。

以上所有函式庫皆完整支援其相關規範。

JsonSchema.Net 的錯誤訊息支援下列語言：

  • 義大利語 it
  • 挪威語 nb-NO
  • 波蘭語 pl-PL
  • 俄語 ru
  • 西班牙語 es
  • 瑞典語 sv-SE
  • 土耳其語 tr-TR

注意：每個語言包皆以獨立 Nuget 套件提供。

你也可以在 Bowtie 查看 JSON Schema 測試套件結果，該平台會在多個不同平台的實作上運行測試套件。

免責聲明：我的函式庫 JsonSchema.Net 與網站 https://jsonschema.net 無任何關聯，僅同為優秀的 JSON Schema 工具。

已棄用套件

以下套件過去也曾屬於本套件組，但現已不再支援。

JsonSchema.Net.CodeGeneration
 從 Schema 產生 C# 程式碼 已棄用，建議改用 Corvus.JsonSchema

其他專案的 System.Text.Json 支援

若你在這裡找不到所需功能，請試試下列優秀專案：

  • JsonCons.Net 作者 @danielaparker
    • JSON Pointer
    • JSON Patch
    • JSON Merge Patch
    • JSON Path
    • JMES Path
  • Corvus.JsonSchema 作者 @mwadams
    • 著重於 C# 程式碼產生的 JSON Schema 驗證
  • （更多即將推出）

如果你使用 JSON 完成了這個套件或上述專案未涵蓋的內容，歡迎建立功能需求議題。
